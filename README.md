# <a name="overview"></a>概述

## <a name="learn-how-to-contribute"></a>了解如何参与

任何感兴趣的人都可以参与这些主题。 参与时，您的工作将在合并之后直接进入内容集。 它将显示在 docs.microsoft.com 中，您将在以下位置作为参与者列出 <https://github.com/MicrosoftDocs/microsoft-365-docs/graphs/contributors> 。

### <a name="quickly-update-an-article-using-githubcom"></a>使用 GitHub.com 快速更新文章

仅进行不常用或小更新的参与者可以直接在 GitHub.com 上编辑文件，而无需安装任何其他软件。 本文向您介绍如何操作。 [此两分钟视频](https://www.microsoft.com/videoplayer/embed/RE1XQTG)还介绍了如何参与。

1. 请确保你已使用 GitHub 帐户登录到 GitHub.com。
2. 转到 docs.microsoft.com 上要编辑的页面。
3. 在页面右侧，单击 "**编辑**" （铅笔图标）。

   ![Docs.microsoft.com 上的 "编辑" 按钮](microsoft-365/media/quick-update-edit.png)

4. GitHub 上的相应主题文件打开，您需要单击 "**编辑此文件**" "铅笔" 图标。

   ![Github.com 上的 "编辑" 按钮](microsoft-365/media/quick-update-github.png)

5. 主题在带横线的编辑页中打开，您可以在其中对文件进行更改。 GitHub 中的文件是使用 Markdown 语言编写和编辑的。 有关使用 Markdown 的帮助，请参阅[主控 Markdown](https://guides.github.com/features/mastering-markdown/)。 选择 "**预览更改**" 选项卡可在转到时查看所做的更改。

6. 完成更改后，请转到页面底部的 "**建议文件更改**" 部分：

   - 需要一个简短的标题。 默认情况下，标题是文件的名称，但您可以对其进行更改。
   - （可选）您可以在 "**添加可选扩展说明**" 框中输入更多详细信息。

   准备就绪后，单击 "绿色**建议文件更改**" 按钮。

   !["建议文件更改" 部分](microsoft-365/media/propose-file-change.png)

7. 在出现的 "**比较更改**" 页上，单击 "绿色**创建拉取请求**" 按钮。

   ![比较更改页面](microsoft-365/media/comparing-changes-page.png)

8. 在出现的 "**打开拉取请求**" 页上，单击 "绿色**创建拉取请求**" 按钮。

   ![打开 "拉取请求" 页](microsoft-365/media/open-a-pull-request-page.png)

> [!NOTE]
> 您在存储库中的权限决定了您在最后几个步骤中看到的内容。 没有特殊权限的用户将看到 "**建议文件更改**" 部分以及随后的确认页（如所述）。 具有创建和批准其自己的 pull 请求的权限的用户将看到类似的 "**提交更改**" 部分，其中包含用于创建新分支和较少的确认页面的额外选项。<br/><br/>点是：单击显示给您的任何绿色按钮，直到不再存在。

将会通知主题元数据中标识的编写器，最终会检查并批准您所做的更改，以便主题将在 docs.microsoft.com 上更新。 如果有问题或更新中存在问题，则编写人员将与你联系。

### <a name="contributing"></a>参与

此项目欢迎发布内容和建议。  大多数发布要求您同意参与者许可协议（CLA）声明您有权使用您的发布内容，并实际为我们授予权限。 有关详细信息，请访问 <https://cla.microsoft.com> 。

提交拉取请求时，CLA 将自动确定是否需要提供 CLA 并相应地装饰 PR （例如，标签、注释）。 只需按照 bot 提供的说明操作即可。 您只需在所有存储库使用我们的 CLA 执行此操作一次。
