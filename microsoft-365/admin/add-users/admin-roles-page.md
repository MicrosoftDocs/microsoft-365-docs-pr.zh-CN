# <a name="get-started-with-the-roles-page"></a>"角色" 页面入门

在 "角色" 页上，可以向用户授予在管理员中心执行任务的权限。 这可帮助您的组织将任务传播给适当的人员，并帮助保护数据的安全。

:::image type="content" source="../../media/roles-main-page.png" alt-text="显示管理员角色的图":::

> [!TIP]
> 是否在查找详细的角色说明？ 查看[Azure Active Directory 中的管理员角色权限](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles#available-roles)和[关于管理员角色的相关信息](https://docs.microsoft.com/office365/admin/add-users/about-admin-roles)。

## <a name="about-the-admin-roles-page"></a>关于 "管理员角色" 页

您可以导出管理列表以及按角色搜索和筛选。

:::image type="content" source="../../media/admin-role-page-options.png" alt-text="筛选或导入管理员角色":::

|||
|:-----|:-----|
|  <br/> |使用 "**导出管理列表**" 获取组织中所有管理员用户的完整列表。 该列表存储在一个 Excel .csv 文件中。   <br/> |
|  <br/> |使用 "**搜索**" 搜索管理员角色，并查看分配到该角色的用户。   <br/> |
|  <br/> |使用**筛选器**更改所显示的管理员角色的视图。   <br/> |

## <a name="get-the-most-out-of-the-roles"></a>充分利用角色

阅读下面的内容，了解不同的管理员角色以及角色可以在组织中执行的任务。

> [!NOTE]
这并不是这些角色所具有的所有权限的详尽列表。 有关每个角色的详细信息，请选择 "**了解更多**" 链接。

### <a name="exchange-admin"></a>Exchange 管理员

将 Exchange 管理员角色分配给需要查看和管理您的用户的电子邮件邮箱、Microsoft 365 组和 Exchange Online 的用户。 他们还可以打开和管理 Microsoft 支持的服务请求。 [了解更多](https://docs.microsoft.com/office365/admin/add-users/about-exchange-online-admin-role)

### <a name="global-admin"></a>全局管理员

将全局管理员角色分配给需要跨 Microsoft online services 对大多数管理功能和数据进行全局访问的用户。 为太多的用户提供全局访问权限会带来安全风险，我们建议你设置 2 至 4 名全局管理员。 只有全局管理员才能为所有用户重置密码，并可对域进行添加和管理。 他们还可以打开和管理 Microsoft 支持的服务请求。 注册 Microsoft online services 的人员将自动成为全局管理员。[了解详细信息](https://docs.microsoft.com/office365/admin/add-users/about-admin-roles#roles-available-in-the-microsoft-365-admin-center)

### <a name="global-reader"></a>全局读取者

将全局读者管理员角色分配给需要查看全局管理员可以查看的所有管理中心的管理功能和设置的用户。 全局读者管理员角色无法编辑任何设置。 [了解更多](https://docs.microsoft.com/office365/admin/add-users/about-admin-roles#roles-available-in-the-microsoft-365-admin-center)

### <a name="helpdesk-admin"></a>支持管理员

将 "支持人员" 管理员角色分配给要重置密码的用户，强制用户注销任何安全问题。 他们还可以打开和管理 Microsoft 支持的服务请求。 帮助台管理员只能帮助非管理员用户和用户分配这些角色：目录读取器、来宾邀请者、支持人员管理员、消息中心读取器和报告读取器。 [了解更多](https://docs.microsoft.com/office365/admin/add-users/about-admin-roles#roles-available-in-the-microsoft-365-admin-center)

### <a name="service-admin"></a>服务管理员

将服务管理员角色分配给需要为 Azure、Microsoft 365 和 Office 365 服务创建服务请求的用户。 [了解更多](https://docs.microsoft.com/office365/admin/add-users/about-admin-roles#roles-available-in-the-microsoft-365-admin-center)

### <a name="sharepoint-admin"></a>SharePoint 管理员

在购买 Microsoft 365 订阅时，会自动创建一个工作组网站，全局管理员将设置为主网站集管理员。 将 SharePoint 管理员角色分配给要访问 SharePoint 管理中心的用户。 具有 SharePoint 管理员角色的用户可以创建和管理网站集，指定网站集管理员和管理用户配置文件。 具有 SharePoint 管理员角色的用户还可以通过 Microsoft 支持管理 Microsoft 365 组并打开服务请求。 [了解更多](https://docs.microsoft.com/sharepoint/sharepoint-admin-role)

### <a name="teams-service-admin"></a>Teams 服务管理员

将 "团队管理员" 角色分配给要访问和管理 & Skype 管理中心的团队的用户。 拥有团队管理员角色的用户也可以通过 Microsoft 支持管理 Microsoft 365 组并打开服务请求。 [了解更多](https://docs.microsoft.com/MicrosoftTeams/using-admin-roles)

### <a name="user-admin"></a>用户管理员

将用户管理员角色分配给要访问和管理用户密码重置和管理用户和组的用户。 他们还可以打开和管理 Microsoft 支持的服务请求。 [了解更多](https://docs.microsoft.com/office365/admin/add-users/about-admin-roles#roles-available-in-the-microsoft-365-admin-center)

::: moniker range="o365-worldwide"

## <a name="compare-roles"></a>比较角色

在管理中心：

- 最大选择3个角色，并选择 "**比较角色**" 以查看每个角色具有的权限。
