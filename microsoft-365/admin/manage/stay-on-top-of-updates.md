# <a name="stay-on-top-of-office-365-changes"></a>掌控 Office 365 更改

使用 Office 365，将在第一时间接收到可用的新产品更新和功能，而不是在几个月或几年后才能使用计划更新。因此，你和你的用户经常将定期体验新增和改进的工作方式，而无需进行代价高昂且耗时的全公司范围的升级。使用这种模型工作的挑战在于随时掌握更改和更新。使用以下方式可在组织内掌控 Office 365 更新。

## <a name="stay-on-top-of-office-365-changes"></a>掌控 Office 365 更改

||||
|:-----|:-----|:-----|
|**功能** <br/> |**说明** <br/> |**如何使用** <br/> |
|**消息中心** <br/> |了解官方服务通知和功能更改。 您可以在 Microsoft 365 管理中心、管理移动应用程序中阅读这些邮件，或在电子邮件中接收每周摘要。 当您看到其他人应对其执行操作的邮件时，与组织中的其他人共享这些邮件。 也可使用服务通信 API 检索消息。  <br/> |登录[管理中心](../admin-overview/about-the-admin-center.md)或[管理移动应用](../admin-overview/admin-mobile-app.md)。 选择 "**运行状况** \> **消息中心**"。 选择要阅读或共享的消息。  <br/> 通过选择 "管理中心" 中的 "**编辑首选项**"，更改您看到的有关邮件的服务或选择加入每周摘要。 你也可以选择退出每周摘要。  <br/> [Office 365 消息中心概述](message-center.md) <br/> |
|**定向发布** <br/> |为自己和组织中的一组选定人员注册定向发布。优先获取最新 Office 365 更新，然后帮助用户了解最新更新或对用户进行相关培训。  <br/> |登录[管理中心](../admin-overview/about-the-admin-center.md)或[管理移动应用](../admin-overview/admin-mobile-app.md)。 Selece**设置** \> **组织配置文件** \>的**发布首选项**。 了解有关[定向发布](release-options-in-office-365.md)的详细信息。  <br/> |
|**产品指南** <br/> |请访问 Microsoft 365 路线图，了解已启动、正在开发、已被取消或之前已发布的功能。 产品指南是 Office 365 更新和更改的官方网站。  <br/> |请经常访问[Microsoft 365 路线图](https://www.microsoft.com/microsoft-365/roadmap)并了解计划的更新和版本。  <br/> |
|**博客** 和 **社区** <br/> |访问 Office 博客、Microsoft 社区和 Microsoft 技术社区，了解有关 Office 365 中更改的详细信息，并与其他用户分享体验。  <br/> |访问 [Office 博客](https://www.microsoft.com/en-us/microsoft-365/blog/)。访问 [Microsoft 社区](https://answers.microsoft.com)。访问 [Microsoft 技术社区](https://techcommunity.microsoft.com)。  <br/> |

> [!NOTE]
> 您必须是全局管理员才能对发布首选项进行更改。
