# <a name="cortana-in-microsoft-365"></a>Microsoft 365 中的 Cortana

Cortana 是个人生产力助手，可提供由 AI 提供的体验，以节省时间并专注于最重要的事项。 Cortana 将帮助你的用户在工作和生活这一天提高个人工作效率。 使用有效的工作或学校帐户登录后，用户可以在 Microsoft 365 体验中通过 Cortana 获得基于云的协助服务，这些服务符合 Office 365 的企业级隐私、安全性和合规性承诺 ("Cortana 企业服务") 。 

作为个人生产力助手，Cortana 旨在提供可安全处理 Office 365 数据（如电子邮件、文件、聊天等）的功能，以节省时间、提高效率并提高用户的工作效率。

今后，我们将 Cortana 的重心放在企业生产力上。

- 与其他 Office 365 服务一致，Cortana 企业服务符合联机服务条款 (OST) 中反映的企业级隐私 [、安全性和合规性承诺 ](https://www.microsoft.com/licensing/product-licensing/products)。

- 新的 Microsoft 365 体验（如"简介电子邮件"和"播放我的电子邮件"）将使用 Cortana 企业服务启用，并完全遵守这些承诺。 这些功能目前可用于全球 (多租户) 。 有关查找使用位置详细信息，请访问 [查看帐户的其他属性值](https://docs.microsoft.com/microsoft-365/enterprise/view-user-accounts-with-microsoft-365-powershell?view=o365-worldwide#view-additional-property-values-for-accounts)。

- 用户可以通过 Windows 10 (版本 2004 和更高版本) 中的 Cortana 连接到此处所述的 Cortana 企业服务，以及客户端应用程序（如 Outlook for iOS 和 Outlook for Android）以遵守单独的许可条款。 

- 现有消费者体验（包括 Windows 10 (版本 1909 和更早版本) 中的 Cortana 以及 iOS 和 Android 上的 Cortana 应用）受 [Microsoft](https://www.microsoft.com/licensing/product-licensing/products) 服务协议和 [Microsoft](https://privacy.microsoft.com/privacystatement) 隐私声明 (管理，请参阅) 下面的"适用于消费者的现有服务"部分。 这些条款还将管理使用使用者凭据登录时提供给用户的 Cortana 企业服务。

## <a name="what-data-is-processed-by-cortana-enterprise-services"></a>Cortana 企业服务处理哪些数据？ 

Cortana 企业服务处理来自用户的查询、满足用户请求所需的 Office 数据，以及由 Microsoft 系统生成的其他遥测以运行该服务。 Cortana 企业服务收集的数据包括用户语音查询的文本表示形式，例如 (语音识别和语音) 。 此文本数据是客户数据，它根据联机服务条款  [进行管理](https://www.microsoft.com/licensing/product-licensing/products)。 它仅用于开发和改进与联机服务条款一致的机器学习模型。

## <a name="what-is-the-governance-model-for-customer-data-in-cortana-enterprise-services"></a>Cortana 企业服务中的客户数据的管理模型是什么？

与其他 Office 365 服务一致，Cortana 企业服务是安全的，并受 [联机服务条款保护](https://www.microsoft.com/licensing/product-licensing/products)。 这包括一组承诺，用于保护客户数据，防止意外丢失、更改、未经授权披露或访问或非法销毁。 客户数据还受到严格的访问限制。 Microsoft 仅使用客户数据来提供所同意的服务，并且出于与这些服务兼容的目的。 有关详细信息，请参阅下表。

## <a name="how-does-microsoft-store-retain-process-and-use-customer-data-in-cortana"></a>Microsoft 如何在 Cortana 中存储、保留、处理和使用客户数据？

下表介绍了 Cortana 企业服务的数据处理。

| 名称 | 说明 |
|:-----|:-----|
|**存储**  <br/> |客户数据存储在 Office 365 云内的 Microsoft 服务器上。 你的数据是租户的一部分。 <br/><br/>语音音频不会保留。  <br/> |
|**保留在地理位置中**  <br/> |客户数据存储在地理位置中 Office 365 云内的 Microsoft 服务器上。 你的数据是租户的一部分。  <br/> |
|**保留**  <br/> |当租户管理员关闭帐户或提出 GDPR 数据主体权限删除请求时，将删除客户数据。 <br/><br/>语音音频不会保留。  <br/> |
|**处理和保密性**  <br/> |参与处理客户数据和个人数据的人员 (i) 将仅根据客户的指示处理此类数据， (ii) 将负责维护此类数据的机密性和安全性，即使他们的参与终止。  <br/> |

## <a name="cortana-enterprise-services-in-microsoft-365-experiences"></a>Microsoft 365 体验中的 Cortana 企业服务

### <a name="cortana-in-windows-10-version-2004-and-later"></a>Windows 10 中的 Cortana (版本 2004 及更高版本) 

Windows 10 中的 Cortana 应用可帮助用户快速获取 Microsoft 365 的信息，使用键入的或说出的查询与人员联系、检查日历、设置提醒等。

Windows 中的 Cortana 可帮助用户更好地管理他们的日程安排和任务。 他们可以在 Microsoft To Do 中添加列表、查找本地信息、获取定义，以及使用必应搜索跟踪最新新闻、天气和财务信息。

Windows 10 版本 2004 及更高版本中的 Cortana 符合 Cortana 企业服务的企业级隐私、安全性和合规性承诺，如联机服务条款 [ (OST ](https://www.microsoft.com/licensing/product-licensing/products)) 中反映。

### <a name="how-to-opt-out-of-cortana-in-windows-10"></a>如何在 Windows 10 中选择退出 Cortana

管理员可以使用 Experience\AllowCortana MDM 策略或通过组策略：计算机配置\管理模板\Windows 组件\搜索\允许 Cortana，在 Windows 10 中为组织配置 Cortana。

从 Windows 10 版本 2004 开始，Cortana 是预安装 Windows 的通用 Windows 平台 (UWP) 应用，并通过 Microsoft Store 定期更新。 若要接收 Cortana 的最新更新，你需要通过 [Microsoft Store 启用更新](https://docs.microsoft.com/windows/configuration/stop-employees-from-using-microsoft-store)。

[详细了解 Windows 10 中的 Cortana](https://docs.microsoft.com/windows/configuration/cortana-at-work/cortana-at-work-overview)

### <a name="cortana-voice-assistance-in-teams"></a>Teams 中的 Cortana 语音协助

通过 Teams 移动应用和 Microsoft Teams 显示设备的 Cortana 语音协助，Microsoft 365 企业版用户可以使用自然语言语音简化通信、协作和与会议相关的任务。 用户可以通过选择位于 Teams 移动应用右上角的麦克风按钮或在 Microsoft Teams 屏幕中说出 &#8220;Cortana&#8221;与 Cortana 说话。 为了快速与团队自由联系，用户可以说查询（如 &#8220;呼叫 Megan&#8221; 或 &#8220;向下一个会议&#8221;。 用户还可通过说&#8220;加入下一&#8221;会议并使用语音协助来共享文件、查看其日历等来加入会议。 这些语音协助体验使用 Cortana 企业级服务提供，这些服务完全遵循 Office 365 的隐私、安全性和合规性承诺，如联机服务条款 [ (OST ](https://www.microsoft.com/licensing/product-licensing/products)) 。

**管理员控制**

默认情况下，将为租户启用 Cortana 语音协助。 管理员可以控制租户中的哪些人可以通过 TeamsCortanaPolicy (Teams 中的 Cortana 语音) 。 可以在用户帐户级别或租户级别设置此策略。 管理员还可使用此策略控件中的 CortanaVoiceInvocationMode 字段来确定 Cortana 是禁用、仅通过一键调用启用，还是使用唤醒词调用启用，以及适用于支持 Cortana 的设备的 (，如 Microsoft Teams) 。 请注意，在针对美国英语版 Microsoft 365 企业版用户发布初始版本时，Teams 移动应用将不支持唤醒词激活，但将来将受支持。

**用户控件**

单个用户可以单击麦克风按钮，在 Teams 移动应用中试用 Cortana 语音帮助。 他们只需说 &#8220;Cortana.&#8221;，就可以在 Microsoft Teams 显示设备上试用 Cortana 语音帮助&#8221;他们还可以控制 Teams 中的 Cortana 是否通过 Teams 移动应用中的设置或 Microsoft Teams 显示器上的设置为设备启用： 

1. 打开 Teams 移动应用，或转到 microsoft Teams () 环境。

2. 在 Teams 移动应用中，转到"设置 **"。** 在 Microsoft Teams 显示器上，选择用户头像，然后选择"设置"。 如果 Cortana 已启用，例如，&#8220;Cortana，请转到"设置".&#8221;

3. 选择 **Cortana**。

4. 根据你是否希望Cortana在设备上获得语音帮助，将开关移到"开"或"关"。

[了解有关 Teams 中的 Cortana 语音协助](https://docs.microsoft.com/microsoftteams/cortana-in-teams)

### <a name="play-my-emails"></a>播放我的电子邮件

通过 Outlook mobile) 连接到的"播放我的电子邮件" (是一种语音驱动的无手体验，用户可在重点收件箱中收听新邮件，并通过手机上的扬声器、耳机或已连接的音频设备更改其一天。 用户可以要求 Cortana 大声读出最近发送的电子邮件，并要求 Cortana 执行标记、存档、删除和跳过邮件等操作。 此功能对于在上下班、多任务或下班时跟进电子邮件尤其有用。 当用户在"播放我的电子邮件"中与 Cortana 交谈时，语音音频请求会直接转到 Cortana 企业服务。 在 Office 365 云中处理用户电子邮件的文本到语音读出。 在此过程中，不会在用户的移动设备上处理 Office 365 数据，并且不会保存任何电子邮件数据。 根据 Microsoft [Online Services](https://www.microsoft.com/licensing/product-licensing/products)条款中的数据保护条款，可以保留语音命令的脚本 (例如"标记为已读"、"next"、"flag"等) 。

当电子邮件受保护时，Cortana 将发出呼叫，在阅读邮件之前短暂暂停，以便为用户提供足够的时间来暂停播放或跳到下一封邮件。 与专用电话呼叫类似，用户应在可能听到机密信息的位置启动播放时保持谨慎。 在这些情况下，建议组织的员工在 Outlook 移动版中使用"播放我的电子邮件"时，在适当的环境中佩戴耳机。

### <a name="opt-out-of-play-my-emails"></a>选择退出"播放我的电子邮件"

个人可以使用以下步骤选择退出"播放我的电子邮件"。

1. 打开 Outlook 移动版。

2. 转到"**设置"。**
  
3. 选择 **"播放我的电子邮件"。**

4. 在要禁用的帐户上将切换开关移到关闭状态。

[了解有关"播放我的电子邮件"](https://support.microsoft.com/help/4558256)

### <a name="briefing-email"></a>简介电子邮件

Cortana 发送一封个性化的简介电子邮件，其中包含你通过便捷方式将任务和承诺标记为已完成或安排专注时间完成这些任务和承诺。 它还包括一天的会议和相关文档的摘要。 Cortana 从用户的电子邮件中提取信息，并存储在其 Exchange Online 邮箱中，直到它合并到"简介"电子邮件中。 在 Exchange Online 邮箱之外，个人数据不会访问。

### <a name="how-to-opt-out-of-briefing-email"></a>如何选择退出简介电子邮件

管理员可以使用 Exchange Online 中的 [PowerShell](https://docs.microsoft.com/briefing/be-admin) 为组织配置简介。 个人可以通过在邮件的页脚中选择"取消订阅"来选择退出 Cortana 的简介电子邮件。

[了解有关简介电子邮件的更多信息](https://support.microsoft.com/help/4558259)

## <a name="how-is-the-delivery-of-cortana-enterprise-services-different-from-the-delivery-of-other-cortana-features-i-may-have-previously-experienced"></a>Cortana 企业服务的交付与之前可能体验过的其他 Cortana 功能的交付如何不同？

以下是考虑 Cortana 在企业中的工作方式的两种方法：

**具有 Cortana 企业服务** 的组织的新体验：Cortana 企业服务旨在满足组织的安全性和合规性需求： 

1. 这是一项新服务，本文档对此进行了讨论。

2. 对于受联机服务条款限制的服务，Microsoft 是数据处理者：Microsoft 收集并使用来自客户的客户数据，只是提供客户请求的在线服务以及客户指示的目的。 根据欧盟的《一般数据保护条例 (GDPR) ，客户是数据控制者。 请参阅[联机服务条款](https://www.microsoft.com/licensing/product-licensing/products)[，并针对我们的商业云客户引入更多隐私透明度](https://blogs.microsoft.com/eupolicy/2019/11/18/introducing-privacy-transparency-commercial-cloud-customers/)。

3. 例如，"播放我的电子邮件"是一项 Cortana 服务，你的用户可以通过 Outlook for iOS 连接到该服务并利用 Cortana 企业服务。 

4. IT 管理员将始终控制 Cortana 的可选连接体验，类似于使用 Office 专业增强版应用程序的可选连接体验。  

面向消费者的现有服务：Cortana 可选连接服务主要针对消费者体验设计，当前在 Windows 10 (版本 1909 及更早版本) 以及 iOS 和 Android 上的 Cortana 应用中提供。

1. 这些体验支持天气、新闻和流量等功能。

2. 租户管理员可以控制 Windows 10 (版本 1909 和更早版本) 中的 Cortana 以及 iOS 和 Android 上的 Cortana 应用是否可以允许 Cortana 连接到 Office 365 租户数据。

关闭 Cortana 对组织的 Microsoft 托管数据的访问权限

1. 在 Microsoft 365 管理中心中，选择 **"设置**  >  **组织** 设置"，然后选择 **Cortana。**

2. 取消选中 **"在 Windows 10 (版本 1909** 和更早版本) 中允许 Cortana，以及 iOS 和 Android 版 Cortana 应用"的复选框，以代表组织人员访问 Microsoft 托管的数据，以禁用 Cortana 连接体验。

3. 选择“**保存更改**”。

对于受 Microsoft 服务协议 [和](https://go.microsoft.com/fwlink/p/?LinkId=2109174) Microsoft [隐私声明约束的服务](https://privacy.microsoft.com/privacystatement)，Microsoft 是数据控制者。 作为数据控制者，Microsoft 根据 Microsoft 隐私声明使用数据改进 [产品和服务](https://privacy.microsoft.com/privacystatement)。
