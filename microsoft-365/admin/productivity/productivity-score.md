# <a name="microsoft-productivity-score-preview"></a>Microsoft 工作效率分数（预览）

Microsoft 365 提供了生产效率解决方案，使组织能够实现其业务目标。 生产效率分数提供了使用这些解决方案的见解，您可以转换为完成工作的方式。它包含： 

- **可见性**，帮助他们了解用户如何以其当前分数的形式在各类员工体验和技术体验（即将推出）中进行细分。 
- **见解**，以确定启用改进体验的机会。 
- 更新技能和系统的**操作**，以便每个人都可以完成其最佳工作。 

分数和洞察力在两个类别中都存在–  

- **员工体验：** 展示了 Microsoft 365 如何帮助创建高效且接洽的员工，具体方法是量化用户如何对内容进行协作、随时随地了解通信方式、了解通信方式以及如何开发会议区域性（即将推出） 

- **技术经验**（即将推出）：帮助你优化设备体验，例如主动修复常见的帮助台问题，并改善电脑启动时间和网络，以确保你的应用程序能够正常运行。  

## <a name="how-it-works"></a>工作原理

### <a name="areas-of-scoring"></a>记分区域 

在每个分数类别中，工作效率分数可提供跨区域的工作转换的见解。 员工体验中涉及的领域包括：

- 协作处理内容  
- 随时随地工作 
- 了解通信样式（即将推出） 
- 开发会议区域性（即将推出） 

### <a name="scoring-user-actions"></a>评分用户操作 

在每个区域中，我们衡量基于研究的关键操作，这些操作代表使组织能够转换为高效组织的工作方式。 对于每个区域，我们将计算在最近28天内执行这些操作的用户的百分比。 

员工体验成绩在所有领域都 curated 为平均分数。 请注意，随着时间的推移，我们将不断向员工体验和技术体验中添加更多的领域。 

### <a name="products-included-in-the-productivity-score"></a>生产分数中包括的产品 

工作效率分数目前包含来自密钥 Microsoft 365 工作负载 OneDrive、SharePoint、Word、Excel、PowerPoint、OneNote、Outlook、Yammer、工作组、Skype 的信号。 

你的得分每天更新，并反映最近28天（包括当前日期）内完成的用户操作。

## <a name="access-and-required-permissions"></a>访问权限和所需权限 

对于员工体验，您必须订阅 Office 365 for business plan 或包含多个用户的 Microsoft 365 业务计划。 

若要拥有访问 Microsoft 工作效率得分的权限，您必须具有以下角色： 

- 全局管理员 
- Exchange 管理员 
- SharePoint 管理员 
- Skype for Business 管理员 
- Teams 管理员 
- 全局读取者 
- 报告读者 

您可以通过在左侧导航栏中选择 "**报告** > **效率分数**" 来访问 Microsoft 365 管理主页中的体验。

## <a name="interpreting-productivity-score"></a>解释工作效率分数 

### <a name="learn-how-your-organization-works"></a>了解你的组织的工作原理 

他的工作效率分数主页以百分比为单位提供当前分数和历史记录，通过基准补充的每个分数类别中的区域的主要见解。 

![工作效率分数主页](../../media/pslanding.png)

1.**工作效率分数**按百分比和<numerator> / <denominator>格式列出，因此您还可以看到绝对积分（分子）和最大可能分数  
1. 此数据透视允许您选择要重点关注的分数类别。 在预览中，您只能查看**员工体验**，但所选内容将很快扩展以包括**技术经验**。 
1. **对等基准**允许您将当前分数与像你这样的组织进行比较。 基准度量值计算为一组相似组织中的度量值的平均值。 该集由在 Microsoft 365 或 Office 365 中具有相似数量的已启用用户、地区、拥有的许可证类型、行业和 tenure 的组织组成。 
1. "**得分类别**" 一节提供了有关每个类别的基准的细分效率分数。
1. **分数历史记录**显示在过去6个月内每个类别中的得分如何移动。
1. 对**内容进行协作**的主要洞察力显示了与此领域中的基准相关的基准的相关措施。 选择 "**查看详细信息**" 以查看 "区域详细信息" 页。
1. 大部分**工作**的主要洞察力显示了与此领域中的基准相关的包含基准的相关措施。 选择 "**查看详细信息**" 以查看 "区域详细信息" 页。  

### <a name="insights-to-identify-opportunities-to-improve-experiences"></a>深入了解如何发现改进体验的机会 

对于每个区域，请在主要洞察力中选择 "**查看详细信息**" 以查看显示其他见解、相关研究和操作的 "区域详细信息" 页，以提高成绩。 .

所有区域详细信息页面都遵循以下结构： 

- 左上角：从主页中保留的主要洞察力。
- 左下角：支持具有上下文操作的见解。  
- 权限：演示区域和工作效率之间的关系的学术研究。

此外，所有见解都包含以下元素： 

- 标头：针对每个洞察力跟踪的关键好处在于或结果;
- 摘要：进一步说明了为什么洞察力中的度量对于组织的工作效率非常重要。 
- 数据可视化：直观地量化组织相对于洞察力中的衡量措施（适用时）的位置。
- 操作：上下文操作，可帮助根据洞察力推动更多所需的结果，从而提高你的成绩。

### <a name="area-detail-page--collaborating-on-content"></a>区域详细信息页面–协作处理内容 

![内容页上的 Collaborationg。](../../media/pscollab.png)

"在内容中进行协作" 页面包含来自工作效率分数主页的主要洞察力。 它具有以下组件：

1. 标头：量化对内容进行协作的用户百分比（关键结果）。
1. 正文：更详细地介绍了协作导致更高的工作效率的方式。
1. 可视化：这将通过基准来直观地量化为此区域的分数分配的每个元素：

    - **读者**：在启用了 Onedrive 和 sharepoint 的用户的基础上量化访问或下载云（Onedrive 和 sharepoint）中的文件的用户。
    - **作者**：在启用了 OneDrive 和 sharepoint 的用户的基础上，量化用户修改、上载、同步、签入、复制或移动云（Onedrive 和 sharepoint）上的文件。
    - **协作**者：在启用了 Onedrive 和 sharepoint 的用户的基础上，对用户进行协作处理云（Onedrive 和 sharepoint）上的文件。 如果两个用户在创建或修改了 Word、Excel、PowerPoint、OneNote 或 PDF 云文档之后在28天的窗口中对其进行读取或编辑，则这两个用户是协作者。

1. **为什么它会**在云文件上提供工作效率的研究链接协作摘要。 **阅读 Forrester 中的信息检索**转到一个研究文章，该文章提供了有关如何 conducive 协作以提高生产率的更多相关信息。
1. 对**多个文件进行协作**的支持洞察力量化了组织中的用户如何跨不同的文件进行协作。 
1. 如果用户是在云上共享内容，同时在内部和外部共享之间进行区分，则支持**共享云文件**的洞察力。
1. 如果用户通过电子邮件共享文件使用的是附件，而不是指向云文件的链接，则支持对**电子邮件云文件**的洞察力。

考虑协作的文件类型为 Word/Excel/PowerPoint/OneNote/PDF。

### <a name="actions-for-update-skills-and-systems"></a>更新技能和系统的操作 

为了提升 actionability，每个支持性洞察力都包含可帮助你转换你的员工体验的在线操作，其中一些可能是配置更改或知晓意识活动。 单击 "这些呼叫" 将在您可能考虑的推荐动作面板选项中打开。 目前启用了对**内容进行协作**的操作。

!["在 Office 文档中协作" 页面。](../../media/psactions1.png)

1. **查看操作**将转到一个窗格，其中列出了与协作处理内容相关的所有操作。
1. **鼓励用户进行协作**，以使用共同创作的教学视频列表，如共同创作和 @mentions 主题。 
1. **自定义外部共享**设置转到有关外部共享的概述的页面，该页面描述了用户共享时所发生的情况，具体取决于他们共享的内容以及与谁共享的内容。  
1. **使用共享链接替换附件**转到说明如何通过电子邮件共享 OneDrive 链接而不是附件的页面，以实现更好的协作。

选择任何动作按钮将打开右侧的 "**建议操作**" 窗格，其中包含每个操作的摘要和指向文档页面的链接。
