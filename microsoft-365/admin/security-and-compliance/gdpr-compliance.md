# <a name="gdpr-simplified-a-guide-for-your-small-business"></a>GDPR 简化：小型企业指南

 *使用 Microsoft 365 企业应用来缓解和管理 GDPR 合规性* 
  
GDPR (《一般数据保护条例) 》是欧盟 (欧盟) 法规，规定组织应如何处理个人数据。 GDPR 将于 2018 年 5 月 25 日生效。 如果你的企业向欧盟出售、提供服务或雇用欧盟公民 [，GDPR](https://ec.europa.eu/info/law/law-topic/data-protection/reform/what-does-general-data-protection-regulation-gdpr-govern_en) 将影响你。 
  
本文可帮助你了解 GDPR 是什么、它的原因以及适用于Microsoft 365如何帮助你的组织遵守 GDPR。
  
## <a name="a-quick-overview-of-the-gdpr"></a>GDPR 快速概述

GDPR 是欧盟法规，它更新并扩展了之前在 1995 年首次 (DPD) 数据保护指令。 GDPR 涉及个人数据（客户、客户、员工或业务合作伙伴的个人）的隐私。 GDPR 的目标是加强欧盟公民的个人数据保护，无论他们驻留在欧盟还是其他地方。 该法规设定了预期，并就如何达到这些目标提出了建议。 当 GDPR 于 2018 年 5 月底生效时，组织必须制定满足 GDPR 要求的措施。
  
GDPR 与数据及其使用方式有关。 将数据视为具有生命周期。 循环从收集数据时开始，在存储数据时继续，并使用它 (处理) 数据，在从系统完全删除数据时结束。 
  
GDPR 涉及以下类型的数据： 
  
- **个人数据：** 如果你可以将数据链接到个人并识别他们，则该数据被视为符合 GDPR 的个人数据。 个人数据的示例包括姓名、地址、出生日期和 IP 地址。 GDPR 甚至将编码的信息 (也称为"假名"信息) 是个人数据，无论数据有多模糊或技术如何（如果数据可以链接到个人）。
    
- **敏感的个人数据** 这是向个人数据添加更多详细信息的数据。 示例包括国家/地区、欧盟成员身份、族裔等。 敏感的个人数据还包括生物识别数据和数据处理。 根据 GDPR，敏感数据的保护规则比个人数据更为严格。 
    
## <a name="gdpr-terms"></a>GDPR 条款

你将看到 GDPR 中经常引用的一些术语。 了解这些术语非常重要。
  
 **同意**
  
GDPR 规定："个人数据的处理应设计为为用户提供服务。" GDPR 希望在处理个人数据时征得同意来实现此目标。 这可能是询问客户是否要从你的公司接收电子邮件的简单操作。 这也意味着当你想要使用营销数据时，你的网站上不再有选择退出复选框。 你必须使用"明确肯定行为"获得明确同意。 此外，你还需要保留有关何时获得或撤销同意的记录。
  
 **数据主体权利**
  
GDPR 规定了数据主体权利，这意味着，对于他们的个人数据，客户、员工、业务合作伙伴、客户、承包商、学生、供应商等有权：
  
- **告知其数据：** 你必须通知个人你使用其数据。 
    
- **有权访问其数据：** 您必须向个人授予访问您保留的任何数据 (例如，通过使用帐户访问或以某种手动方式) 。 
    
- **请求数据纠正：** 个人可以要求你更正不准确的数据。 
    
- **请求删除数据：** 此权利也称为"擦除权"，允许个人请求公司收集的任何个人数据在使用它或共享它的所有系统中被删除。 
    
- **请求受限处理：** 个人可以请求你禁止或限制其数据。 但是，它仅适用于某些情况。 
    
- **具有数据可移植性：** 个人可以要求将其数据转移到另一个公司。 
    
- **对象：** 个人可以反对其数据用于各种用途，包括直接营销。 
    
- **请求不要受自动决策制定（包括分析）影响：** GDPR 对使用数据来配置文件人员并基于该分析自动做出决策有严格的规则。 
    
## <a name="common-small-business-questions-about-the-gdpr"></a>有关 GDPR 的常见小型企业问题

### <a name="im-a-sole-proprietor---do-i-really-have-to-worry-about-the-gdpr"></a>我是独一无二的一员 -我是否确实需要担心 GDPR？

GDPR 关于你处理的数据，而不是你拥有的员工数量。 它会影响各种规模的公司，甚至影响唯一的公司。 但是，员工少于 250 名的公司确实有一些豁免，例如减少记录保留，但仅当你确定数据处理不会影响个人权利并且偶尔进行处理时。
  
例如，非个人数据的处理将免受免除或需要减少措施。 但是，如果您处理被视为"特殊类别敏感数据"的任何数据，即使只是偶尔处理，您也将必须记录此数据处理。 "偶尔处理"的定义是模糊的，但它旨在应用于一次或很少使用的数据。
  
还应确保您收集的个人数据受到保护。 这意味着您需要对其进行加密，并确保使用至少一个密码控制对它的访问。 将客户数据保留于桌面上的电子表格中，不提供保护不符合 GDPR 预期。 
  
### <a name="how-can-i-tell-if-our-company-website-is-gdpr-compliant"></a>如何判断公司网站是否符合 GDPR？

要询问自己的第一个问题是：您是否正在网站上的任何位置收集个人数据？ 例如，您可能有一个要求输入姓名和电子邮件地址的联系人窗体。 如果你想要发送营销电子邮件，请确保添加一个"选择加入"复选框，准确解释你将使用数据。 只有当收件人选中该框时，您才能将他们的个人数据用于营销目的。
  
此外，检查存储数据的数据库是否受保护。 你的 Web 托管公司或云存储供应商将能够就这一点提出建议。 如果使用适用于Microsoft 365，则数据存储符合 GDPR。 
  
### <a name="my-company-is-outside-europe-does-the-gdpr-really-affect-us"></a>我的公司在欧洲以外。 GDPR 是否确实影响我们？

GDPR 是一项保护欧盟公民的法规。 如果你的公司现在处理欧盟公民，或者希望将来会受到影响。 这适用于居住在欧盟国家/地方的公民和居住在其他地方的公民。
  
请考虑以下示例：
  
- 向欧盟公民雇用汽车的美国政府在收集和处理客户数据时需要满足 GDPR 要求。 当公司接受客户数据并确保数据安全存储时，需要征得公司同意。 他们还需要确保客户可以应用其所有数据主体权利。 
    
- 一家澳大利亚公司在线销售产品，其用户设置在线帐户。 GDPR 数据主体权利与同意适用于打开帐户的欧盟公民。 公司将需要确保客户可以应用其所有数据主体权利。 
    
- 一家国际收藏家收集与人们有关的数据，并使用它发送更新和请求获取资金。 GDPR 规定：'...出于直接营销目的处理个人数据可能会被视为出于合法利益而执行。" 但是，组织应负责证明他们的兴趣优先于数据主体的利益。 公司 (或在这种情况下，) 应始终获得通知、明确、选择加入同意。
    
如果客户数据跨边界移动，GDPR 也适用。 如果使用云计算进行数据存储，则需要确保服务完全符合 GDPR。 如果数据存储在数据保护记录较差的位置，则可能会变得很复杂。 如果使用适用于Microsoft 365，我们将提供正确的法律文档，以涵盖 GDPR 要求。
  
### <a name="sure-i-collect-data-but-some-other-company-stores-it-does-that-get-me-off-the-hook"></a>当然，我收集数据，但有些其他公司会存储数据。 这是否将我挂机？

根据 GDPR，如果你收集数据，你在一定程度上受到影响。 GDPR 具有数据处理者和数据控制者的概念： 
  
- **数据控制者：** 个人或组织 (可以拥有联合) 控制者，以决定如何、什么以及为什么收集数据。 他们可能会使用另一个公司的云服务器来存储它。 例如，收集客户数据的网站是控制者。 
    
- **数据处理者：** 代表控制者存储数据的个人或组织 (数据) 根据请求处理这些数据。 例如，Microsoft 365 商业应用版数据存储充当处理者，并且完全符合 GDPR。 
    
    组织或系统可以同时充当控制者和处理者。 Microsoft 365企业版可以充当这两者并符合 GDPR。
    
### <a name="can-i-still-send-out-marketing-emails-to-my-old-customers"></a>我是否仍可以将营销电子邮件发送给我的旧客户？

你需要确保你的客户，甚至是那些已经拥有的客户，已同意使用他们的数据进行市场营销。 你之前可能已获取同意，以及显示同意的记录。 如果是这样，则你已设置为继续营销。 如果没有，你需要从客户获取权限才能继续营销他们。 这通常涉及到发送一封电子邮件，要求客户转到您的网站并选择一个选项以同意接收以后的电子邮件。 
  
### <a name="do-i-have-to-worry-about-the-gdpr-when-i-recruit-new-employees-what-about-current-employees"></a>当招募新员工时，我是否担心 GDPR？ 当前员工呢？

GDPR 不仅影响客户数据，还影响客户数据。它扩展到员工数据。 新招募人员通常使用社交媒体平台（如 LinkedIn）来定位。 未经明确许可，请确保不要存储任何潜在的招募数据。
  
对于现有员工和新员工合同，合同末尾的签名不一定假定同意，尤其是在合同中使用的非肯定条款时。 在这种情况下，必须以与子句关联的明确方式获取同意。 这意味着取决于员工合同，但在某些情况下，您可以使用"合法利益"并添加员工数据处理通知，以确保员工了解您将使用其数据执行哪些操作。
  
## <a name="satisfy-privacy-concerns-using-microsoft-365-for-business"></a>使用适用于企业Microsoft 365隐私问题

遵守 GDPR 与确保个人数据受到保护有关。 GDPR 有一个称为"设计时的隐私"和"默认"的概念。 这意味着数据保护应"提供"到系统和产品，以便解决隐私问题成为第二性质。 
  
与大型企业一样，小型企业需要便利，而不会降低安全性。 Microsoft 365企业版专为员工不足 300 人的公司设计。 小型企业可以使用 Microsoft 基于云的工具来提高业务效率。 通过Microsoft 365，小型企业可以管理电子邮件、文档，甚至会议和事件。 它还具有内置的安全措施和设备管理，它们对于 GDPR 合规性至关重要。
  
Microsoft 365企业组织可以通过以下方式帮助你完成 GDPR 流程：
  
- **发现：** 实现 GDPR 合规性的一个重要步骤是了解你拥有的数据。 
    
- **管理：** 控制对数据的访问并管理其使用是 GDPR 的一个组成部分。 Microsoft 365企业策略基于要应用于设备的策略保护业务数据。 在员工远程工作的年龄中，设备管理至关重要。 Microsoft 365企业应用包括设备管理功能，这些功能确保数据在所有设备上受到保护。 例如，你可以指定企业Windows 10设备都通过安全Windows Defender。 
    
- **保护：Microsoft 365** 企业安全设计。 其设备管理和数据保护控件可跨业务网络（包括远程设备）运行，以帮助确保数据安全。 Microsoft 365商业应用提供了一些控件，如Office应用程序中的隐私设置和文档的加密。 借助Microsoft 365，你可以执行 GDPR 合规性监视，以确保你设置了正确的保护级别。 
    
- **报告：** GDPR 非常注重报告。 即使具有一名员工的企业处理大量数据，也要求记录并报告其过程。 Microsoft 365企业需要满足小型组织的报告要求。 
    
    使用审核日志等工具，可以跟踪并报告数据移动。 报告包括对收集和存储的数据进行分类、对数据执行哪些操作以及数据传输。 
    
客户、员工和客户越来越了解数据隐私的重要性，现在希望公司或组织遵守该隐私。 Microsoft 365企业组织提供了工具，可让你实现和维护 GDPR 合规性，而不会对业务造成巨大挑战。
