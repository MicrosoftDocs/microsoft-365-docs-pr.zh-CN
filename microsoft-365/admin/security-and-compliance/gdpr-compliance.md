# <a name="gdpr-simplified-a-guide-for-your-small-business"></a>GDPR 简化：适用于小型企业的指南

 *使用 Microsoft 365 商业版缓解和管理 GDPR 合规性* 
  
一般数据保护法规（GDPR）是一种欧盟（EU）法规，规定组织应如何处理个人数据。 GDPR 是由于2018年5月25日强制实施的。 如果你的企业销售、向或雇用欧盟的公民提供服务，则[GDPR](https://ec.europa.eu/info/law/law-topic/data-protection/reform/what-does-general-data-protection-regulation-gdpr-govern_en)将对你造成影响。 
  
本文可帮助您了解 GDPR 是什么、什么原因以及 Microsoft 365 业务如何帮助您的组织遵守 GDPR。
  
## <a name="a-quick-overview-of-the-gdpr"></a>GDPR 的快速概述

GDPR 是一种欧盟法规，它更新并扩展之前的数据保护指令（DPD）在1995中的首次代理。 GDPR 与个人数据的隐私相关，具体是客户、客户、员工或业务合作伙伴。 GDPR 的目标是加强对欧盟公民的个人数据保护，无论它们驻留在欧盟还是其他地方。 该法规将设置预期并建议如何实现。 如果 GDPR 在后期强制实施可能是2018，则组织必须具有满足 GDPR 要求的措施。
  
GDPR 是关于数据及其使用方式的全部。 将数据视为具有生命周期。 在您收集数据时，将会启动周期，并在存储数据时继续运行并使用它（处理），并在您将其从系统中完全删除时结束。 
  
GDPR 与以下类型的数据相关： 
  
- **个人数据：** 如果您可以将数据链接到个人并识别它们，则会将该数据视为个人的 GDPR。 个人数据的示例包括姓名、地址、出生日期和 IP 地址。 GDPR 将编码信息（也称为 "匿名" 信息）视为个人数据，无论数据的隐匿程度或技术如何，如果数据可以链接到个人，则不考虑这些数据。
    
- **敏感的个人数据**这是向个人数据添加更多详细信息的数据。 示例包括宗教、贸易联合成员资格、种族来源等。 敏感的个人数据还包括生物特征数据和 DNA。 在 GDPR 下，敏感数据具有比个人数据更严格的保护规则。 
    
## <a name="gdpr-terms"></a>GDPR 术语

您将在 GDPR 中看到一些经常引用的术语。 请务必了解这些术语。
  
 **同意**
  
GDPR 状态： "应将个人数据处理设计为提供 mankind 服务。" GDPR 希望在处理个人数据时使用同意实现此目标。 这可能是询问客户是否要接收来自贵公司的电子邮件的简单操作。 此外，如果您想要将数据用于市场营销，还意味着您的网站上不再有可选的复选框。 您必须使用 "clear affirmative act" 进行明确同意。 此外，您还需要在获得或撤销同意时保留记录。
  
 **数据使用者权力**
  
GDPR 建立数据主体权限，这意味着，对于其个人数据、客户、员工、业务合作伙伴、客户、合同工、学生、供应商等，您有权执行以下操作：
  
- **了解有关其数据的信息：** 您必须通知个人您对其数据的使用。 
    
- **具有对其数据的访问权限：** 您必须为个人提供对其所保留的任何数据的访问权限（例如，通过使用帐户访问或采用某些手动方式）。 
    
- **请求数据纠正：** 个人可以要求您更正不准确的数据。 
    
- **请求删除数据：** 也称为 "从右侧擦除"，这种权限可让个人请求公司所收集的任何个人数据在使用它的所有系统中删除，或将其共享。 
    
- **请求限制处理：** 个人可以要求您禁止显示或限制其数据。 但是，它仅适用于在特定情况下。 
    
- **具有数据可移植性：** 个人可以要求将其数据转移到其他公司。 
    
- **对象：** 个人可以将其数据用于各种用途（包括直接营销）。 
    
- **要求不要受自动决策制定的制约，包括分析：** GDPR 严格规定了如何使用数据来分析人员并根据该分析自动化决策。 
    
## <a name="common-small-business-questions-about-the-gdpr"></a>有关 GDPR 的常见小型业务问题

### <a name="im-a-sole-proprietor---do-i-really-have-to-worry-about-the-gdpr"></a>我是一个唯一的 proprietor-我确实要考虑 GDPR 吗？

GDPR 是关于您处理的数据，而不是您拥有的员工数。 它影响各种规模的公司，甚至是唯一的 proprietors。 但是，员工数少于250的公司确实有一些例外，例如减少记录保留量，但仅当您确定数据处理不会影响个人的权限且偶尔处理时。
  
例如，非个人数据的处理将免除或需要缩减量值。 但是，如果您处理的任何数据被视为 "特殊类别敏感数据"，即使只是偶尔，也需要记录此数据处理。 "偶尔处理" 的定义是不明确的，但它旨在适用于一次或很少使用的数据。
  
此外，还应确保所收集的个人数据受到保护。 这意味着您需要对其进行加密，并确保使用至少一个密码控制对它的访问。 无保护将客户数据保存在桌面上的电子表格中不会满足 GDPR 的预期。 
  
### <a name="how-can-i-tell-if-our-company-website-is-gdpr-compliant"></a>如何判断我们的公司网站是否符合 GDPR？

要问自己的第一个问题是：是否在网站上的任何位置收集个人数据？ 例如，您可能有一个要求输入姓名和电子邮件地址的联系人窗体。 如果要发送市场营销电子邮件，请确保添加 "自愿加入" 复选框，以确切说明您将使用的数据。 只有当收件人选中此框时，才能出于营销目的使用个人数据。
  
此外，请检查存储数据的数据库是否受保护。 您的 web 托管公司或云存储供应商将能够建议这样做。 如果使用 Microsoft 365 商业版，则数据存储 GDPR 兼容。 
  
### <a name="my-company-is-outside-europe-does-the-gdpr-really-affect-us"></a>我的公司在欧洲之外。 GDPR 确实会影响我们吗？

GDPR 是保护欧盟公民的法规。 如果你的公司现在与欧盟公民交易，或者你希望将来也会受到影响。 这适用于居住在 EU 州和居住在其他地方的公民。
  
请考虑以下示例：
  
- 向欧盟公民雇用汽车的美国公司需要在收集和处理客户数据时满足 GDPR 要求。 当公司获取客户数据并确保数据安全存储时，公司将需要获得同意。 他们还需要确保客户可以应用其所有数据主体权限。 
    
- 澳大利亚公司在线销售产品，并且其用户设置了在线帐户。 GDPR 数据主体权限和许可将应用于打开帐户的欧盟公民。 公司将需要确保客户可以应用其所有数据主体权限。 
    
- 国际慈善机构收集有关捐赠人的数据，并使用它来发送捐赠的更新和请求。 GDPR 状态： ".。。直接营销目的的个人数据处理可能会被视为对合法利益进行处理。 但是，组织中的责任证明他们的利益覆盖了数据主体的利益。 公司（或在本示例中，慈善组织）应始终获得通知、明确、自愿同意。
    
如果客户数据跨边框移动，GDPR 也适用。 如果对数据存储使用云计算，则需要确保服务完全符合 GDPR。 如果数据存储在数据保护记录较差的位置，则可能会变得复杂。 如果使用 Microsoft 365 商业版，我们将提供正确的法律文档来满足 GDPR 要求。
  
### <a name="sure-i-collect-data-but-some-other-company-stores-it-does-that-get-me-off-the-hook"></a>当然，我收集数据，但某些其他公司存储了数据。 是否可以让我关闭挂接？

在 GDPR 下，如果收集的数据受到一定程度的影响。 GDPR 具有数据处理器和数据控制器的概念： 
  
- **数据控制器：** 个人或组织（可以拥有联合控制器），用于确定收集数据的方式、内容和原因。 他们可以使用其他公司的云服务器存储它。 例如，收集客户数据的网站是一个控制器。 
    
- **数据处理器：** 代表控制器存储数据并在请求时处理这些数据的个人或组织。 例如，Office 365 业务数据存储充当处理器，并且完全符合 GDPR 标准。 
    
    组织或系统既可充当控制器，也可充当处理器。 Microsoft 365 业务可以同时充当和遵守 GDPR。
    
### <a name="can-i-still-send-out-marketing-emails-to-my-old-customers"></a>是否仍可以向旧客户发送市场营销电子邮件？

你需要确保你的客户（甚至你有多年的用户）同意将其数据用于营销。 您可能已捕获许可以及显示它的记录。 如果是这样，你将全部设置为继续营销。 如果不是，则需要从客户处获取权限以继续向其营销。 这通常需要发送一封电子邮件，要求客户转到您的网站，并选择同意接收未来电子邮件的选项。 
  
### <a name="do-i-have-to-worry-about-the-gdpr-when-i-recruit-new-employees-what-about-current-employees"></a>在招聘新员工时，我是否需要担心 GDPR？ 当前员工怎么办？

GDPR 不只影响客户数据，而是它也会扩展到员工数据。 新的 recruits 通常位于使用社交媒体平台（如 LinkedIn）的位置。 确保不存储任何可能的招聘数据，而无需其快速权限。
  
与现有员工和新员工合同一样，合同结束时的签名不一定表示同意，尤其是在合同中使用非 affirmative 子句时。 在这种情况下，您必须以与子句相关联的显式方式捕获同意。 这意味着这取决于您的员工合同，但在某些情况下，您可以使用 "合法利息"，并添加员工数据处理通知，以确保员工了解您将对其数据执行的操作。
  
## <a name="satisfy-privacy-concerns-using-microsoft-365-business"></a>使用 Microsoft 365 商业版满足隐私问题

与 GDPR 兼容，是为了确保个人数据受到保护。 GDPR 有一个概念，它在设计和默认情况下称为隐私。 这意味着数据保护应为系统和产品 "baked"，以便在另一种情况中满足隐私问题。 
  
就像更大的客户，在不牺牲安全性的情况下，小型企业需要方便。 Microsoft 365 商业版专为少于300员工的公司而设计。 小型公司可以使用基于 Microsoft 云的工具来提高业务效率。 使用 Microsoft 365 商业版，小型企业可以管理电子邮件、文档甚至会议和事件。 它还具有内置的安全措施和设备管理，这对 GDPR 合规性至关重要。
  
Microsoft 365 商业版可通过以下方式帮助您使用 GDPR 过程：
  
- **发现：** GDPR 合规性的一个重要步骤是了解您拥有的数据。 
    
- **管理：** 控制对数据的访问和管理其使用是 GDPR 不可或缺的一部分。 Microsoft 365 商业版根据您想要应用于设备的策略来保护业务数据。 在员工远程工作的年龄方面，设备管理非常关键。 Microsoft 365 业务包括设备管理功能，可确保跨所有设备保护数据。 例如，可以指定通过 Windows Defender 保护企业中的所有 Windows 10 设备。 
    
- **保护：** Microsoft 365 商业版旨在实现安全性。 其设备管理和数据保护控制可在您的企业网络中工作，包括远程设备，以确保数据安全。 Microsoft 365 商业版可提供 Office 应用程序中的隐私设置和文档加密等控件。 使用 Microsoft 365 商业版，您可以执行 GDPR 合规性监视，以确保您具有适当级别的保护集。 
    
- **报告：** GDPR 将重点放在报告上。 即使是一名员工的业务，如果该业务处理大量数据，也需要对其过程进行记录和报告。 Microsoft 365 业务对较小组织的报告要求不太有麻烦。 
    
    审核日志等工具允许您跟踪和报告数据移动。 报告包括对您收集和存储的数据、对数据的处理方式以及数据传输进行分类。 
    
客户、员工和客户端正在变得更清楚数据隐私的重要性，并且现在希望公司或组织遵守该隐私。 Microsoft 365 Business 为你提供了实现和维护 GDPR 合规性的工具，而无需向你的企业提供巨大的 upheaval。
