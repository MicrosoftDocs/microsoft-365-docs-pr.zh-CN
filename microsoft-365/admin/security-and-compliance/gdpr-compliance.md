<span data-ttu-id="c85e5-115">GDPR 是一项欧盟法规，它更新并扩展了 1995 年首次实施的先前数据保护指令 （DPD）。</span><span class="sxs-lookup"><span data-stu-id="c85e5-115">The GDPR is an EU regulation that updates and expands the earlier Data Protection Directive (DPD) first enacted in 1995.</span></span> <span data-ttu-id="c85e5-116">GDPR 关注个人数据的隐私，即客户、客户、员工或业务合作伙伴。</span><span class="sxs-lookup"><span data-stu-id="c85e5-116">The GDPR is concerned with the privacy of an individual's data, be that individual a client, customer, employee, or business partner.</span></span> <span data-ttu-id="c85e5-117">GDPR 的目标是为欧盟公民（无论他们驻留在欧盟还是其他地方）加强个人数据保护。</span><span class="sxs-lookup"><span data-stu-id="c85e5-117">The GDPR's goal is to strengthen personal data protection for EU citizens, whether they reside in the EU or elsewhere.</span></span> <span data-ttu-id="c85e5-118">该法规规定了期望，并就如何实现这些期望提供建议。</span><span class="sxs-lookup"><span data-stu-id="c85e5-118">The regulation sets out expectations and advises on how to achieve them.</span></span> <span data-ttu-id="c85e5-119">组织必须制定满足 GDPR 要求的措施。</span><span class="sxs-lookup"><span data-stu-id="c85e5-119">Organizations must have measures in place that satisfy the requirements of the GDPR.</span></span>
  
<span data-ttu-id="c85e5-120">GDPR 与数据及其使用方式有关。</span><span class="sxs-lookup"><span data-stu-id="c85e5-120">The GDPR is all about data and how it's used.</span></span> <span data-ttu-id="c85e5-121">将数据视为具有生命周期。</span><span class="sxs-lookup"><span data-stu-id="c85e5-121">Think of data as having a life cycle.</span></span> <span data-ttu-id="c85e5-122">收集数据时，周期开始，在存储和使用数据时继续，并在从系统中完全删除数据时结束。</span><span class="sxs-lookup"><span data-stu-id="c85e5-122">The cycle starts when you collect data, continues as you store it and use it (processing), and ends when you completely delete it from your systems.</span></span> 
  
<span data-ttu-id="c85e5-123">GDPR 涉及以下类型的数据：</span><span class="sxs-lookup"><span data-stu-id="c85e5-123">The GDPR is concerned with the following types of data:</span></span> 
  
- <span data-ttu-id="c85e5-124">**个人数据：** 如果可以将数据链接到个人并识别这些数据，则就 GDPR 而言，该数据被视为个人数据。</span><span class="sxs-lookup"><span data-stu-id="c85e5-124">**Personal data:** If you can link data to an individual and identify them, then that data is considered personal with respect to the GDPR.</span></span> <span data-ttu-id="c85e5-125">个人数据示例包括姓名、地址、出生日期和 IP 地址。</span><span class="sxs-lookup"><span data-stu-id="c85e5-125">Examples of personal data include name, address, date of birth, and IP address.</span></span> <span data-ttu-id="c85e5-126">GDPR 甚至将编码的信息（也称为"假名"信息）视为个人数据，而不考虑数据的模糊程度或技术性，前提是数据可以链接到个人。</span><span class="sxs-lookup"><span data-stu-id="c85e5-126">The GDPR considers even encoded information (also known as "pseudonymous" information) to be personal data, regardless of how obscure or technical the data is, if the data can be linked to an individual.</span></span>
    
- <span data-ttu-id="c85e5-127">**敏感个人数据** 这是向个人数据添加更多详细信息的数据。</span><span class="sxs-lookup"><span data-stu-id="c85e5-127">**Sensitive personal data** This is data that adds more details to personal data.</span></span> <span data-ttu-id="c85e5-128">示例包括宗教、联合成员身份、种族原点等。</span><span class="sxs-lookup"><span data-stu-id="c85e5-128">Examples include religion, trade union membership, ethnic origin, and so on.</span></span> <span data-ttu-id="c85e5-129">敏感个人数据还包括生物识别数据和 DNA。</span><span class="sxs-lookup"><span data-stu-id="c85e5-129">Sensitive personal data also includes biometric data and DNA.</span></span> <span data-ttu-id="c85e5-130">在 GDPR 下，敏感数据的保护规则比个人数据更为严格。</span><span class="sxs-lookup"><span data-stu-id="c85e5-130">Under GDPR, sensitive data has more stringent protection rules than personal data.</span></span> 
    
## <a name="gdpr-terms"></a><span data-ttu-id="c85e5-131">GDPR 条款</span><span class="sxs-lookup"><span data-stu-id="c85e5-131">GDPR terms</span></span>

<span data-ttu-id="c85e5-142">GDPR 建立数据主体权限，这意味着，对于其个人数据，客户、员工、业务合作伙伴、客户、承包商、学生、供应商等有权：</span><span class="sxs-lookup"><span data-stu-id="c85e5-142">The GDPR establishes data subject rights, which means that, with respect to their personal data, customers, employees, business partners, clients, contractors, students, suppliers, and so forth have the right to:</span></span>
