# <a name="microsoft-bookings-frequently-asked-questions"></a>Microsoft Bookings 常见问题解答

## <a name="general"></a>常规

### <a name="what-is-microsoft-bookings"></a>什么是 Microsoft Bookings？

Microsoft Bookings 是一款 Microsoft 365 应用，可轻松安排和管理约会。 Bookings 包括基于 Web 的预订日历，并且与 Outlook 集成以优化员工的日历，让客户可以灵活地预订最适合他们的时间。 自动通知电子邮件可以减少无显示，并且组织通过减少重复的计划任务来节省时间。 Bookings 可帮助你通过 Skype 或 Microsoft Teams 轻松进行虚拟约会，并帮助你通过 Teams 中的 Bookings 应用管理日常日程安排。 通过自定义的内置功能，Bookings 旨在满足任何组织的多个部分的需求。

### <a name="how-do-i-use-microsoft-bookings"></a>如何使用 Microsoft Bookings？

Bookings 是一项联机服务，因此无需下载任何内容。 只需转到 Microsoft 365 Web 体验中的应用选择器。 管理员还可使用 Bookings 配套应用，随时了解客户及其约会的最新信息。

### <a name="who-has-access-to-microsoft-bookings"></a>谁有权访问 Microsoft Bookings？

默认情况下，对于全球的 Microsoft 365 商业高级版、Microsoft 365 商业标准版、A3、A5、E3 和 E5 客户，预订可用且处于活动状态。 Bookings 也可在由世纪通运营的 Office 365 中提供。

### <a name="will-my-customers-see-my-personal-or-business-calendar"></a>我的客户会看到我的个人日历还是商务日历？

对于您选择注册的服务、时间以及员工，您的客户将只看到联机发布的 Bookings 日历。

### <a name="how-do-i-change-a-bookable-attribute-under-manage-staff"></a>如何更改"管理人员"下的可预订属性？

添加员工后，他们将收到一封电子邮件以接受或拒绝该请求。 他们可以打开相同的电子邮件并选择"拒绝"，使其不可预订。 我们目前无法从应用内部切换此功能。

### <a name="how-do-end-users-access-the-bookings-app"></a>最终用户如何访问 Bookings 应用？

任何获得 Microsoft 365 商业高级版、Microsoft 365 商业标准版、A3、A5、E3、E5 许可或已添加为管理员或查看者角色的员工的人员都可以在 Microsoft 365 Web 体验的应用选择器中访问 Bookings。 还有一个适用于 iOS 和 Android 的配套应用。

### <a name="can-i-use-bookings-in-my-own-web-site"></a>能否在我自己的网站中使用 Bookings？

是的。 我们提供了一种通过 iFrame 在网站中嵌入 Bookings 日历的方法。 链接嵌入代码位于 Bookings 应用内的 **Bookings** 页面选项卡中。

### <a name="can-i-use-bookings-even-if-i-dont-have-a-web-page-for-my-business"></a>即使我的企业没有网页，我能否使用 Bookings？

是的。 我们在 Web 应用中的"预订页面"选项卡 **中提供一** 个指向你的预订页面的链接。 你只需将链接提供给客户或客户，他们将看到你的企业的最新可用性信息。 或者，可以在社交媒体中共享预订页面，甚至可以使用嵌入功能将其托管在 iFrame 中。 您还可以通过禁用直接搜索引擎索引并仅限制对组织内部用户的访问权限来控制哪些人可以访问页面。

### <a name="can-our-it-department-control-whether-end-users-can-access-bookings"></a>我们的 IT 部门是否可以控制最终用户是否可以访问 Bookings？

默认情况下，预订在 Microsoft 365 商业高级版、Microsoft 365 商业标准版、M365 A3/A5 和 E3/E5 租户中可用，但管理员可以在 Microsoft 365 管理中心中将其关闭（如果他们选择）。 为此， [请按照以下说明操作](turn-bookings-on-or-off.md)。

如果使用 OWA 邮箱策略禁用对 Bookings 的访问，则所有用户都将拥有 Bookings 许可证，但是，当他们尝试访问应用时，只有在他们是策略的一部分或已作为员工添加到现有 Bookings 日历时，他们才能成功。 此处 [的](turn-bookings-on-or-off.md) "仅允许所选用户创建 Bookings 日历"部分中的详细信息。

## <a name="pricing-and-licensing"></a>定价和许可

### <a name="how-do-i-get-microsoft-bookings"></a>如何获取 Microsoft Bookings？

预订适用于具有 Microsoft 365 商业高级版、Microsoft 365 商业标准版、A3、A5、E3 和 E5 许可证以及 iOS 和 Android 配套应用的客户。 Bookings 不作为独立应用提供。 Outlook Web App或 Web 上的 Outlook，以使用 Bookings，因为它在 Outlook 中存储数据。

Bookings 许可证为产品提供完整功能，包括创建和管理日历。 当用户被添加为管理员或查看者角色中的员工时，它还使用户能够查看和编辑现有日历。

### <a name="do-i-need-to-migrate-my-email-account-from-my-current-provider-to-microsoft-365"></a>是否需要将我的电子邮件帐户从当前提供商迁移到 Microsoft 365？

你可以保留当前的提供商，但 Bookings 会向 Microsoft 365 帐户中使用的电子邮件发送所有通知。

### <a name="can-my-employees-use-bookings-without-a-microsoft-365-account"></a>我的员工能否在没有 Microsoft 365 帐户的情况下使用 Bookings？

是的。 可以使用任何电子邮件添加员工，当有人与员工一起预订约会时，他们仍将收到电子邮件确认和日历邀请。

### <a name="can-i-create-more-than-one-bookings-calendar-under-the-same-microsoft-365-account-and-switch-between-them"></a>能否在同一 Microsoft 365 帐户下创建多个 Bookings 日历，并切换它们？

是的。 可以使用一个帐户创建和管理多个 Bookings 日历。 可以在 Bookings Web 应用中使用业务名称旁边的连接号在两者之间进行切换。

### <a name="what-if-my-organization-has-mixed-licenses-with-f1f3-e1-and-e3e5"></a>如果我的组织具有 F1/F3、E1 和 E3/E5 混合许可证，如何？

我们知道许多组织混合使用 Microsoft 365 许可。 例如，客户可能拥有总部员工 M365 E3 许可证，但 M365 E1 (F1 或 F3) 分店员工的许可证。

本示例中，拥有 M365 E3 许可证的总部员工具有对 Bookings 的完全访问权限，这意味着他们可以创建新日历、编辑设置、添加员工、发布预订页面、在日历内创建和管理约会以及拉取报告。

那些拥有 E1/F1/F3 许可证或没有许可证的应用商店员工仍可以以来宾角色添加到日历，然后预订约会，并且他们将在预订时收到确认电子邮件。 他们仍可在可用/计划时间进行预订，如"员工"选项卡中所列。Bookings 日历可用性按预定小时数和服务时段限制可预订时间。

如果此时已经通过 Bookings Web 应用安排了约会，则应用商店员工在 Bookings 中也将显示为不可用。 通过 Bookings 预订的约会将在 Bookings 内员工日历上反映为忙碌。 具有未启用 Bookings 许可证的员工仍会对他们在 Bookings 中的可用性产生个人日历影响，只要他们位于同一租户内。

"来宾"角色中的人员可以查看客户在初始约会创建过程中提供的任何信息。 例如，如果需要预订约会的应用商店代表在约会前致电客户，他们可以访问客户在计划过程中提供的信息。 已预订的员工将有权访问确认电子邮件中显示的所有信息，以及 .ics 日历事件 (例如客户电话号码（如果已输入) ）。

"来宾"角色中的人员将无法访问 Bookings Web 应用来更改设置或查看和管理约会， (、取消和重新安排) 。 但是，他们可以代表客户使用自助服务页面进行约会，方式与客户进行约会的方式相同。

我们建议使用符合 Bookings 资格的许可证向每个应用商店的经理或管理员授予许可，以设置和管理页面和约会。 然后，其余员工会与获得 Bookings 许可的员工合作，重新安排或取消预订。

## <a name="product-features"></a>产品功能

### <a name="where-do-bookings-calendars-show-up-in-my-microsoft-365-tenant"></a>Bookings 日历在哪里显示在我的 Microsoft 365 租户中？**

每个新的 Bookings 日历在 Exchange 中创建一个相应的邮箱，以及 Azure Active Directory (AAD) 中的相关条目，其中条目列为未授权用户。

### <a name="can-i-delete-a-previously-created-bookings-calendar"></a>能否删除以前创建的 Bookings 日历？

若要删除 Bookings 日历，您必须删除 Exchange 中的关联邮箱。

唯一有权访问通过 Web 应用 () 创建的 Bookings 日历的人员是已添加为员工（以管理员或查看者角色）的人员。 租户管理员将能够查看 Exchange 和 AAD 中所有 Bookings 邮箱的列表。

### <a name="are-video-conferencing-meetings-integrated-into-the-bookings-app"></a>视频会议会议是否集成到 Bookings 应用中？

使用 Skype 或 Microsoft Teams 客户端时，Bookings 中提供联机会议。 可以在服务级别启用联机会议，你选择 skype (或 Teams) 的会议客户端将为已预订员工默认设置的会议客户端。 若要加入联机会议，每个新约会都将有一个附加到约会的唯一会议链接，此外，事件和确认电子邮件中还包括简单的加入选项。

可以，您可以为每个服务创建单独的 Bookings 日历，也可以转到"服务"选项卡，编辑服务，在页面顶部，你将看到仅共享该特定服务的 URL 的选项。

我们建议启用"预订"页面中的"客户数据使用许可"复选框，并自定义文字以正确传达组织的披露或同意要求。 另一个选项是向服务添加一个自定义字段，其中包括指向同意表单的链接，并要求客户确认他们已完成表单，然后才能继续约会。 您还可以添加同意表单 URL 作为确认和提醒电子邮件中的附加说明，但这不会阻止用户预订约会。

客户能够在完全删除预订时选择特定员工，具体方法为从"预订"页面或"服务"部分取消选择"允许客户为预订选择特定人员"选项。 预订将基于员工的可用性，自动为预订随机分配可用员工。

### <a name="can-bookings-support-many-concurrent-booking-requests-and-self-service-page-visits"></a>Bookings 可以支持许多并发预定请求和自助服务页面访问吗？

预订可以一次支持大量访问者和预订。 如果页面遇到大量的流量，用户将收到"服务器繁忙"错误。 在页面加载时以及进行预订时更新约会可用性。 如果多个用户尝试同时预订同一个约会，则 Bookings 将只允许一个人预订该时间，并通知其他潜在客户，并提示他们查找其他时间。

## <a name="privacy"></a>隐私

### <a name="where-is-bookings-data-stored"></a>Bookings 数据存储在何处？

Bookings 是一款 Microsoft 365 应用，这意味着所有数据存储在 Microsoft 365 平台和 Exchange 中。 Bookings 遵循 Microsoft 设置的所有数据存储策略，这些策略与所有 Office 应用遵循的策略相同。 Bookings 使用 Exchange 中的共享邮箱存储客户、员工、服务和约会详细信息。 Exchange 中共享邮箱的合规性策略也适用于 Bookings 邮箱。 所有客户 (包括客户在预订) 在 Bookings 中捕获并存储在应用内时提供的信息，因此存储在 Exchange 中。 Microsoft Bookings 使用 Microsoft 365 使用的相同策略，可以在此处 [找到这些策略。](https://www.microsoft.com/online/legal/v2/?docid=22&langid=en-us)

### <a name="is-there-a-way-to-centrally-manage-bookings-calendars-for-all-users"></a>是否有一种方法可以集中管理所有用户的 Bookings 日历？

每个 Bookings 日历都是独立维护的。 除了单击主页上的"搜索"下拉列表后，没有合并视图。

### <a name="how-are-users-authenticated"></a>如何对用户进行身份验证？

访问 Bookings Web 应用涉及通过常规 Azure Active Directory 身份验证 (AAD) 身份验证。 通过网页链接，每个人都可以使用自助服务预订页面。 但是，当选择"要求我的组织使用 **Microsoft 365 或 Office 365** 帐户来预订"设置时，将限制页面仅在使用 AAD 身份验证 (Microsoft 365 租户) 。

:::image type="content" source="media/bookings-faq-access-ctrl.png" alt-text="Bookings 中的预订页面访问控制":::

### <a name="does-customer-data-leave-the-production-system-for-any-reason"></a>客户数据是否出于任何原因离开生产系统？

是的，由于 Bookings 是 Exchange 的一部分，我们允许 Graph API 和 Exchange Web 服务 (EWS) API 供客户下载他们有权访问的数据。

### <a name="is-there-the-ability-to-customize-email-addresses-for-customer-replies"></a>能否为客户回复自定义电子邮件地址？

可以定义客户向哪个电子邮件地址发送答复。 此设置位于"业务信息 **"选项卡** 上。通过常用的 Exchange 邮箱重命名过程，还可以在 Exchange 中更改 Bookings 邮箱名称本身。

### <a name="we-would-like-for-the-customer-to-accept-our-terms-and-conditions-during-the-booking-process-is-that-possible"></a>我们希望客户在预订过程中接受我们的条款和条件。 这是否可行？

此功能已存在，用于符合 GDPR 要求。 "预订"页面的"发布详细信息"部分下有一个自定义许可字段。 只需选中该框并编写你自己的自定义同意消息，它将在自助服务页面上作为必填字段显示。 将记录客户授予同意的日期和时间。

此部分不支持格式文本，但认为可以在业务信息部分下添加链接，并会显示在同意复选框正下方。

### <a name="how-can-bookings-associated-mailboxes-in-exchange-be-identified"></a>如何标识 Exchange 中与 Bookings 关联的邮箱？

以下命令可用于检索所有 Bookings 邮箱以及有权访问这些邮箱的用户的信息。

`Get-Mailbox -RecipientTypeDetails SchedulingMailbox -ResultSize:Unlimited | Get-MailboxPermission |Select-Object Identity,User,AccessRights | Where-Object {($_.user -like '*@*')}`

## <a name="technical-questions"></a>技术问题

### <a name="if-a-user-schedules-an-appointment-for-a-service-consisting-of-an-online-meeting-how-do-they-receive-the-microsoft-teams-link-to-the-meeting-if-they-leave-the-optional-email-address-field-empty"></a>如果用户为包含联机会议的服务安排约会，如果他们将可选电子邮件地址字段留空，如何接收会议 Microsoft Teams 链接？

如果系统没有客户的电子邮件信息，则确认邮件无法发送给客户，因此他们不会收到 Teams 链接。 解决方案是，将电子邮件地址设置为必填字段，您可以在 Bookings 的"服务"选项卡上这样做。 请注意，每当在 Bookings 中创建新服务时，客户电子邮件字段默认设置为必填字段。

### <a name="how-are-staff-assigned-to-appointments-when-anyone-is-selected-or-when-customers-do-not-have-the-ability-to-choose-a-staff-member-for-the-appointment"></a>当选择"任何人"或客户无法为约会选择员工时，如何为约会分配员工？

在这些方案中，在预订约会时可用的所有员工中，会随机分配员工。 目前，无法确保随机分配的约会在员工的所有成员中均等分配，或限制分配给每个员工的约会数。 如果在进行大量随机分配后需要跨员工完成负载平衡，可以在 Bookings 页面的"日历"选项卡中手动完成。

### <a name="how-can-users-opt-out-of-receiving-sms-messages-if-they-have-already-opted-in-previously"></a>如果之前已选择接收短信，用户如何选择不接收短信？

当预订启用了短信的约会时，将提示客户每次预订时选中"选择加入"框。 客户还可以直接在短信提供商 (短信) 指示他们希望消息停止，然后不会向该电话号码发送其他文本。 如果客户选择不接收来自提供商（而不是 Bookings 本身）的文本消息，他们以后将不再收到短信，即使从 Bookings 页面选择接收短信。

## <a name="troubleshooting"></a>疑难解答

### <a name="this-staff-members-calendar-cant-be-synced"></a>此员工的日历无法同步

如果您收到错误"此员工的日历无法同步"，本部分包含帮助您解决错误的多个疑难解答过程。

> [!NOTE]
> 对于订阅了 Microsoft 365 商业标准版、Microsoft 365 A3 或 Microsoft 365 A5 的客户，默认情况下会启用预订。 对于拥有 Office 365 企业版 E3 和 Office 365 企业版 E5 但默认情况下已关闭的客户，预订也可用。 若要开始，请参阅 [获取对 Microsoft Bookings 的访问权限](get-access.md)。 若要打开或关闭 Bookings，请参阅[为组织打开或关闭 Bookings。](turn-bookings-on-or-off.md)

如果要查找有关同步员工日历的信息，请参阅向 [Bookings 添加员工](add-staff.md)。 确保选中" **员工详细信息** "页上的 **"日历上的事件影响可用性** "。

### <a name="the-staff-member-needs-to-change-their-calendar-permissions"></a>员工需要更改其日历权限

为了使 Bookings 与员工日历正确同步，每个员工必须共享其日历。 每个员工都必须按照这些说明共享自己的日历。

1. 登录到 Microsoft 365，然后从应用启动器 **中选择 Outlook。**

   > [!div class="mx-imgBorder"]
   > ![应用启动器中的 Outlook 图像](../media/bookings-applauncher-outlook.png)

1. 在页面顶部，选择"**共享日历** \> **"。**

   ![日历共享菜单的图像](../media/bookings-share-calendar.png)

1. 在"**组织内部人员**"部分，选择"我的组织"旁边的下拉框，然后选择"我忙碌时 **可以查看"。**

   > [!NOTE]
   > 此设置与 Bookings 和您组织的其他用户共享您的忙/闲信息。 它不包括任何其他信息，例如已安排的约会类型、计划约会的人或位置。

   > [!div class="mx-imgBorder"]
   > ![启用"我忙碌时可查看"的日历共享屏幕的图像](../media/bookings-view-busy.png)

1. 选择“**完成**”。

### <a name="we-cant-find-a-staff-member-in-our-organization"></a>我们找不到组织的员工

如果员工已离开公司，并且你已从 Microsoft 365 或 Office 365 组织中删除他们，你仍然需要从 Bookings 中删除该用户，这样他们就不会显示在日历中，并阻止客户预订约会。

1. 登录到 Microsoft 365，然后转到 **预订** \> **员工**。

1. 将鼠标光标悬停在员工姓名的上方。 **编辑****和删除** 图标显示在"电话 **"列的** 右侧。

1. 选择" **删除"** 图标。

1. 在确认框中，选择"确定 **"。**

> [!NOTE]
> 如果你有多个预订日历，则必须从所有预订日历中删除用户。

若要检查您是否具有多个日历，请在左侧菜单中，选择公司名称旁边的下拉箭头，然后选择"打开 **"。** 如果只有一个日历，则只会看到添加预订日历的选项。 本示例显示还有一个名为"Contoso CO"的日历。

![显示第二个日历的"选择日历"屏幕的图像](../media/bookings-choose-calendar.png)

### <a name="theres-a-syncing-error-we-cant-identify"></a>有一个无法识别的同步错误

这是一个暂时性错误，通常由于 Internet 连接不稳定而发生。 这些问题通常在几分钟后自行解决。

如果长时间后仍看到此错误或经常看到此错误，请联系支持人员。
