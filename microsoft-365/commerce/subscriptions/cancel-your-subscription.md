# <a name="cancel-your-subscription"></a>取消订阅

::: moniker range="o365-21vianet"

> [!NOTE]
> 管理中心正在发生改变。 如果你的体验与此处提供的详细信息不匹配，请参阅[有关新版 Microsoft 365 管理中心](https://docs.microsoft.com/microsoft-365/admin/microsoft-365-admin-center-preview?view=o365-21vianet&preserve-view=true)。

::: moniker-end

*资格：* 如果分配给用户的许可证少于 25 个，可以随时在 Microsoft 365 管理中心内取消 Microsoft 365 商业版试用版或付费订阅。 如果你向用户分配了超过 25 个许可证，请将其减少为少于 25 个，或致电支持人员 [取消订阅](../../admin/contact-support-for-business-products.md)。

*退还：* 在下一个计费周期内，将向你退还所有按比例分配的点数。

> [!NOTE]
> 如果你有对同一产品（如 Microsoft 365 商业高级版）的多个订阅，取消其中一个订阅不会影响其他订阅内的已购买许可证或服务。

## <a name="before-you-begin"></a>准备工作

你必须是全局管理员或帐单管理员才能执行本文中的任务。 有关详细信息，请参阅 [关于管理员角色](../../admin/add-users/about-admin-roles.md)。

## <a name="what-happens-when-you-cancel-a-subscription"></a>取消订阅时会发生什么情况

如果在期限结束之前取消订阅，订阅状态将直接进入禁用状态。 在大多数国家和地区，大多数订阅的禁用状态会持续 90 天。 订阅处于禁用状态时，管理员仍然可以访问和备份其组织的数据，但我们建议管理员在取消订阅之前[备份其数据](back-up-data-before-switching-plans.md)，尤其是在该订阅是其唯一订阅的情况下。 管理员还可以在订阅处于禁用状态时重新激活订阅。

90 表示后，订阅将进入已删除状态。 在取消 90 天后，您留下的任何数据都可能会被删除，最晚在取消后的 180 天内删除。 在取消订阅达到已删除状态之前，你无法从该订阅中删除付款方式。

### <a name="what-to-expect-for-you-and-your-users-if-you-cancel-a-subscription"></a>取消订阅对你和你的用户的影响。
  
- **管理员访问权限** 管理员仍可登录和访问管理中心，并根据需要购买其他订阅。 作为全局管理员或帐单管理员，你有 90 天的时间重新激活 [所有](reactivate-your-subscription.md) 数据保持不变的订阅。

- **用户访问** 用户将无法使用 OneDrive for Business 等服务，或无法访问团队网站上的电子邮件或文档等客户数据。 Word 和 Excel 等 Office 应用程序最终将进入只读的缩减功能模式，并显示 [未经授权的产品通知](https://support.microsoft.com/office/0d23d3c0-c19c-4b2f-9845-5344fedc4380.aspx)。

若要了解更多信息，请参阅我的订阅结束时，我的数据和 [访问会发生什么情况？](what-if-my-subscription-expires.md)

> [!IMPORTANT]
> 如果希望在典型的禁用期结束之前删除订阅数据，可以 [关闭帐户](../close-your-account.md)。

## <a name="other-steps-you-might-have-to-take"></a>您可能必须执行的其他步骤

### <a name="change-custom-domain-settings"></a>更改自定义域设置

如果将自定义域与订阅一同使用，则必须执行一些额外步骤，然后才能取消订阅。 如果你没有自定义域，可以跳到保存 [数据](#save-your-data)。

#### <a name="change-your-domain-nameserver-records-if-needed"></a>根据需要更改域名服务器 (记录) 

如果设置自定义域，则添加了 DNS 记录，以便该域可以与 Microsoft 365 服务一起工作。 在删除域之前，请务必在 DNS 主机上更新 DNS 记录，如域 MX 记录。

例如，在 DNS 主机上更改 MX 记录。 发送到你的域的电子邮件将停止发送到你的 Microsoft 地址，而是转到你的新电子邮件提供商。  (MX 记录确定域电子邮件的发送位置。) 

- 如果名称服务器 (NS) 记录指向 Microsoft [365 名称](../../admin/setup/add-domain.md)服务器，则对 MX 记录的更改不会生效，除非更改 NS 记录以指向新的 DNS 主机 (请参阅步骤 2) 。

- 在更新 MX 记录之前，请告知用户计划切换其电子邮件的日期，以及计划使用的新电子邮件提供商。 此外，如果用户希望将其现有 Microsoft 电子邮件移动到新提供商，则必须执行额外步骤。

- 在你更改 MX 记录时，请遵照本文中的其余步骤。

#### <a name="update-your-domain-mx-and-other-dns-records-if-youre-using-a-custom-domain"></a>如果使用自定义域记录， (域 MX 和其他 DNS 记录) 

如果在设置域时将名称服务器 (NS) 记录切换到 Microsoft 365，则必须在计划使用的 DNS 主机上设置或更新 MX 记录和其他 DNS 记录，然后将 NS 记录更改为该 DNS 主机。

如果在设置域时未切换 NS 记录，则当您更改 MX 记录时，您的邮件会开始直接发送到新地址。

若要更改 NS 记录，请参阅["删除域"。](../../admin/get-help-with-domains/remove-a-domain.md)

### <a name="save-your-data"></a>保存数据

取消生效后，用户将失去访问其数据的访问权限。 在取消订阅之前，请让他们将其 OneDrive for Business 或 SharePoint Online 文件保存到其他位置。 你留下的任何客户数据可能在 30 天后删除，并且将在取消后的 180 天内删除。

- 若要将电子邮件、联系人、任务和日历信息移动到另一个帐户，请参阅[将电子邮件、联系人和日历导出或备份到 Outlook .pst 文件](https://support.microsoft.com/office/14252b52-3075-4e9b-be4e-ff9ef1068f91.aspx)。

- 要将 SharePoint Online 环境（OneDrive for Business 或团队网站）中的文档库或列表内容（如联系人）保存到文件共享或本地计算机，请参阅[手动迁移 SharePoint Online 内容](https://support.microsoft.com/kb/2783484)。

### <a name="uninstall-office-optional"></a>卸载 Office (可选) 

如果已取消订阅，并且未将用户移动到包含[](move-users-different-subscription.md)Microsoft 365 的不同订阅，Microsoft 365 将运行在缩减功能模式下。 发生这种情况时，用户只能读取和打印文档，并且 Microsoft 365 应用程序显示未授权 [的产品通知](https://support.microsoft.com/office/0d23d3c0-c19c-4b2f-9845-5344fedc4380.aspx)。 若要避免任何混淆，请要求用户卸载计算机上的 [Office](https://support.microsoft.com/office/9dd49b83-264a-477a-8fcc-2fdf5dbf61d8.aspx)。
