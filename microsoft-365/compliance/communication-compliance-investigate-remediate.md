# <a name="investigate-and-remediate-communication-compliance-alerts"></a>调查和修正通信合规性警报

配置通信合规性策略后，你将开始在 Microsoft 365 合规中心收到符合策略条件的邮件问题的警报。 按照此处的工作流说明来调查和修正警报问题。

## <a name="investigate-alerts"></a>调查警报

调查策略检测到的问题的第一步是查看 Microsoft 365 合规中心中的通信合规性警报。 通信合规性解决方案区域中有几个区域可帮助您快速调查警报，具体取决于您喜欢查看警报分组的方法：

- **通信合规性策略页**：登录使用 Microsoft 365 组织中管理员帐户的凭据时，选择"通信合规性"以显示 [https://compliance.microsoft.com](https://compliance.microsoft.com) 通信合规性 **策略** 页。  此页面显示为 Microsoft 365 组织配置的通信合规性策略和指向推荐策略模板的链接。 列出的每个策略包括需要检查的警报计数、已升级和已解决的项目数、策略的状态以及上次策略扫描的日期和时间。 选择策略将显示与策略匹配的所有挂起警报，选择特定警报以启动策略详细信息页面并启动修正操作。
- **警报**：导航到 **"通信合规性** 警报"以显示过去 30 天按策略匹配  >  分组的警报。 通过此视图，你可以快速查看哪些通信合规性策略正在生成按严重性排序的最多警报。 若要启动修正操作，请选择与警报关联的策略以启动"策略 **详细信息"** 页。 从"策略详细信息"页中，你可以查看"概述"页上活动的摘要，查看"挂起"页上的警报消息并采取行动，或在"已解决"页上查看已关闭警报 **的历史记录。** 
- **报告**：导航到 **"通信合规性**  >  **报告**"以显示通信合规性报告小组件。 每个小组件都提供通信合规性活动和状态的概述，包括访问有关策略匹配和修正操作的更深入见解。

### <a name="using-filters"></a>使用筛选器

下一步是对邮件进行排序，以便更轻松地调查警报。 从" **策略详细信息** "页中，通信合规性支持对多个邮件字段进行多级筛选，以帮助你快速调查和查看具有策略匹配项的邮件。 筛选可用于每个已配置策略的挂起和已解析项目。 您可以为策略配置筛选器查询，也可以配置和保存自定义和默认筛选器查询，以用于每个特定策略。 配置筛选器的字段后，你将看到筛选器字段显示在警报消息队列的顶部，您可以为特定筛选器值配置这些字段。

有关筛选器和字段详细信息的完整列表 [，请参阅功能](communication-compliance-feature-reference.md#filters) 参考文章中的"筛选器"。

#### <a name="to-configure-a-filter"></a>配置筛选器

1. 使用 Microsoft 365 组织中的管理员账户凭据登录 [https://compliance.microsoft.com](https://compliance.microsoft.com)。

2. 在 Microsoft 365 合规中心中，转到“**通信合规性**”。

3. 选择 **"策略** "选项卡，然后选择用于调查的策略，双击以打开"策略 **"** 页。

4. 在"**策略**"页上，选择"挂起"或"已解决 **"选项卡以显示** 要筛选的项目。

5. 选择 **"筛选器** "控件以打开" **筛选器** 详细信息"页。

6. 选中一个或多个复选框以启用这些警报的筛选器。 可以从多个筛选器中选择，包括 *日期*、*发件人**、主题/标题**、分类器* 等。

7. 如果要将所选的筛选器另存为默认筛选器，请选择"**另存为默认值"。** 如果要使用此筛选器作为保存的筛选器，请选择"**完成"。**

8. 如果要将所选筛选器另存为筛选器查询，请选择"在配置至少一个筛选器值后保存查询控件"。 输入筛选器查询的名称，然后选择"保存 **"。** 此筛选器仅可用于此策略，并列在"筛选器详细信息"页的"保存的筛选器查询 **"** 部分。

    ![通信合规性筛选器详细信息控件](../media/communication-compliance-filter-detail-controls.png)

### <a name="using-near-and-exact-duplicate-analysis"></a>使用几乎完全相同的重复分析

通信合规性策略会自动扫描和预分组邮件的接近和确切重复项，而无需任何其他配置步骤。 此视图允许您对类似邮件进行一对一或一组快速操作，从而减少审阅者的邮件调查负担。 检测到重复项时，"近重复项"和/或"精确重复项"控件将显示在修正操作工具栏中。 如果找不到接近或完全相同的重复项，则此视图不可用。

#### <a name="to-remediate-duplicates"></a>修正重复项

1. 使用 Microsoft 365 组织中的管理员账户凭据登录 [https://compliance.microsoft.com](https://compliance.microsoft.com)。

2. 在 Microsoft 365 合规中心中，转到“**通信合规性**”。

3. 选择 **"策略** "选项卡，然后选择用于调查的策略，双击以打开"策略 **"** 页。

4. 在" **策略"** 页上，选择"挂起 **"** 或"已解决 **"** 选项卡以显示重复邮件。

5. 选择 **"近重复项"** 或" **完全重复项"控件** 以打开重复项详细信息页。

6. 选择一个或多个邮件以修正这些邮件的操作控件。

7. 选择 **"解析****"、"通知**"、"升级"或"**下载**"，将操作作为默认筛选器应用于选定的重复邮件。 

8. 对 **邮件** 完成修正操作后，选择"关闭"。

    ![通信合规性完全重复控制](../media/communication-compliance-duplicates-controls.png)

## <a name="remediate-alerts"></a>修正警报

无论你从哪里开始查看警报或配置的筛选，下一步是采取措施来修正警报。 在"策略"或"通知"页上，**通过以下工作流****启动警报** 修正。

### <a name="step-1-examine-the-message-basics"></a>步骤 1：检查邮件基础知识

 有时，从源或主题可以很明显地发现可以立即修正邮件。 可能是邮件有误或与策略匹配不正确，应解析为误报。 选择 **误报控件** 可立即解决警报并从挂起的警报队列中删除。 在源或发件人信息中，您可能已经知道在这种情况下应该如何路由或处理邮件。 请考虑使用 **标记作为** 控件 **或升级** 控件，将标记分配给适用的邮件或将邮件发送给指定的审阅者。

![通信合规性修正控制](../media/communication-compliance-remediation-controls.png)

### <a name="step-2-examine-the-message-details"></a>步骤 2：检查邮件详细信息

查看邮件基础知识后，可以打开一条消息来检查详细信息并确定进一步修正操作。 选择邮件以查看完整的邮件头和正文信息。 可以使用几种不同的视图来帮助你确定正确的操作过程：

- **源视图**：此视图是大多数基于 Web 的邮件平台中通常看到的标准邮件视图。 邮件头信息的格式采用正常样式，并且邮件正文支持带 imbeded 图形文件和换行文本。
- **文本视图**：文本视图显示邮件的仅行号文本视图，并包括邮件和附件中与关联的通信合规性策略中匹配的字词的关键字突出显示。 关键字突出显示可以帮助您快速扫描长邮件和附件中感兴趣的区域。 在某些情况下，突出显示的文本可能仅在匹配策略条件的邮件的附件中。 不显示嵌入文件，并且此视图的行号有助于在多个审阅者之间引用相关详细信息。
- **批注视图**：此视图允许审阅者直接在保存到邮件视图中的邮件上添加批注。
- **用户历史记录**：用户历史记录视图显示由发送邮件的用户的任何通信合规性策略生成的所有其他警报。
- **邮件详细信息视图**：邮件元数据和配置信息的高级视图。
- **模式检测到 (预览**) ：随着时间的推移，许多攻击和威胁操作都涉及用户重复出现相同行为的实例。 模式 *检测到* 的通知显示在警报详细信息中，并引起对警报的注意。 模式检测基于每个策略，并评估过去 30 天内至少由发件人向同一收件人发送两个邮件时的行为。 调查人员和审阅者可以使用此通知识别重复行为，以根据情况评估警报。

    ![通信合规性消息视图控件](../media/communication-compliance-message-views.png)

### <a name="step-3-decide-on-a-remediation-action"></a>步骤 3：确定修正操作

现在，你已查看了警报消息的详细信息，您可以选择多个修正操作：

- **Resolve**： **Selecting** the Resolve control immediately removes the message from the **Pending alerts** queue and no further action can be taken on the message. 通过选择 **"** 解决"，实际上你已关闭警报而不进一步分类，并且无法重新打开它以执行进一步操作。 所有解析的邮件都显示在"已解决 **"** 选项卡中。
- **误报**：在邮件审阅工作流过程中，您始终可以在任何时间点将邮件解析为误报。 误报表示警报不可操作或警报进程生成警报不正确。 无法重新打开邮件，所有误报邮件都显示在"已解决 **"** 选项卡中。
- **Power Automate (预览) ：** 使用 Power Automate 流自动处理警报消息的任务。 默认情况下，当用户具有通信合规性警报流模板时，通信合规性包括通知管理器，审阅者可以使用该模板为具有消息通知的用户自动执行通知过程。 有关在通信合规性中创建和管理 Power Automate 流详细信息，请参阅 [通信合规性功能参考](communication-compliance-feature-reference.md#power-automate-flows) 文章。
- **标记为**：将邮件标记为符合 *、* 不符合或与组织的策略和标准相关时有问题。 添加标记和标记注释可帮助你针对升级或作为其他内部审阅过程的一部分进行微筛选策略警报。 标记完成后，还可以选择解析邮件以将其从挂起的审阅队列中移开。
- **通知**：可以使用"通知" **控件向** 通知分配自定义通知模板，并将警告通知发送给用户。 选择在通信合规性设置区域中配置的适当通知模板，然后选择"通过电子邮件发送给发送邮件的用户的提醒"并解决问题。
- **升级**：使用 **升级** 控件，可以选择组织中其他人员应查看邮件。 从通信合规性策略中配置的审阅者列表中选择，以发送一封电子邮件通知，请求对邮件警报进行其他审阅。 所选审阅者可以使用电子邮件通知中的链接直接转到已升级至这些项目进行审阅的项目。
- **上报以进行调查**：使用 **升级进行** 调查控制，你可以为单个或多个邮件创建新的 [](overview-ediscovery-20.md)高级电子数据展示案例。 你将提供新案例的名称和注释，并自动将发送与策略匹配的邮件的用户分配为案例保管人。 管理案例不需要任何其他权限。 创建案例不会解析或创建邮件的新标记。 在修正过程中创建高级电子数据展示案例时，总共可以选择 100 条消息。 支持通过通信合规性监视的所有通信频道中的邮件。 例如，为用户打开新的高级电子数据展示案例时，可以选择 50 条 Microsoft Teams 聊天、25 封 Exchange Online 电子邮件和 25 封 Yammer 邮件。
- **改进 (预览) ：** 根据分类器类型匹配创建的警报可能需要反馈以帮助最大程度地减少组织中误报。 使用 **"改进分类** "控件提供有关通信合规性分类是否有效的反馈，或建议其他可训练的分类器用于此类型的匹配。 你可以确认分类器是匹配还是不匹配，或者建议其他可训练的分类器在将来与这种类型的警报活动关联。

    1. 从警报列表中选择一条消息。
    2. 选择省略号并选择"**改进分类"。**
    3. 在 **"详细分类器反馈**"窗格中，如果项目为真正的正，请选择"**匹配"。**  如果项目错误地作为误报包含在类别中，请选择"**不匹配"。**
    4. 如果有另一个分类器更适用于该项目，请从"建议 **其他可训练** 分类器"列表中选择它。 此反馈将触发其他分类器来评估项目。

    > [!TIP]
    > 可以通过选择所有项目，然后在命令栏中选择"提供详细反馈"来同时提供 **对多个项目** 的反馈。

    5. 选择 **"发送** 反馈"以发送对 **"** 匹配"和"非 **匹配** "分类的评估，并建议其他可训练的分类器。 当你向分类器提供 30 个反馈实例时，它会自动重新分析。 重新培训可能需要 1-4 小时才能完成。 分类器每天只能重新训练两次。

    > [!IMPORTANT]
    > 此信息将转到租户中的分类器 **，不会返回到 Microsoft。**

    若要了解有关重新分类分类器以确保通信合规性，请参阅通信合规性文章中的"如何重新分类 [分类](classifier-how-to-retrain-comms-compliance.md) 器"。

    ![通信合规性改进分类](../media/communication-compliance-improve-classifier.png)

- **删除 Teams 中的** 消息：使用 Teams 控件 **中的** "删除"消息，可以阻止来自 Microsoft Teams 频道和 1：1 和群聊的警报中标识的不当消息和内容。 已删除的消息和内容将替换为策略提示，该提示说明阻止邮件和应用于从视图中删除邮件的策略。 策略提示中向收件人提供了一个链接，以了解有关适用策略和审阅过程的信息。 发件人会收到阻止的邮件和内容的策略提示，但可以查看阻止邮件的详细信息和有关删除的上下文内容。

    ![从 Microsoft Teams 中删除邮件](../media/communication-compliance-remove-teams-message.png)

### <a name="step-4-determine-if-message-details-should-be-archived-outside-of-communication-compliance"></a>步骤 4：确定是否应在通信合规性之外存档邮件详细信息

如果需要在单独的存储解决方案中存档邮件，可以导出或下载邮件详细信息。 选择下载 **控件** 会自动将所选邮件添加到 。可保存到 Microsoft 365 外部存储的 ZIP 文件。
