# <a name="build-and-manage-assessments-in-compliance-manager"></a>在合规性管理器中构建和管理评估

**本文内容：** 了解如何通过创建和管理 **评估**来自定义您的组织的合规性管理器。 本文将引导您了解如何创建评估、如何将其组织为 **组**、使用 **控件**、接受 **更新**和导出评估 **报告**。

> [!IMPORTANT]
> 你的组织可用的评估取决于你的许可协议。 [查看详细信息](https://go.microsoft.com/fwlink/?linkid=2132371)。

## <a name="introduction-to-assessments"></a>评估简介

合规性管理器可帮助你管理对你的组织适用的规章和证书评估合规性。 评估是来自特定法规、标准或策略的控件组。 合规性管理器通过提供预建评估来轻松跟踪合规性，这涵盖了各种行业和区域的法规和认证。

每个评估都是通过 [评估模板](compliance-manager-templates.md)创建的。 模板充当包含必要控件、改进操作和 Microsoft 评估完成的 Microsoft 操作的框架。 为你的组织设置最相关的评估可帮助你实施策略和操作过程来限制合规性风险。

您的所有评估都列在 "评估" 页上。 了解有关 [如何筛选你的评估视图和解释状态状态的](compliance-manager-setup.md#assessments-page)详细信息。

## <a name="data-protection-baseline-default-assessment"></a>数据保护基线默认评估

为入门，Microsoft 在合规性管理器中提供了**microsoft 365 数据保护基准**的**默认**评估。 此基准评估具有一组用于数据保护和常规数据管理的关键法规和标准的控件。 此基线主要从 NIST CSF (美国国家标准 and 技术协会 Cybersecurity Framework) and ISO (国际标准化组织) 以及 FedRAMP (联邦风险和授权管理计划) 和 GDPR (常规数据保护法规的欧盟) 。

此评估用于在配置任何其他评估之前，首次使用合规性管理器时计算最初的合规性分数。 合规性管理器从 Microsoft 365 解决方案中收集初始信号。 你将快速了解你的组织是如何相对于关键数据保护标准和管理法规执行的，并查看建议采取的改进措施。

在构建和管理自己的评估以满足组织的特定需求时，合规性管理器变得更加有用。

## <a name="assessment-creation-overview"></a>评估创建概述

您可以通过三种方法来设置评估：

1. [使用预建评估](#use-a-pre-built-assessment)。
2. [扩展预建评估以满足您自己的需求](#extend-a-pre-built-assessment)。
3. [创建您自己的自定义评估](#create-your-own-custom-assessment)。

> [!NOTE]
> 只有拥有全局管理员或合规性管理器管理角色的用户才可以创建和修改评估。 了解有关 [角色和权限](compliance-manager-setup.md#set-user-permissions-and-assign-roles)的详细信息。

**使用预建评估**

通过选择已由合规性管理器设置的评估启动您的合规性旅程。 我们为符合行业、区域和常见数据保护标准（如 GDPR 和 ISO 27001）的法规和认证提供了广泛的 [模板](compliance-manager-templates.md) 选择。 模板包含用于帮助您满足特定证书的要求的控件和改进操作。 开始 [建立评估](#use-a-pre-built-assessment)时，系统将要求你选择一个模板。

**扩展预建评估以满足您的需求**

您可以修改合规性管理器评估（我们称为 "扩展" 的过程），方法是添加自己的控件和操作以更好地满足组织的需求。 例如，如果您通常需要遵守 HIPAA，但需要其他数据保护或安全控制，则可以通过向其添加自己的控制措施来扩展其 HIPAA 模板。 有关 [扩展预建评估](#extend-a-pre-built-assessment)的说明，请参阅。

**创建自己的自定义评估**

您可以从头开始创建自己的评估，以准确地跟踪您的组织所需的内容。 创建您自己的评估要求您首先为合规性管理器中的评估创建自己的模板。 请参阅 [创建您自己的自定义评估](#create-your-own-custom-assessment)的说明。

## <a name="understand-groups-before-creating-assessments"></a>在创建评估之前了解组

在创建或修改评估之前，请务必了解组的工作方式。 在创建评估时，需要在此过程中将其分配给一个组。 这就是我们建议您在创建评估之前规划评估的分组策略的原因。

### <a name="what-are-groups"></a>什么是组

组是允许您组织评估的容器。 您可以按符合您的方式对评估进行分组，如按年或按规定进行分组，或者基于组织的部门或地理位置进行分组。 以下是两个组及其基础评估的示例：

- **FFIEC 为评估2020**
  - FFIEC 为
- **数据安全性和隐私评估**
  - ISO 27001:2013
  - ISO 27018:2014

当同一个组中的两个不同评估共享由您管理的改进操作时，您对操作的实现详细信息或状态所做的任何更新将自动同步到组中任何其他评估中的相同操作。 此同步使您能够实现一个改进操作，并在多个管理法规方面满足几个要求。

### <a name="how-to-create-a-group"></a>如何创建组

在 [创建新评估](#to-create-an-assessment)的过程中，您需要创建一个组。

无法将组创建为独立实体。 一个组必须至少包含一个评估。 若要创建组，您需要首先创建一个评估以放置在该组中。

### <a name="what-to-know-when-working-with-groups"></a>使用组时需要了解的内容

- 组名称在您的组织中必须是唯一的。
- 组没有安全属性。 所有权限都与评估相关联。
- 将评估添加到组后，不能更改分组。
- 在同一组中的不同评估中的相关评估控件在完成后将自动更新。
- 如果将新评估添加到现有组中，则会将该组中评估的常见信息复制到新评估中。
- 组可以包含对同一认证或法规的评估，但每个组只能包含针对特定产品认证对的一个评估。 例如，组不能包含针对 Office 365 和 NIST CSF 的两个评估。 仅当一个组与同一个产品的对应证书或法规不同时，该组才可以包含对同一产品的多个评估。
- 删除评估会破坏该评估与组之间的关系。
- 无法删除组。
- 当对多个组中出现的改进进行更改时，该更改将反映在该改进操作的所有实例中。

## <a name="use-a-pre-built-assessment"></a>使用预建评估

从合规性管理器模板创建评估有两个起始点。

您可以通过选择 " **添加评估** " 按钮，然后在 "评估创建向导" 中运行，从评估页开始此过程。 此过程的步骤如下所示。

您还可以从 "评估模板" 页开始，方法是查找所需的模板，并从列表中选择该模板以到达 "详细信息" 页。 在 "模板详细信息" 页上，选择 " **创建评估**"。 然后，将使用已选择的模板输入向导。

### <a name="to-create-an-assessment"></a>创建评估

1. 了解要向其分配评估的组，或者准备好为此评估创建一个新的组。 [了解有关组的详细信息](#understand-groups-before-creating-assessments)。  

2. 转到合规性管理器中的 " **评估** " 页，并选择 " **添加评估**"。 在大型浮出控件窗格中将显示一个评估向导。

3. **选择模板**：选择要用作评估基础的模板。 选择您选择的模板旁边的单选按钮，然后选择 " **下一步**"。

4. **名称和组：** 在 " **评估名称** " 字段中输入评估的名称。 评估名称在组中必须是唯一的。 如果评估的名称与任何给定组中的另一个评估的名称相匹配，则会收到一条错误，要求您创建一个不同的名称。

5. 将评估服务分配给一个组。 您可以：
    - 选择 " **使用现有组** 将其分配给已创建的组";和
    - 选择 " **创建新组** " 以创建新组，并将此评估分配给它：
        - 确定组的名称，并将其输入到单选按钮下的字段中。
        - 您可以通过选择相应的框， **从现有组**（如实现和测试详细信息和文档）复制数据。

    完成后，选择 " **下一步**"。

6. **审阅并完成：** 向导的最后一个屏幕显示为评估选择的模板、名称和组。 您可以从屏幕上的链接中编辑这些设置中的任何设置，这将返回到向导中的相关步骤。 准备就绪后，选择 " **创建评估**"。

7. 下一个屏幕确认您已成功创建新的评估。 选择 " **完成** " 以关闭向导，新评估的 "详细信息" 页将显示在屏幕上。

如果在选择 "**创建评估**" 后看到 "**评估失败**" 屏幕，请选择 "**重试**" 以重新创建评估。

您可以通过选择[评估的详细信息页面](#monitor-assessment-progress-and-controls)右上角的 "**编辑名称**" 按钮，在创建评估后更改评估的名称。

## <a name="extend-a-pre-built-assessment"></a>扩展预建评估

您可以通过向评估的模板添加自己的控件和改进操作来修改预建评估。 此过程在合规性管理器中称为 "扩展 Microsoft 模板"。 扩展评估的模板时，它将接收由 Microsoft 发布的任何更新，如果存在对相关规章或 (产品的更改，则可能会发生此问题。请参阅 [接受对评估的更新](#accepting-updates-to-assessments)) 。

你将通过 **评估模板** 页面（而不是 **评估** 页面）开始完成此过程。

**开始之前**

若要准备此过程，首先需要组合一个特殊格式的 Excel 电子表格，以导入所需的模板数据。 扩展过程中使用的 [格式化 Excel 文件](compliance-manager-templates.md#formatting-your-template-data-with-excel) 有特殊要求。 请参阅以下附加要点以帮助防止导入过程中出现错误：

- 您的电子表格应仅包含要添加到评估中的操作和控件。 
- 电子表格不能包含要修改的评估中已有的任何控件或操作。
- 请考虑在模板标题中包括 "分机"，例如，"GDPR – [公司名称] 扩展"。 这样，可以更轻松地在 " **评估模板** " 页上的列表中查找不同于标准 Microsoft 提供的模板或具有类似名称的自定义模板。

设置电子表格的格式后，请执行以下步骤。

**扩展合规性管理器模板的步骤**

1. 转到 " **评估模板** " 页，并选择 " **新建模板**"。 将打开 "模板创建" 向导。

2. 选择要创建的模板的类型。 在这种情况下，选择 " **扩展 Microsoft 模板**"，然后 **选择**""。

3. 屏幕右侧将显示一个 "模板选择" 飞出窗格。 使用 " **搜索** " 来应用用于查找所需模板的筛选器

4. 找到模板后，选择其名称左侧的单选按钮，然后选择 " **保存**"。

5. 下一个屏幕显示您选择的模板。 如果正确，请选择 " **下一步**"。  (如果不正确，请选择 " **选择其他模板** " 以再次选择。 ) 

6. 在 " **上载文件** " 屏幕中，选择 " **浏览** " 以查找并上传包含所有所需模板数据的格式化的 Excel 文件。

7. 如果您的文件没有问题，下一个屏幕将显示上传的文件的名称。 选择 " **下一步** " 继续 (如果需要更改文件，请选择 " **上传其他文件** ") 。

    - 如果文件有问题，则在顶部出现一条错误消息，说明出现了什么问题。 您需要修复并重新上载您的文件。 如果电子表格格式不正确，或者某些域中存在无效信息，则会导致出错。
 
8. " **审阅" 和 "完成** " 屏幕显示模板的改进操作和控件数以及最大分数。 准备好审核后，选择 " **下一步**"。  (如果需要进行更改，请选择 " **上载其他文件**"。 ) 

9. 最后一个屏幕确认已创建新模板。 选择 " **完成** " 退出向导。

10. 你将看到新模板的详细信息页。 从这里，你可以通过选择 " **创建评估**" 来创建评估。 有关指南，请从步骤 #4 中的 [评估创建说明](#to-create-an-assessment)开始。

## <a name="create-your-own-custom-assessment"></a>创建自己的自定义评估

若要在合规性管理器中创建自定义评估，需要创建自己的模板。 若要创建自己的模板，首先需要组合一个格式化的 Excel 电子表格，以导入所需的模板数据。 此外，它还有助于在创建评估时提前决定将评估分配到哪个组 (了解有关 [组](#what-are-groups)) 的详细信息。

**若要创建自定义评估，请执行以下步骤：**

1. **格式化您的 Excel 文件。** 首先，使用 [这些说明](compliance-manager-templates.md#formatting-your-template-data-with-excel)将模板数据格式化为 Excel 电子表格。

2. 按照以下[说明](compliance-manager-templates.md#create-a-new-template)**创建模板**。

3. 从模板**创建评估**。 您可以先打开模板的详细信息页面，选择 " **创建评估**"，或者转到 " **评估** " 页面并选择 " **创建评估**"。

4. "评估创建向导" 将出现在大型浮出控件窗格中。 在这里，您可以按照 [评估创建说明](#to-create-an-assessment)中的步骤 #3 的指导，使用您的新的自定义模板进行评估。

## <a name="delete-an-assessment"></a>删除评估

删除评估会将其从评估页面上的列表中删除。 请注意以下关于删除评估的要点：

- **删除评估是永久性的;您无法再取回它。** 如果要再次使用同一评估，则需要重新创建它。
- 如果评估中未出现评估中的改进操作，则在删除评估时，将删除这些评估。
- 建议您先 [导出评估报告](#export-an-assessment-report) ，然后再将其永久删除。

若要删除评估，请执行以下步骤：

1. 从 **评估** 页中，选择要删除的评估以打开该评估的 "详细信息" 页。

2. 选择屏幕右上角的 " **删除评估** "。

3. 将显示一个窗口，要求您确认是否要永久删除该评估。 选择 " **删除评估** " 以关闭该窗口。 你将看到一个确认窗口，表明评估已从合规性管理器中删除。

如果您删除组中的唯一评估，则该组也会从合规性管理器中删除。

> [!NOTE]
> 您不能删除所有评估。 组织需要至少一次评估合规性管理器才能正常工作。 如果您想要删除的评估是唯一的评估，请在删除其他评估之前添加另一个评估。

## <a name="monitor-assessment-progress-and-controls"></a>监视评估进度和控制

每个评估都有一个详细信息页面，可在完成评估的过程中提供一览的一览视图。 页面显示您在完成控件中的进度，以及这些控件中的关键改进操作的测试状态。

### <a name="overview-tab"></a>概述选项卡

"概述" 选项卡包含一个显示评估结束百分比的图形。 此图包含来自你所拥有的操作的点细目，以及 Microsoft 所拥有的操作的点，因此你可以看到完成此评估所需的更多分数。

评估中的控件的主要改进操作是按对赢得积分的最大潜在影响顺序列出的。 关联的关系图详细说明了您的改进操作的聚合测试状态，以便您可以快速估量已测试的内容以及仍需要完成的操作。

若要访问各个改进操作，请访问 " **控件** " 选项卡或 " **改进操作** " 选项卡。

### <a name="controls-tab"></a>控件选项卡

"控件" 选项卡显示映射到评估的每个控件的详细信息。 **控件状态细分**图按系列显示控件的状态，因此您可以一目了然地查看控件的哪些组需要关注。

在图表下，表中列出了评估中每个控件的详细信息。 控件按控件系列分组。 展开每个系列名称以显示它所包含的各个控件。 为每个控件列出的信息包括：

- **控件标题**
- **状态**：反映控件中的改进操作的测试状态 
    - 已**通过**-所有改进操作的测试状态均为 "已通过"，或者至少有一个已传递且其余值为 "超出范围"
    -  **失败** -至少有一个改进操作的测试状态为 "Failed"
    - **None** -尚未测试所有改进操作
    - **超出范围** -此评估的所有改进操作均超出范围
    - **进度** 改善操作的状态不是上面列出的状态，可能包括 "正在进行中"、"部分信贷" 或 "未检测"。
- **控件 ID**：控件的标识号，由其对应的法规、标准或策略分配
- **实现分数**：完成操作所得到的分数，即实现的总积分数 
- **您的操作**：要完成的操作的总数已完成的操作数
- **Microsoft 操作**： microsoft 完成的操作数 

若要查看控件的详细信息，请从表的行中选择它。 "控件详细信息" 页将显示一个图，指示该控件中的操作的测试状态。 图下方的表格显示了该控件的关键改进操作。

从列表中选择 "改进操作"，以深入了解改进操作的详细信息页面。 详细信息页面显示了测试状态和实现说明，并启动了推荐的解决方案。

### <a name="your-improvement-actions-tab"></a>您的改进操作选项卡

改进操作的选项卡列出了由组织管理的评估中的所有控件。 状态栏详细介绍了评估中的改进操作的聚合测试状态，以便您可以快速估量已测试的内容以及仍需完成的操作。 在此栏下面是改进操作和关键详细信息的完整列表，包括：测试状态、潜在和获得的积分数、相关的法规和标准、适用的解决方案、操作类型和控制族。 了解有关 [操作如何影响合规性分数](compliance-score-calculation.md#action-types-and-points)的详细信息。

选择要查看其详细信息页面的改进操作，然后选择 " **立即启动** " 链接以打开要执行操作的解决方案。

### <a name="microsoft-actions-tab"></a>Microsoft 操作选项卡

Microsoft "操作" 选项卡列出了由 Microsoft 管理的评估中的所有操作。 该列表显示了关键操作详细信息，包括：测试状态、有助于总体符合性分数的点、相关的规章和标准、适用的解决方案、操作类型和控制族。 选择要查看其详细信息页面的改进操作。

了解有关[如何跟踪和评分控件和改进操作的](compliance-score-calculation.md)详细信息。

## <a name="accepting-updates-to-assessments"></a>接受对评估的更新

如果有可供评估的更新，你将看到一条通知，可以选择接受更新，也可以将其推迟一段时间。

### <a name="what-causes-an-update"></a>什么导致更新

当存在影响计分的基础模板更改时，将发生评估更新。 更改可能需要根据法规更改或产品更改来调整控制映射或其他指导。 评估更新可以从组织发起 (例如，在 [修改自定义模板](compliance-manager-templates.md#modify-a-template) 时) 也可以从 Microsoft 进行修改。

如果 Microsoft 更新了您扩展的合规性管理器模板，则在接受这些更新之后，您的评估将继承这些更新。 您的评估将保留您在扩展时应用于评估的其他属性。

您创建的自定义评估不会接收来自 Microsoft 的任何模板更新。 自定义评估可以接收改进操作更新，但任何 Microsoft 用于控制评估和改进操作之间的映射的更新不适用于自定义模板。

> [!NOTE]
> 对评估的更新仅适用于组级别。 如果从同一模板生成两个不同组中的两个评估，每个评估都将有一个挂起的更新通知，并且您需要分别接受对各自组中每个评估的更新。

#### <a name="where-youll-see-assessment-update-notifications"></a>您将在其中看到评估更新通知

评估详细信息页面还显示更新的评估旁边的 **挂起的更新** 标签。 选择该评估以获取其详细信息页面。

"评估详细信息" 页顶部附近有一条消息，显示该评估可进行更新。 选择横幅中的 " **审阅更新** " 按钮，以查看特定更改并接受或推迟更新。

评估详细信息页面也可能列出了其旁边有 **挂起的更新** 标签的改进操作。 这些更新用于对改进操作本身进行特定更改，并需要单独接受。 访问 [接受对改进操作的更新](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions) 以了解详细信息。

#### <a name="review-update-to-accept-or-defer"></a>查看更新以接受或推迟

从 "评估详细信息" 页面中选择 " **查看更新** " 后，屏幕右侧将显示一个浮出控件窗格。 浮出控件窗格提供以下关于待处理更新的关键详细信息：

- 模板标题
-  (Microsoft、组织或特定用户的更新源) 
- 更新的创建日期
- 说明更新的概述
- 有关这些更改的具体详细信息，包括对合规性分数的影响、评估的完成进度以及对改进操作和控件的特定数量的更改。

选择 " **更新的模板** " 链接将下载包含挂起更新的模板版本的控制数据的 Excel 文件。 选择 " **当前模板** " 链接将下载现有模板的文件，而不进行更改。

若要接受更新并对评估进行更改，请选择 " **接受更新**"。 接受的更改是永久性的。

如果选择 " **取消**"，则不会将更新应用于评估。 但是，在接受更新之前，你将继续查看 **待处理的更新** 通知。

**为什么我们建议接受更新**

接受更新有助于确保你具有使用解决方案的最新指南，并采取适当的改进措施以帮助你满足现有证书的要求。

**您可能需要推迟更新的原因**

如果你正在完成评估，你可能需要确保你已完成对此评估的工作，然后才接受对可能中断控制映射的评估的更新。 您可以稍后推迟更新，方法是在 "审阅更新" 浮出控件窗格中选择 " **取消** "。

## <a name="export-an-assessment-report"></a>导出评估报告

您可以将评估导出到您组织中的合规性利益干系人或外部审计员和主管机构的 Excel 文件。 在 "评估详细信息" 页上，选择页面顶部附近的 " **生成报告** " 按钮，该按钮将创建可以保存和共享的 Excel 文件。

报告是在导出的日期和时间进行评估的快照。 它包含由您和 Microsoft 管理的控件的详细信息，包括实施状态、测试日期和测试结果。
