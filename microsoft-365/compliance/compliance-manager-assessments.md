# <a name="build-and-manage-assessments-in-compliance-manager"></a>在合规性管理器中生成和管理评估

**本文内容：** 了解如何通过创建和管理评估来为组织自定义合规性 **管理器**。 本文将引导你完成如何创建评估、如何将它们组织成组、使用控制措施、接受 **更新** 以及导出 **评估报告。**

> [!IMPORTANT]
> 组织可用的评估取决于您的许可协议。 [查看详细信息](https://go.microsoft.com/fwlink/?linkid=2132371)。

## <a name="introduction-to-assessments"></a>评估简介

合规性管理器可帮助你管理对适用于组织的法规和认证评估的合规性。 评估是特定法规、标准或策略中的控制措施分组。 合规性管理器通过提供涵盖各种行业和区域法规和认证的预建评估，轻松开始跟踪合规性。

每个评估都是从评估 [模板创建的](compliance-manager-templates.md)。 模板充当一个框架，其中包含完成评估所需的控制措施、改进操作和 Microsoft 操作。 为组织设置最相关的评估可以帮助您实施策略和操作过程，以限制合规性风险。

所有评估都列在评估页面上。 了解有关如何[筛选评估视图和解释状态状态的信息。](compliance-manager-setup.md#assessments-page)

## <a name="data-protection-baseline-default-assessment"></a>数据保护基线默认评估

为了开始操作，Microsoft 在合规性管理器中提供了 **Microsoft 365** 数据保护基线的默认评估。 此基线评估对数据保护和一般数据管理的关键法规和标准具有一组控制措施。 此基线主要从 NIST CSF (国家标准和技术网络安全框架) 和 ISO (国际标准化组织) 以及欧盟) 的 FedRAMP (联邦风险和授权管理计划) 和 GDPR (一般数据保护条例中绘制元素。

此评估用于在配置任何其他评估之前，在首次访问合规性管理器时计算初始合规性分数。 合规性管理器收集来自 Microsoft 365 解决方案的初始信号。 你将一目了然地了解组织相对于关键数据保护标准和法规的表现，并查看建议采取的改进措施。

随着构建和管理自己的评估以满足组织的特定需求，合规性管理器将变得更加有用。

## <a name="assessment-creation-overview"></a>评估创建概述

有三种方法可以设置评估：

1. [使用预建评估](#use-a-pre-built-assessment)。
2. [扩展预建评估以满足自己的需求](#extend-a-pre-built-assessment)。
3. [创建自己的自定义评估](#create-your-own-custom-assessment)。

> [!NOTE]
> 只有具有全局管理员或合规性管理器管理角色的用户才能创建和修改评估。 详细了解角色 [和权限](compliance-manager-setup.md#set-user-permissions-and-assign-roles)。

**使用预建评估**

通过选择合规性管理器已设置的评估，开始合规性之旅。 我们提供与行业、地区和通用[](compliance-manager-templates.md)数据保护标准（如 GDPR 和 ISO 27001）一致的法规和认证的广泛选择模板。 模板包含用于帮助你满足特定认证要求的控制措施和改进操作。 开始生成评估时，将要求您 [选择模板](#use-a-pre-built-assessment)。

**扩展预建评估以满足您的需求**

您可以通过添加自己的控制措施和操作来修改合规性管理器评估（我们称其为"扩展"的过程）以更好地满足组织的需求。 例如，如果你通常需要遵守 HIPAA，但需要额外的数据保护或安全控件，可以通过向 HIPAA 模板添加自己的控件来扩展 HIPAA 模板。 请参阅扩展 [预建评估的说明](#extend-a-pre-built-assessment)。

**创建自己的自定义评估**

你可以完全从头开始创建自己的评估，以精确跟踪组织的需求。 创建自己的评估需要您首先在合规性管理器中创建自己的评估模板。 请参阅有关创建自己的 [自定义评估的说明](#create-your-own-custom-assessment)。

## <a name="understand-groups-before-creating-assessments"></a>在创建评估之前了解组

创建或修改评估之前，了解组如何工作非常重要。 创建评估时，需要在评估过程中将其分配给组。 这就是我们建议你在创建评估之前规划评估的分组策略的原因。

### <a name="what-are-groups"></a>什么是组

组是允许您组织评估的容器。 可以按按年或按法规或基于组织的部门或地理位置按逻辑对评估进行分组。 下面是两个组及其基础评估的示例：

- **FFIEC IS assessment 2020**
  - FFIEC IS
- **数据安全和隐私评估**
  - ISO 27001：2013
  - ISO 27018：2014

当同一组中的两个不同评估由你管理共享改进操作时，对操作的实施详细信息或状态进行的任何更新将自动同步到组中任何其他评估中的相同操作。 通过此同步，您可以实施一项改进操作，并满足多个法规中的多个要求。

### <a name="how-to-create-a-group"></a>如何创建组

在创建新评估的过程中 [创建一个组](#to-create-an-assessment)。

组不能创建为独立实体。 一个组必须至少包含一个评估。 若要创建组，需要首先创建评估以放入组中。

### <a name="what-to-know-when-working-with-groups"></a>使用组时需要知道哪些方面

- 组名称在组织中必须是唯一的。
- 组没有安全属性。 所有权限都与评估相关联。
- 向组添加评估后，无法更改分组。
- 完成时，同一组中不同评估中的相关评估控制措施将自动更新。
- 如果向现有组添加新评估，则该组中评估的常见信息将复制到新评估中。
- 组可以包含针对相同认证或法规的评估，但每个组只能包含一个特定产品认证对的评估。 例如，组不能包含 Office 365 和 NIST CSF 的两项评估。 只有针对同一产品的相应认证或法规不同时，组才能包含同一产品的多个评估。
- 删除评估会破坏该评估与组之间的关系。
- 无法删除组。
- 当对在多个组中出现的改进进行了更改时，该更改将反映在该改进操作的所有实例中。

## <a name="use-a-pre-built-assessment"></a>使用预建评估

从合规性管理器模板创建评估有两个起点。

可以通过选择"添加评估"按钮，然后通过评估创建向导开始评估页面的过程。 此过程的步骤如下。

您还可以从评估模板页面开始，具体方法为查找您想要的模板，然后从列表中选择模板以到达其详细信息页面。 在模板详细信息页上，选择"**创建评估"。** 然后，输入向导，同时已选择模板。

### <a name="to-create-an-assessment"></a>创建评估

1. 了解你要将评估分配给哪个组，或准备为此评估创建新组。 [了解有关组更多信息](#understand-groups-before-creating-assessments)。  

2. 转到合规性管理器 **中的** 评估页面，然后选择"**添加评估"。** 评估向导将显示在大型飞出窗格中。

3. **选择模板**：选择模板作为评估的基础。 你将看到分为包括和高级类别的模板列表， ([模板](compliance-manager-templates.md#template-types-included-and-premium-active-and-inactive) 类型了解详细信息) 。 选择所选模板旁边的单选按钮，然后选择"下一 **步"。**

4. **名称和组：** 在"评估名称"字段中输入 **评估的名称** 。 评估名称在组中必须是唯一的。 如果评估名称与任意给定组中另一个评估的名称匹配，您将收到一个错误，要求您创建其他名称。

5. 将评估分配给组。 您可以：
    - 选择 **"使用现有** 组"将其分配给已创建的组;或
    - 选择 **"创建新组** "以创建新组并为其分配此评估：
        - 确定组的名称，在单选按钮下方的字段中输入该名称。
        - 通过 **选择相应的框**，可以从现有组复制数据，如实现和测试详细信息和文档。

    完成后，选择"下一 **步"。**

6. **查看并完成：** 向导的最后一个屏幕显示为评估选择的模板、名称和组。 你可以从屏幕上的链接编辑其中任何设置，这将返回到向导中的相关步骤。 准备就绪后，选择"创建 **评估"。**

7. 下一个屏幕确认已成功创建新评估。 选择 **"** 完成"关闭向导，屏幕上将显示新评估的详细信息页面。

如果在选择"创建 **评估"后** 看到"评估 **失败"** 屏幕，请选择"重试"以重新创建评估。

创建评估后，可以通过选择评估详细信息页面右上角的"编辑名称"按钮来更改[评估的名称](#monitor-assessment-progress-and-controls)。

## <a name="extend-a-pre-built-assessment"></a>扩展预建评估

可以通过向评估模板添加自己的控制措施和改进操作来修改预建评估。 此过程在合规性管理器中称为"扩展 Microsoft 模板"。 扩展评估模板时，它将收到 Microsoft 发布的任何更新，当相关法规或产品变更时，可能会 (接受评估更新) 。 [](#accepting-updates-to-assessments)

您将从评估模板页面（而不是评估页面）**开始完成** 此过程。

**开始之前**

若要准备此过程，首先需要组合一个特殊格式的 Excel 电子表格，以导入必要的模板数据。 扩展过程中使用的 [格式化 Excel 文件](compliance-manager-templates.md#formatting-your-template-data-with-excel) 有特殊要求。 请参阅以下其他点以帮助防止导入过程中出现错误：

- 电子表格应仅包含要添加到评估的操作和控制。 
- 电子表格不能包含要修改的评估中已存在的任何控件或操作。
- 请考虑在模板的标题中包括"扩展"，例如，"GDPR – [你的公司名称] 扩展"。 这样一来，在评估模板页面上的列表中标识与 Microsoft 提供的标准模板或名称相似的自定义模板不同，就更容易了。

设置电子表格格式后，请按照以下步骤操作。

**扩展合规性管理器模板的步骤**

1. 转到"评估 **模板"** 页，然后选择 **"创建新模板"。** 将打开模板创建向导。

2. 选择要创建的模板类型。 在这种情况下，选择 **"扩展 Microsoft 模板"，** 然后选择 **"Microsoft 模板"。**

3. 模板选择飞出窗格显示在屏幕的右侧，其中显示所有模板的列表及其活动状态或非活动状态。 **已激活的模板** 计数器显示当前使用的模板数量在可用总数中。 如果超过限制，消息栏将发出通知。 有关详细信息 [，请参阅](compliance-manager-templates.md#template-types-included-and-premium-active-and-inactive) 模板类型。

4. 模板选择飞出窗格显示在屏幕的右侧。 使用 **搜索** 应用筛选器以查找您想要的模板

5. 找到模板后，选择其名称左侧的单选按钮，然后选择"保存 **"。**

6. 下一个屏幕将显示所选的模板。 如果正确，请选择"下 **一步"。**  (如果不正确，请选择 **"选择其他模板** "以再次) 

7. 在 **"上载文件** "屏幕上，选择" **浏览** "以查找并上载包含所有所需模板数据的格式化 Excel 文件。

8. 如果文件没有问题，下一个屏幕将显示已上载文件的名称。 如果需要 **更改** (，请选择"下一步"继续操作，选择"上载其他 **) 。**

    - 如果文件有问题，顶部的错误消息将说明问题所在。 你需要修复并重新上传文件。 如果电子表格的格式不正确，或者某些字段中的信息无效，则会导致错误。
 
9. " **审阅和完成** "屏幕显示改进操作和控件的数量以及模板的最大分数。 准备好批准时，选择"下一 **步"。**  (如果需要进行更改，请选择"上载 **其他文件**") 

10. 最后一个屏幕确认已创建一个新模板。 选择 **"完成** "退出向导。

11. 你将到达新模板的详细信息页面。 在这里，可以通过选择"创建评估 **"来创建评估**。 有关指导，请从#4评估创建说明 [的步骤开始](#to-create-an-assessment)。

## <a name="create-your-own-custom-assessment"></a>创建自己的自定义评估

在合规性管理器中创建自定义评估需要您创建自己的模板。 若要创建自己的模板，您将首先组合格式化的 Excel 电子表格以导入必要的模板数据。 它还有助于提前确定创建评估时将评估分配到的组， ([组) 。](#what-are-groups)

**按照以下步骤创建自定义评估：**

1. **设置 Excel 文件的格式。** 首先，按照以下说明将模板数据格式化为 Excel [电子表格](compliance-manager-templates.md#formatting-your-template-data-with-excel)。

2. **按照以下说明**[创建模板](compliance-manager-templates.md#create-a-new-template)。

3. **从模板创建** 评估。 可以先打开模板的详细信息页面 **并选择"创建** 评估"，或转到评估页面并选择"**创建评估"。** 

4. 评估创建向导将显示在大型飞出窗格中。 在这里，可以使用新的自定义模板#3评估创建说明的步骤开始遵循指导。 [](#to-create-an-assessment)

## <a name="delete-an-assessment"></a>删除评估

删除评估会将其从评估页面上的列表中删除。 请注意以下有关删除评估的要点：

- **删除评估是永久性的;你无法返回它。** 如果要再次使用相同的评估，则需要重新创建它。
- 如果评估中的改进操作未显示在任何其他评估中，则删除评估时将删除这些改进操作。
- 建议 [在永久删除](#export-an-assessment-report) 评估之前导出评估报告。

若要删除评估，请按照以下步骤操作：

1. 从 **评估页面** ，选择要删除的评估，以打开该评估的详细信息页面。

2. 选择 **屏幕** 右上角的"删除评估"。

3. 将显示一个窗口，要求您确认是否要永久删除评估。 选择 **"删除评估** "关闭窗口。 你会收到一个确认窗口，表明你的评估已从合规性管理器中删除。

如果删除组中的唯一评估，则该组也会从合规性管理器中删除。

> [!NOTE]
> 无法删除所有评估。 组织需要至少一项评估，合规性管理器正常运行。 如果要删除的评估是唯一的评估，请添加另一个评估，然后再删除另一个评估。

## <a name="monitor-assessment-progress-and-controls"></a>监视评估进度和控制措施

每个评估都有一个详细信息页面，其中提供了完成评估进度的概览。 该页显示您完成控件的进度，以及这些控件中关键改进操作的测试状态。

### <a name="overview-tab"></a>"概述"选项卡

"概述"选项卡包含显示完成评估的百分比的图形。 此图包含你所拥有的操作和 Microsoft 所拥有的操作中的点细分，因此你可以看到完成评估还需要多少点。

评估中控制措施的关键改进操作按对获得分数的潜在影响最大顺序列出。 关联的图详细介绍了改进操作聚合的测试状态，以便快速判断已经过测试了哪些内容以及需要完成哪些操作。

若要访问各个改进操作，请访问" **控件** "选项卡或 **"改进操作"** 选项卡。

### <a name="controls-tab"></a>控件选项卡

"控件"选项卡显示映射到评估的每个控制措施的详细信息。 **控件状态细分** 图表按系列显示控件的状态，因此您可以一目了然地查看哪些控件分组需要关注。

在图表下方，表格列出了有关评估中每个控件的详细信息。 控件按控件系列分组。 展开每个系列名称以显示其包含的单个控件。 列出的每个控件的信息包括：

- **控件标题**
- **状态**：反映控件中改进操作的测试状态 
    - **已通过** - 所有改进操作的测试状态为"已通过"，或至少通过了一项，其余操作均"超出范围"
    -  **失败** - 至少有一个改进操作的测试状态为"失败"
    - **无** - 所有改进操作均未经过测试
    - **超出范围** - 所有改进操作均超出此评估范围
    - **正在进行** - 改进操作的状态与上面列出的状态不同，可能包括"正在进行"、"部分信用"或"未检测"
- **控制 ID：** 控件的标识号，由相应法规、标准或策略分配
- **已实现的** 点数：完成操作获得的点数（在可达到的分数总数中） 
- **操作**：要完成的操作总数中已完成的操作数
- **Microsoft 操作**：Microsoft 完成的操作数 

若要查看控件的详细信息，请从其表中的行中选择它。 控件详细信息页显示一个指示该控件内操作的测试状态的图形。 下图显示了该控件的关键改进操作。

从列表中选择一个改进操作，以深入了解改进操作的详细信息页。 详细信息页显示测试状态、实现说明，并启动到建议的解决方案中。

### <a name="your-improvement-actions-tab"></a>"改进操作"选项卡

改进操作选项卡列出了评估中由组织管理的所有控制措施。 状态栏详细说明评估中改进操作聚合的测试状态，以便快速衡量已经过测试的情况以及需要完成的操作。 栏下方是改进操作的完整列表和关键详细信息，包括：测试状态、潜在分数和挣点数、相关法规和标准、适用的解决方案、操作类型和控制系列。 了解有关操作 [如何参与合规性分数的更多信息](compliance-score-calculation.md#action-types-and-points)。

选择改进操作以查看其详细信息页面，然后选择"立即启动" **链接以** 打开解决方案以采取措施。

### <a name="microsoft-actions-tab"></a>Microsoft 操作选项卡

"Microsoft 操作"选项卡列出了评估中由 Microsoft 管理的所有操作。 该列表显示了关键操作详细信息，包括：测试状态、参与总体合规性分数的分数、相关法规和标准、适用的解决方案、操作类型和控制系列。 选择改进操作以查看其详细信息页面。

详细了解如何 [跟踪控制措施和改进操作并评分。](compliance-score-calculation.md)

## <a name="accepting-updates-to-assessments"></a>接受评估更新

当更新可用于评估时，你将看到一条通知，并可以选择接受更新或延迟更新，稍后再进行。

### <a name="what-causes-an-update"></a>导致更新的原因

当存在影响评分的基础模板更改时，将发生评估更新。 更改可能涉及根据法规更改或产品更改调整控制映射或其他指南。 评估更新可能源自 (，例如，在 Microsoft 中修改) 模板[](compliance-manager-templates.md#modify-a-template)时。

如果 Microsoft 更新你扩展的合规性管理器模板，你的评估将在你接受这些更新后继承这些更新。 在扩展评估时，评估将保留应用于评估的其他属性。

您创建的自定义评估不会收到来自 Microsoft 的任何模板更新。 自定义评估可以接收改进操作更新，但控制评估和改进操作之间的映射的任何 Microsoft 更新不适用于自定义模板。

> [!NOTE]
> 评估更新仅适用于组级别。 如果你有两个基于同一模板构建的评估，它们存在于两个不同的组中，则每个评估将具有一个挂起的更新通知，并且你将需要单独接受对其各自组中每个评估的更新。

#### <a name="where-youll-see-assessment-update-notifications"></a>你将在哪里看到评估更新通知

评估详细信息页面 **还会在评估** 旁边显示一个挂起的更新标签，并带有更新。 选择该评估以进入其详细信息页面。

评估详细信息页面顶部附近的消息显示该评估提供了更新。 选择 **横幅中的** "查看更新"按钮，查看特定更改并接受或延迟更新。

评估详细信息页可能还列出了旁边有"挂起的更新" **标签的** 改进操作。 这些更新适用于对改进操作本身进行的特定更改，需要单独接受。 访问 ["接受更新以改进操作](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions) "以了解更多信息。

#### <a name="review-update-to-accept-or-defer"></a>查看更新以接受或延迟

从 **评估详细信息** 页面选择"审阅更新"后，屏幕右侧将显示一个飞出窗格。 该飞出窗格提供了下面有关待批更新的关键详细信息：

- 模板标题
- Microsoft、 (或特定用户用户的更新源) 
- 更新的创建日期
- 说明更新的概述
- 有关更改的特定详细信息，包括对合规性分数的影响、评估完成进度以及改进操作和控制措施的特定更改数量。

选择 **"已更新的模板** "链接将下载一个 Excel 文件，其中包含包含待定更新的模板版本的控件数据。 选择 **"当前模板** "链接将下载现有模板的文件，而不进行更改。

若要接受更新并更改评估，请选择"接受 **更新"。** 接受的更改是永久性的。

如果选择" **取消**"，更新将不会应用于评估。 但是，您将继续看到"挂起的更新" **通知，** 直到您接受更新。

**我们建议接受更新的原因**

接受更新有助于确保你拥有有关使用解决方案和采取相应改进操作的最新指南，以帮助你满足当前认证的要求。

**为什么您可能需要延迟更新**

如果正在完成评估，你可能希望确保在接受可能中断控制映射的评估更新之前已完成有关评估的工作。 可以在审阅更新飞出窗格中选择"取消"，将更新延迟一段时间。

## <a name="export-an-assessment-report"></a>导出评估报告

可以将评估导出到 Excel 文件中，供组织中合规性利益干系人或外部审核员和监管机构使用。 在评估详细信息页上，选择页面顶部附近的"生成报告"按钮，这将创建可保存和共享的 Excel 文件。

该报告是导出日期和时间的评估快照。 它包含由你和 Microsoft 管理的控制措施的详细信息，包括实现状态、测试日期和测试结果。
