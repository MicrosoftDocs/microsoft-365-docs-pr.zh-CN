# <a name="get-started-with-compliance-manager"></a>合规性管理器入门

**本文内容：** 本文将帮助您设置合规性管理器。 了解如何 **访问** 合规性管理器、 **设置角色和权限**以及配置 **改进操作的自动测试**。 浏览 **合规性管理器仪表板** 并了解主要页面： "改进操作" 页、"解决方案" 页、"评估" 页和 "评估模板" 页。

## <a name="who-can-access-compliance-manager"></a>谁可以访问合规性管理器

合规性管理器可供具有 Office 365 和 Microsoft 365 许可证的组织使用。 评估可用性和管理功能取决于许可协议。  [查看服务说明详细信息](https://go.microsoft.com/fwlink/?linkid=2132371)。

## <a name="before-you-begin"></a>准备工作

您的组织的 Microsoft 365 全局管理员可能是第一个访问合规性管理器的用户。 我们建议全局管理员登录并设置用户权限，如下文首次访问合规性管理器时所述。

## <a name="sign-in"></a>登录

1. 请转到 [microsoft 365 合规性中心](https://compliance.microsoft.com/) 并使用 microsoft 365 全局管理员帐户 **登录** 。
2. 在左侧导航窗格中选择 " **合规性管理器** "。 你将收到 [合规性管理器仪表板](#understand-the-compliance-manger-dashboard)。

指向 access 合规性管理器的直接链接为 [https://compliance.microsoft.com/compliancemanager](https://compliance.microsoft.com/compliancemanager) 。

## <a name="set-user-permissions-and-assign-roles"></a>设置用户权限并分配角色

合规性管理器使用基于角色的访问控制 (RBAC) 权限模型。 只有分配有角色的用户才可以访问合规性管理器，并且每个用户允许的操作受 [角色类型](#role-types)的限制。

### <a name="where-to-set-permissions"></a>设置权限的位置

拥有组织的全局管理员角色的人员可以在 Microsoft 365 合规性中心中设置用户权限，也可以在 Azure Active Directory (Azure AD) 中设置用户权限。

若要在 Microsoft 365 合规性中心中设置权限和分配角色，请执行以下步骤：

1. 从[Microsoft 365 合规性中心](https://compliance.microsoft.com/)的任意位置选择左侧导航中的 "**权限**"。

2. 在顶部附近，选择 **"要在 Office 365 中查看和管理角色" 中的链接，请转到此处。** 一个新的选项卡将打开到 Office 365 安全 & 合规性中心 ([了解你为什么重定向](microsoft-365-compliance-center.md#frequently-asked-questions)) 。

3. 查找要向其添加一个或多个用户的角色组，并选中该组名称左侧的框。  (请参阅 [下面的角色和相关功能的列表](#role-types)。 对角色名称进行模仿的角色组名称。 ) 

4. 在该组的浮出控件窗格中，选择 "**成员**" 标题下的 "**编辑**"。

5. 选择 " **选择成员**"。 将显示另一个浮出控件窗口。

6. 选择 " **+ 添加** " 以选择要添加到组中的一个或多个用户。

7. 选中您要添加的名称旁边的复选框，然后选择底部的 " **添加** " 按钮。

8. 完成分配用户后，选择 " **完成**"，然后选择 " **保存**"，然后单击 " **关闭**"。

##### <a name="more-about-the-office-365-secruity--compliance-center"></a>有关 Office 365 Secruity & 合规中心的详细信息

了解有关 [Office 365 安全 & 合规中心中的权限](https://docs.microsoft.com/microsoft-365/security/office-365-security/permissions-in-the-security-and-compliance-center)的详细信息。

如果您没有访问 Office 365 安全与合规中心的权限，或者您需要在 Microsoft 服务信任门户中访问经典版合规性管理器，则服务信任门户中的管理员设置将提供另一种方法来分配角色 ([查看说明](meet-data-protection-and-regulatory-reqs-using-microsoft-cloud.md#assigning-compliance-manager-roles-to-users)) 。 请注意，此类角色在功能方面更具限制。

##### <a name="more-about-azure-ad"></a>有关 Azure AD 的详细信息

若要在 Azure AD 中分配角色和设置权限，请参阅 [向使用 Azure Active Directory 的用户分配管理员角色和非管理员角色](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-users-assign-role-azure-portal)。

拥有未安装 Office 365 或 Microsoft 365 订阅的 Azure AD 标识的用户将无法访问 Microsoft 365 合规性中心中的合规性管理器。 若要在访问合规性管理器中寻求帮助，请联系 [cmresearch@microsoft.com](mailto:cmresearch@microsoft.com)。

## <a name="settings-for-automated-testing-and-user-history"></a>自动测试和用户历史记录的设置

Microsoft 365 合规性中心中的合规性管理器设置允许您启用和禁用改进操作的自动测试。 这些设置还允许您管理与改进操作相关联的用户的数据，其中包括将改进操作重新分配给其他用户的功能。  只有具有全局管理员或合规性管理器管理员角色的用户才可以访问合规性管理器设置。

### <a name="set-up-automated-testing"></a>设置自动测试

合规性管理器中的某些改进操作也受到 [Microsoft 安全分数](https://docs.microsoft.com/microsoft-365/security/mtp/microsoft-secure-score)的监控。 您可以设置对联合监控的操作的自动测试，这意味着在安全分数中测试和更新操作时，这些结果将与合规性管理器中的相同操作进行同步，并计入合规性分数。

默认情况下，为合规性管理器新建的组织启用自动测试。 当您首次部署 Microsoft 365 或 Office 365 时，需要大约7天的时间来完全收集数据，并将其划分为合规性分数。  如果启用了自动测试，则不会更新操作的测试日期，但其测试状态将会更新。 在创建新的评估时，分数将自动包含 Microsoft 控制分数和安全得分集成。

您的组织的全局管理员可以随时更改自动测试的设置。 您可以关闭常见改进操作的自动测试，或为单个操作启用自动测试。 按照下面的说明更改自动测试设置。

#### <a name="to-manage-your-automated-testing-settings"></a>若要管理自动测试设置，请执行以下操作：

1. 从[Microsoft 365 合规性中心](https://compliance.microsoft.com/)的任意位置选择左侧导航中的 "**设置**"。

2. 在 "设置" 页上，选择 " **合规性管理器**"。

3. 从左侧导航栏中选择 " **自动测试** "。

4. 选择适用的按钮，为所有改进操作启用自动测试，将所有操作关闭，或通过单个操作打开。

5. 如果选择 " **打开每个改进操作**"，则会有一个列表显示可供选择的所有可用的提高操作。  选中您希望自动测试的任何操作旁边的框。

6. 选择 " **保存** " 以保存设置。 您将在屏幕顶部收到一条确认消息，指示您所做的选择已保存。 如果收到故障通知，请重试。

**注意：** 只有全局管理员才能打开或关闭所有操作的自动更新。 合规性管理器管理员可以为单个操作启用自动更新，而不是全局执行所有操作。

### <a name="manage-user-history"></a>管理用户历史记录

" **管理用户历史记录** " 设置可帮助您快速识别在合规性管理器中处理了 "改进" 操作的用户。 与提高操作相关联的可识别用户数据包括任何实现和测试工作已完成、文档上传的文档以及所输入的任何注释。 了解和检索此类型的数据可能是组织自己的合规性需求所必需的。

用户历史记录设置还允许您将所有改进操作从一个用户重新分配给另一个用户。

**查找用户历史记录设置：**

1. 从 [Microsoft 365 合规性中心](https://compliance.microsoft.com/)的任意位置选择左侧导航中的 "设置"。

2. 在 "设置" 页上，选择 " **合规性管理器**"。

3. 从左侧导航中选择 " **管理用户历史记录** "。

" **管理用户历史记录** " 页显示了一个列表，其中列出了所有用户分配给改进操作的电子邮件地址。 使用 " **搜索** " 按钮，通过在电子邮件地址中键入来快速查找特定用户。

在每个用户的电子邮件地址的右侧，" **选择** " 下拉菜单提供了导出报表、重新分配改进操作或删除历史记录的选项。 有关每个选项的详细信息，请参阅以下各节。

#### <a name="export-a-report-of-user-history-data"></a>导出用户历史记录数据的报告

您可以导出一个 Excel 文件，其中包含当前分配给用户的改进操作的列表。  该报告还列出了该用户上载的所有证据文件。 此信息可帮助您重新分配打开的改进操作。

该报告反映了在其创建日期后的改进行动的状态。 它并不是对其状态或工作分配的所有以前更改的历史报告 (了解如何 [从 "改进操作" 页) 中导出报告](compliance-manager-improvement-actions.md#export-a-report) 。

**按照下面的步骤，按用户导出报表：**

1. 从[Microsoft 365 合规性中心](https://compliance.microsoft.com/)的任意位置选择左侧导航中的 "**设置**"。

2. 在 "设置" 页上，选择 " **合规性管理器**"。

3. 从左侧的导航栏中选择 " **管理用户历史记录** "。

4. 通过搜索列表电子邮件地址，或通过选择 " **搜索** " 并输入用户的电子邮件地址来查找预期的用户。

5. 从 " **选择** " 下拉菜单中，选择 " **导出报告**"。

6. 生成报告的 Excel 文件后，可以打开它并将其保存到本地计算机。

#### <a name="reassign-improvement-actions-to-another-user"></a>将改进操作重新分配给另一个用户

您可以将提高操作从一个用户重新分配给另一个用户。 重新分配操作时，文档上载历史记录不会更改，但最初上载文档的用户的名称不再出现在改进操作中。

**按照下面的步骤操作，将 "提高" 操作重新分配给另一个用户：**

1. 从[Microsoft 365 合规性中心](https://compliance.microsoft.com/)的任意位置选择左侧导航中的 "**设置**"。

2. 在 "设置" 页上，选择 " **合规性管理器**"。

3. 从左侧的导航栏中选择 " **管理用户历史记录** "。

4. 通过搜索列表电子邮件地址或通过选择 " **搜索** " 并输入该用户的电子邮件地址来查找用户。

5. 从 " **选择** " 下拉菜单中，选择 " **重新分配改进操作**"。 将显示 **重新分配改进操作** 的浮出控件窗格。

6. 在 " **搜索用户** " 字段中，输入要 *向*其分配改进操作的用户的姓名或电子邮件地址。

7. 当您看到 " **改进操作将分配给**" 下一个预期用户的名称时，请选择该用户，然后选择 " **分配操作**"。

8. 重新分配完成后，将在弹出窗格中看到一条确认消息，确认已将以前用户的所有改进操作重新分配给新用户。 如果您收到重新分配失败通知，请关闭该窗口，然后重试。 若要关闭弹出窗口窗格，请选择 " **完成**"。

新的受理人会收到一封电子邮件，他们已将其分配给改进操作。 该电子邮件包含到改进操作的详细信息页面的直接链接。
 
 > [!NOTE]
> 如果重新分配具有挂起的更新的操作，如果重新分配后接受更新，则重新分配的电子邮件中的操作的直接链接将会中断。 您可以通过在接受更新后将操作重新分配给用户来解决此问题。 了解有关 [改进操作更新](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions)的详细信息。

#### <a name="delete-user-history"></a>删除用户历史记录

删除用户的历史记录会将其作为改进操作的所有者删除，并将从合规性管理器中的所有其他字段中删除它们的名称。 当您删除用户的历史记录时，他们拥有的提高操作将不会显示 **分配给** 的值，直到分配新用户。 上载到 "改进" 操作的任何文档将显示 " **用户已删除** " 以代替已删除的用户的姓名。 删除用户历史记录是永久性的。

若要删除用户的历史记录，请执行以下步骤：

1. 从[Microsoft 365 合规性中心](https://compliance.microsoft.com/)的任意位置选择左侧导航中的 "**设置**"。

2. 在 "设置" 页上，选择 " **合规性管理器**"。

3. 从左侧的导航栏中选择 " **管理用户历史记录** "。

4. 通过搜索列表电子邮件地址或通过选择 " **搜索** " 并输入该用户的电子邮件地址来查找用户。

5. 从 " **选择** " 下拉菜单中，选择 " **删除历史记录**"。

6. 将显示一个窗口，要求您确认是否永久删除了用户的历史记录。 若要继续删除，请选择 " **删除历史记录**"。 若要离开而不删除历史记录，请选择 " **取消**"。

7. 您将在 " **管理用户历史记录** " 页上返回，其中有一条确认消息，用户的历史记录已被删除。

## <a name="understand-the-compliance-manger-dashboard"></a>了解合规性管理器仪表板

合规性管理器仪表板旨在向你提供当前合规性状况的一览视图。

![合规性管理器-仪表板](../media/compliance-manager-dashboard.png "合规性管理器仪表板")

### <a name="overall-compliance-score"></a>总体合规性分数

你的合规性分数在顶部一目了然。 它显示的百分比取决于完成解决关键数据保护标准和法规的提高操作所能实现的积分。 来自 [microsoft 操作](compliance-manager-assessments.md#microsoft-actions-tab)的点，这些操作是由 microsoft 托管的，也是按符合性分数计。

当您首次进入合规性管理器时，您的初始分数将基于 [Microsoft 365 数据保护基准](compliance-manager-assessments.md#data-protection-baseline-default-assessment)。 对所有组织可用的此基准评估是一组包含常见行业法规和标准的控件。 合规性管理器扫描您现有的 Microsoft 365 解决方案，并根据您当前的隐私和安全设置为您提供初步评估。 在添加与您的组织相关的评估时，您的分数将对您更有意义。

**了解详细信息：** [了解如何计算合规性分数](compliance-score-calculation.md)。

### <a name="key-improvement-actions"></a>关键改进操作

此部分列出了您现在可以执行的首要改进操作，以对您的总体合规性分数产生最大的积极影响。 选择 " **查看所有改善措施** " 以转到 "改进操作" 页面。

### <a name="solutions-that-affect-your-score"></a>影响你的成绩的解决方案

本节重点介绍了包含可积极影响你的成绩的改进操作的解决方案，以及这些解决方案中未完成的改进操作的数量。 选择 " **查看所有解决方案** 以访问解决方案" 页。

### <a name="compliance-score-breakdown"></a>合规性分数细分

本节提供了以两种不同的方式更详细地查看分数：

- **类别**：显示数据保护类别中整体得分的百分比，例如 "保护信息" 或 "管理设备"。
- **评估**：显示管理特定合规性和数据保护标准、法规或法律（如 GDPR 或 NIST 800-53）的评估进度的百分比。

### <a name="filtering-your-dashboard-view"></a>筛选你的仪表板视图

您可以筛选仪表板视图，以仅查看与特定法规和标准、解决方案、操作类型、评估组或数据保护类别相关的项目。 以这种方式筛选视图还将筛选仪表板上的分数，并根据筛选条件显示已实现的总积分数。

应用筛选器：

1. 选择仪表板右上侧的 " **筛选器** "。
2. 从 " **筛选器** " 浮出控件窗格中选择筛选条件，然后选择 " **应用**"。

在应用筛选器之后，你将看到实时调整的分数。 合规性分数百分比和细目信息以及改进操作和解决方案现在仅适用于您的筛选器条件所覆盖的数据。 如果你注销合规性管理器，则在重新登录时将保留你的筛选视图。

若要删除筛选器：

- 在 " **应用的筛选器** " 标题高于您的合规性分数后，选择要删除的单个筛选器旁边的 **X** ;和
- 选择仪表板右上侧的 " **筛选器** "，然后在 " **筛选器** " 浮出控件窗格中，选择 " **清除筛选器**"。

## <a name="improvement-actions-page"></a>改进操作页

[改进操作](compliance-manager-improvement-actions.md) 是由您的组织管理的操作。 使用改进操作有助于集中化合规性活动并与数据保护法规和标准保持一致。 每个改进操作都提供了详细的实施指导和将您引导您进入适当解决方案的链接。 可将改进操作分配给组织中的用户，以执行实施和测试工作。 您还可以在 "改进" 操作中存储文档、注释和记录状态更新。

### <a name="view-your-improvement-actions"></a>查看改进操作

合规性管理器仪表板显示你的 **主要改进操作。** 若要查看所有改进操作，请选择仪表板上的 "改善操作" 选项卡，此选项卡将带你转到 "改进操作" 页面。 您还可以选择 "查看仪表板上的关键改进操作列表下的所有改进操作"，以转到 "改进操作" 页面。

"改进操作" 页面显示组织管理的所有改进操作。 可以在每个评估中查看由 Microsoft 管理的操作 (了解有关 [microsoft 操作](compliance-manager-assessments.md#microsoft-actions-tab)) 的详细信息。

如果您的 "改进操作" 页面上有很长的操作列表，则筛选视图可能很有帮助。 选择 "操作" 列表右上角的 " **筛选器** "。 当 " **筛选器** " 浮出控件窗格出现时，选择基于法规和标准、解决方案和组的标准。 您还可以通过选择右上角的 " **组** " 来自定义您的视图。 从下拉菜单中，选择按组、解决方案、类别、操作类型或状态进行查看。

此页面的默认视图不显示测试状态为 "已 **通过**" 的提高操作。 若要查看已通过测试的操作，请选中 "筛选器浮出控件" 窗格中的 " **传递** " 框。 仅具有已 **通过** 的测试状态与得分接近的操作。 某些操作可能会显示 " **挂起的更新" 标签。** 了解有关 [改进操作更新](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions)的详细信息。

"改进操作" 页显示每个改进操作的以下数据点：

- **实现点**：完成操作后可用总可用点数
- **法规**：与操作相关的法规或标准
- **组**：为其分配操作的组
- **解决方案**：可在其中执行操作的解决方案
- **评估**：包含操作的评估
- **类别**：相关的数据保护类别 (例如，保护信息、管理设备等 ) 
- **测试状态**：
    - **无** –未记录状态更新
    - **未评估** -测试尚未启动
    - 已成功测试已**通过**的实施
    - **失败的低风险** -测试失败，低风险
    - **失败中等风险** -测试失败，中等风险
    - **失败的高风险** -测试失败，高风险
    - **超出范围** -操作不在评估范围内，不会影响你的成绩
    - **若要检测-若要** 进行手动测试，表明已实现操作但未测试;对于自动测试，指示某个操作正在等待自动化结果
    - **无法检测** -无法确定自动状态
    - **部分测试** –对部分点获奖的自动化评分

**了解详细信息：** 了解 [如何分配和执行改进操作的工作](compliance-manager-improvement-actions.md)。

## <a name="solutions-page"></a>解决方案页

"解决方案" 页显示按解决方案组织的挣和潜在点的份额。 查看此视图中的剩余点和改进操作可帮助您了解哪些解决方案需要更直接关注。

通过选择合规性管理器仪表板上的 " **解决方案** " 选项卡查找 "解决方案" 页。 您还可以选择 "查看下面的 **所有解决方案** **，这些解决方案将影响您** 的仪表板的右上部分中的分数。

### <a name="filtering-your-solutions-view"></a>筛选解决方案视图

筛选解决方案视图：

1. 选择评估列表左上角的 " **筛选器** "。
2. 在 " **筛选器** " 浮出控件窗格中，在所需条件的旁边放置一个复选 (标准和管理法规、解决方案、操作类型、合规性管理器组、类别) 。
3. 选择 " **应用** " 按钮。 筛选窗格将关闭，你将看到筛选出的视图。

您还可以通过从评估列表上方的 **组** 下拉菜单中选择分组类型来修改视图，以查看按组、产品或法规进行的评估。

### <a name="taking-action-from-the-solution-page"></a>从 "解决方案" 页采取操作

"解决方案" 页显示已连接到 "改进操作" 的组织解决方案。 该表列出了每个解决方案对总体成绩的贡献、该解决方案中实现的分数，以及在该解决方案中进行了分组以增加成绩的剩余改进操作数。

可以通过以下两种方法从该屏幕执行操作：

1. 在预期解决方案的行上，在 " **剩余操作** " 列下，选择超链接编号。 你将看到 "改进操作" 屏幕的筛选视图，其中显示了该解决方案的未经测试的改进操作。

2. 在预期解决方案的行中的 " **打开解决方案** " 列下，选择 " **打开**"。 你将在 Microsoft 365 和 Office 365 安全与合规中心中看到解决方案或位置，你可以采取建议的措施。

## <a name="assessments-page"></a>评估页

"评估" 页面列出了您为您的组织设置的所有 [评估](compliance-manager-assessments.md) 。 您的合规性分数分母由所有跟踪的评估决定。 在添加更多评估时，您将看到 "改进操作" 页面上列出的更多改进操作，并且您的合规性分数分母将增加。

"评估" 页面汇总了有关每个评估的关键信息：

- **评估**：评估的名称
- **状态**：
    - **完成** -所有控件的状态均为 "已通过"，或者至少传递了一个，其余部分为 "超出范围"
    - **不完整** –至少一个控件的状态为 "failed"
    - **None** -尚未测试所有控件
    - **进度** 改善操作具有任何其他状态，包括 "正在进行中"、"部分贷方" 或 "未检测到"
- **评估进度**：完成工作所占的百分比（按成功测试的控件数衡量）
- **改进操作**：满足控件实现的已完成操作的数量
- **Microsoft 操作**：满足 microsoft 控件实现的已完成操作的数量
- **组**：评估所属的组的名称
- **产品**：关联的 Microsoft 365 服务
- **法规**：适用于评估的规章标准、策略或法律

### <a name="filtering-your-assessments-view"></a>筛选评估视图

筛选评估的视图：

1. 选择评估列表左上角的 " **筛选器** "。
2. 在 " **筛选器** " 浮出控件窗格中，检查所需的条件。
3. 选择 " **应用** " 按钮。 筛选窗格将关闭，您将看到筛选出的视图。

您还可以通过从评估列表上方的 **组** 下拉菜单中选择分组类型来修改视图，以查看按组、产品或法规进行的评估。

### <a name="default-assessment"></a>默认评估

默认情况下，您将在 "评估" 页上看到 " [数据保护基准](compliance-manager-assessments.md#data-protection-baseline-default-assessment) 评估"。 合规性管理器还提供了多个用于生成评估的预建 [模板](compliance-manager-templates-list.md) 。

## <a name="assessment-templates-page"></a>"评估模板" 页

模板是用于在合规性管理器中创建评估的框架。 "评估模板" 页将显示模板和主要详细信息的列表。 此列表包括合规性管理器提供的模板以及你的组织修改或创建的任何模板。 您可以应用筛选器，以查找基于证书、产品范围、国家、行业和创建的模板的模板。

从其行中选择一个模板，以显示其详细信息页，其中包含有关该模板的说明以及有关证书、范围和控件详细信息的详细信息。 在此页面上，您可以选择相应的按钮来创建评估、将模板数据导出到 Excel 或修改模板。

**了解详细信息：** 了解 [如何使用评估模板](compliance-manager-templates.md)。
