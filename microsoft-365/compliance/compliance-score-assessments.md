# <a name="customize-compliance-score-preview-with-assessments"></a>使用评估功能自定义合规性分数（预览）

**本文内容：** 了解如何通过设置已准备好使用**评估**来自定义合规性分数。 了解如何修改评估的**模板**并创建您自己的自定义评估以满足组织的需求。 本文还介绍了如何将评估组织为**组**、使用**控件**和导出评估**报告**。

## <a name="introduction-to-assessments"></a>评估简介

合规性分数可帮助您管理对您的组织适用的法规和认证的评估合规性。 评估是来自特定法规、标准或策略的控件组。 合规性分数通过提供可供使用的评估涵盖各种行业和区域的法规和认证，使您能够轻松地开始跟踪合规性。

每个评估都是通过模板创建的，它充当包含完成评估所需的控件和改进操作的框架。 为你的组织设置最相关的评估可帮助确保你实施可限制合规性风险的策略和操作过程。

您的所有评估都列在 "评估" 页上。 [了解](compliance-score-setup.md#assessments-page)有关如何筛选你的评估视图和解释状态状态的详细信息。

## <a name="data-protection-baseline-default-assessment"></a>数据保护基线默认评估

为入门，Microsoft 在合规性分数中提供了**默认**评估，其中包含 Microsoft 365 数据保护基准。 此基准是一组控件，包括用于数据保护和常规数据管理的关键法规和标准。 此基线主要从 NIST CSF （美国国家标准和技术协会 Cybersecurity Framework）和 ISO （国际标准化组织）以及 FedRAMP （联邦风险和授权管理计划）和 GDPR （欧盟的常规数据保护法规）中提取元素。

在配置任何其他评估之前，此评估功能用于计算首次遵守合规性得分时的初始分数。 合规性分数收集来自 Microsoft 365 解决方案的初始信号。 你将快速了解你的组织是如何相对于关键数据保护标准和管理法规执行的，并查看建议采取的改进措施。

由于每个组织都有特定的需求，因此合规性分数取决于您设置和管理您自己的评估以帮助尽可能地减少和缓解风险。

## <a name="assessment-creation-overview"></a>评估创建概述

您可以通过三种方法来设置评估：

1. 选择 "已准备好使用评估"。
2. 修改[评估的模板](compliance-score-templates.md)以满足您自己的需求。
3. 创建您自己的自定义评估。

用户必须拥有全局管理员、合规性管理员、合规性数据管理员或安全管理员角色才能创建或修改评估。 了解有关[角色和权限](compliance-score-setup.md#set-user-permissions-and-assign-roles)的详细信息。

> [!NOTE]
> 在符合性分数中创建或修改的任何评估也将在合规性管理器中反映出来。 了解有关[合规性分数和合规性管理器之间关系](compliance-score.md#relationship-to-compliance-manager)的详细信息。

### <a name="ready-to-use-assessments"></a>准备好使用评估

通过从符合性分数选择用于创建评估的[模板](compliance-score-templates.md)中进行选择，启动合规性旅程。 模板包含每个特定评估所需的控件和改进操作。 合规性分数的模板涵盖与行业、地区和常见数据保护标准（如 GDPR 和 ISO 27001）对应的法规和认证。

**若要设置评估**，请在[从向导开始构建评估](#create-an-assessment)时选择其中一个模板。

### <a name="modify-the-template-of-an-assessment"></a>修改评估的模板

您可以修改评估合规性分数模板以便更好地满足组织的需求。 例如，如果您通常需要遵守 HIPAA 但需要其他数据保护或安全控制，则可以采用 HIPAA 模板并添加自己的自定义字段，以创建新的评估，以根据您所需的方式监视进度。 在公共预览版中，需要转到合规性管理器来修改模板。

**若要修改评估的模板**，请按照以下说明操作：

1. 查看合规性分数中的可用模板列表，以确定要修改的[模板](compliance-score-templates.md)。
2. [若要使用扩展过程自定义模板，请参阅合规性管理器说明](working-with-compliance-manager.md#customize-a-template-through-the-extension-process)。
3. 创建模板并将其导入到合规性管理器之后，您就可以在合规性分数中创建新的评估。 按照使用 "[评估创建向导](#create-an-assessment)" 生成评估的过程进行操作。

> [!NOTE]
> 详细了解公共预览版中[合规性分数和合规性管理器之间的关系](compliance-score.md#relationship-to-compliance-manager)。

### <a name="create-your-own-custom-assessment"></a>创建自己的自定义评估

您可以从头开始创建自己的评估，以准确地跟踪您的组织所需的内容。 与上面所述的修改过程一样，创建您自己的评估需要您先在合规性管理器中创建自己的评估模板。

**若要创建自己的自定义评估**，请按照以下说明操作：

1. 阅读如何[在合规性管理器中创建自己的模板](working-with-compliance-manager.md#create-your-own-template-and-import-it-into-compliance-manager)。
2. 创建模板并将其导入到合规性管理器之后，您就可以在合规性分数中创建新的评估。 按照使用 "[评估创建向导](#create-an-assessment)" 生成评估的过程进行操作。

## <a name="understand-groups-before-creating-assessments"></a>在创建评估之前了解组

在创建或修改评估之前，请务必了解组的工作方式。 在创建评估时，需要在该过程中将其分配给一个组。 因此，我们建议您在创建评估之前规划评估的分组策略。

### <a name="what-are-groups"></a>什么是组

组是允许您组织评估的容器。 您可以按符合您的方式对评估进行分组，如按年或按规定进行分组，或者基于组织的部门或地理位置进行分组。 以下是两个组及其基础评估的示例：

- **FFIEC 是评估2020**
  - Office 365 + FFIEC 是
  - Intune + FFIEC 为
- **数据安全性和隐私评估**
  - Office 365 + ISO 27001:2013
  - Office 365 + ISO 27018:2014

当同一个组中的两个不同评估共享由您管理的改进操作时，您对操作的实现详细信息或状态所做的任何更新将自动同步到组中任何其他评估中的相同操作。 此同步使您能够实现一个改进操作，并在多个管理法规方面满足几个要求。

### <a name="how-to-create-a-group"></a>如何创建组

在[创建新评估](#create-an-assessment)的过程中，您需要创建一个组。

无法将组创建为独立实体;一个组必须至少包含一个评估。 若要创建组，必须首先创建要放入组中的评估。

### <a name="what-to-know-when-working-with-groups"></a>使用组时需要了解的内容

- 组名称在您的组织中必须是唯一的。
- 组没有安全属性。 所有权限都与评估相关联。
- 将评估添加到组后，不能更改分组。
- 在同一组中的不同评估中的相关评估控件在完成后将自动更新。
- 如果将新评估添加到现有组中，则会将该组中评估的常见信息复制到新评估中。
- 组可以包含对同一认证或法规的评估，但每个组只能包含针对特定产品认证对的一个评估。 例如，组不能包含针对 Office 365 和 NIST CSF 的两个评估。 仅当一个组与同一个产品的对应证书或法规不同时，该组才可以包含对同一产品的多个评估。
- 删除评估会破坏该评估与组之间的关系。
- 无法删除组。
- 当对多个组中出现的改进进行更改时，该更改将反映在该改进操作的所有实例中。

## <a name="create-an-assessment"></a>创建评估

若要在合规性分数中创建评估，请执行以下步骤：

1. 从评估页中，选择 "**添加评估**"。 在大型浮出控件窗格中将显示一个评估向导。

2. **选择一个模板：** 选择要用作评估基础的模板。 您可以选择 "已准备好使用" 模板，也可以选择已修改或创建的模板。 选择您选择的模板旁边的单选按钮，然后选择 "**下一步**"。

> [!NOTE]
> 有关[创建和修改模板的说明](working-with-compliance-manager.md#templates)，请参阅合规性管理器中处理的过程。

3. **名称和组：** 在 "**评估名称**" 字段中输入评估的名称。 评估名称在组中必须是唯一的。 如果评估的名称与任何给定组中的另一个评估的名称相匹配，则会收到一条错误，要求您创建一个不同的名称。

4. 将评估服务分配给一个组。 您可以：
    - 选择 "**使用现有组**将其分配给已创建的组";和
    - 选择 "**创建新组**" 以创建新组，并将此评估分配给它。 确定组的名称，并将其输入到单选按钮下的字段中。

5. **审阅并完成：** 向导的最后一个屏幕显示为评估选择的模板、名称和组。 您可以从屏幕上的链接中编辑这些设置中的任何设置，这将返回到向导中的相关步骤。 一旦您对设置感到满意，请选择 "**创建评估**"。

6. 下一个屏幕确认您已成功创建新的评估。 选择 "**完成**" 以关闭向导，新评估的 "详细信息" 页将显示在屏幕上。

如果在选择 "**创建评估**" 后看到 "**评估失败**" 屏幕，请选择 "**重试**" 以重新创建评估。

## <a name="delete-an-assessment"></a>删除评估

删除评估会将其从评估页面上的列表中删除。 **删除评估是永久性的;您无法再取回它。** 如果想要再次进行相同的评估，则必须从头开始重新创建。 如果评估中的改进操作未在任何其他评估中显示，则在删除评估时将删除这些操作。 建议您先导出评估报告，然后再将其永久删除。

若要删除评估，请执行以下步骤：

1. 从评估页中，选择要删除的评估以打开该评估的 "详细信息" 页。
2. 选择屏幕右上角的 "**删除评估**"。
3. 将显示一个窗口，要求您确认是否要永久删除该评估。 选择 "**删除评估**" 以关闭该窗口。 你将收到一个确认窗口，指出评估已从合规性分数中删除。

如果您删除组中的唯一评估，则该组也将从合规性分数中删除。

## <a name="monitor-assessment-progress-and-controls"></a>监视评估进度和控制

每个评估都有一个详细信息页面，可在完成评估的过程中提供一览的一览视图。 页面显示您在完成控件中的进度，以及这些控件中的关键改进操作的测试状态。

### <a name="overview-tab"></a>概述选项卡

"概述" 选项卡包含一个显示评估结束百分比的图形。 此图包含来自你所拥有的操作的点细目，以及 Microsoft 所拥有的操作的点，因此你可以看到完成此评估所需的更多分数。

评估中的控件的主要改进操作是按对赢得积分的最大潜在影响顺序列出的。 关联的关系图详细说明了您的改进操作的聚合测试状态，以便您可以快速估量已测试的内容以及仍需要完成的操作。

若要访问各个改进操作，请转到 "控件" 选项卡。

### <a name="controls-tab"></a>控件选项卡

"控件" 选项卡显示映射到评估的每个控件的详细信息。 **控件状态细分**图按系列显示控件的状态，因此您可以一目了然地查看控件的哪些组需要关注。

在图表下，表中列出了评估中每个控件的详细信息。 控件按控件系列分组。 展开每个系列名称以显示它所包含的各个控件。 为每个控件列出的信息包括：

- **控件标题**
- **状态**：反映控件中的改进操作的测试状态 
    - 已**通过**-所有改进操作的测试状态均为 "已通过"，或者至少有一个已传递且其余值为 "超出范围"
    -  **失败**-至少有一个改进操作的测试状态为 "Failed"
    - **None** -尚未测试所有改进操作
    - **超出范围**-此评估的所有改进操作均超出范围
    - **进度**改善操作的状态不是上面列出的状态，可能包括 "正在进行中"、"部分信贷" 或 "未检测"。
- **控件 ID**：控件的标识号，由其对应的法规、标准或策略分配
- **实现分数**：完成操作所得到的分数，即实现的总积分数 
- **您的操作**：要完成的操作的总数已完成的操作数
- **Microsoft 操作**： microsoft 完成的操作数 

若要查看控件的详细信息，请从表的行中选择它。 "控件详细信息" 页将显示一个图，指示该控件中的操作的测试状态。 图下方的表格显示了该控件的关键改进操作。

从列表中选择 "改进操作"，以深入了解改进操作的详细信息页面。 详细信息页面显示了测试状态和实现说明，并启动了推荐的解决方案。

## <a name="export-an-assessment-report"></a>导出评估报告

您可以按照[以下说明](working-with-compliance-manager.md#reports)将评估导出到您组织中的合规性利益干系人或外部审计员和管理机构的 Excel 文件。 你需要访问合规性管理器以导出报告。

报告是在导出的日期和时间进行评估的快照。 它包含由您和 Microsoft 管理的控件的详细信息，其中包括实施状态、测试日期、测试结果以及指向已上载证据文档的链接。
