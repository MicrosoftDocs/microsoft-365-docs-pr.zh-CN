# <a name="data-protection-impact-assessments"></a>数据保护影响评估

根据一般数据保护条例 (GDPR)，需要由数据控制者准备数据保护影响评估 (DPIA)，用于处理“可能对自然人的权力和自由产生高风险”的操作。Microsoft Azure 或 Dynamics 365 本身并不一定要求由使用它的数据控制者创建 DPIA。更准确地说，是否需要 DPIA 取决于数据控制者部署、配置每个产品的*方式*和背景。

此文档的目的是为数据控制者提供相关信息，帮助他们确定是否需要 DPIA 以及要包含的详细信息。

- [Office 365](gdpr-dpia-office365.md)
- [Azure](gdpr-dpia-azure.md)
- [Dynamics 365](gdpr-dpia-dynamics.md)
- [Microsoft 支持和专业服务](gdpr-dpia-prof-services.md)

有关 GDPR 合规性 Microsoft 数据保护的详细信息，请参阅***服务信任门户***中的[合规性](https://servicetrust.microsoft.com/ComplianceManager)。
