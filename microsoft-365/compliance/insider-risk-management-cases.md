# <a name="insider-risk-management-cases"></a>内部风险管理案例

案例是内部风险管理的核心，可让你深入调查和处理由策略中定义的风险指标生成的问题。 当需要执行进一步操作以解决用户的合规性相关问题时，从警报手动创建案例。 每个案例的范围都为一个用户，并且可以将该用户的多个警报添加到现有案例或新案例。 

调查案例的详细信息后，您可以采取以下措施：

- 向用户发送通知
- 以恶意方式解决大小写
- 与 ServiceNow 实例或电子邮件收件人共享案例
- 升级高级电子数据展示调查案例

## <a name="cases-dashboard"></a>事例仪表板

内部风险管理案例 **仪表板** 允许你查看案例并采取行动。 仪表板上的每个报表小部件显示最近 30 天的信息。

- **活动案例**：正在调查的活动案件总数。
- **过去 30 天内的** 事例：创建的事例总数，按"活动"和"已关闭 *"**状态* 排序。
- **统计信息**：活动案例的平均时间，以小时、天或月列出。

事例队列列出了组织的所有活动案例和已关闭案例，以及以下事例属性的当前状态：

- **案例** 名称：案例名称，在确认警报并创建案例时定义。  
- **状态**：案例的状态 *，活动或**关闭*。
- **用户**：案例的用户。 如果启用用户名匿名化，则显示匿名信息。
- **打开案例的时间**：自案例打开以来经过的时间。
- **策略警报总数**：案例中包含的策略匹配数。 如果向案例添加了新警报，此数量可能会增加。
- **Last updated**： The time that has been passed since there has an added case note or change in the case state.
- **Last updated by**： The name of the insider risk management analyst or investigator that last updated the case.

![内部风险管理案例仪表板](../media/insider-risk-cases-dashboard.png)

使用 **搜索** 控件搜索特定文本的事例名称，并使用事例筛选器按以下属性对事例进行排序：

- 状态
- 打开时间案例、开始日期和结束日期
- 上次更新时间、开始日期和结束日期

## <a name="filter-cases"></a>筛选事例

根据组织中活动的内部风险管理策略的数量和类型，查看大型事例队列可能很具有挑战性。 使用事例筛选器可帮助分析员和调查人员按多个属性对事例进行排序。 若要筛选"案例"仪表板上的 **警报**，请选择 **"筛选器"** 控件。 可以按一个或多个属性筛选事例：

- **状态**：选择一个或多个状态值以筛选案例列表。 选项为 *"活动"和*"*关闭"。*
- **打开案例的时间**：选择打开案例的开始日期和结束日期。
- **上次更新** 时间：选择更新案例的开始日期和结束日期。

## <a name="investigate-a-case"></a>调查案例

深入调查内部风险管理警报对于采取正确的纠正措施至关重要。 内部风险管理案例是深入了解用户风险活动历史记录和警报详细信息，以及探索暴露给风险的内容和消息的中心管理工具。 风险分析师和调查人员还使用案例来集中审阅反馈和备注并处理案例解决。

选择案例将打开事例管理工具，并允许分析员和调查人员深入了解事例的详细信息。

### <a name="case-overview"></a>案例概述

" **案例概述** "选项卡汇总了该案例的警报活动和风险级别历史记录。 

- " **警报"** 小部件显示案例的策略匹配项，包括警报状态、警报风险严重性和检测到警报时。 
- 风险 **级别历史记录** 图表显示过去 30 天内的用户风险级别。 通过该图表，分析师和调查人员可以快速查看一段时间的总体用户风险趋势。 
- " **风险活动内容** "小部件汇总了添加到案例的警报中包含的数据和内容类型。 此小组件提供了在这种情况下处于风险中的整个数据和内容集的全部视图。

所有 **案例管理** 选项卡上均提供了"案例详细信息"窗格，并汇总了风险分析师和调查人员的案例详细信息。 它包括以下方面：

- **案例名称**：案例名称，前缀为自动生成的案例序列号，以及第一个确认警报匹配的策略模板所关联的风险名称。 
- **案例状态**：案例的当前状态 *，活动或**关闭*。
- **用户的风险评分**：当前计算案例的用户风险级别。 此分数每 24 小时计算一次，并使用与用户关联的所有活动警报中的警报风险评分。
- **已确认警报**：已确认该案例的用户的警报列表。
- **相关内容**：按内容源和类型排序的内容列表。 例如，对于 SharePoint Online 中的案例警报内容，你可能会看到列出的文件夹或文件名与本例中警报的风险活动相关联。

![内部风险管理案例详细信息](../media/insider-risk-case-details.png)

### <a name="alerts"></a>警报

" **警报"** 选项卡汇总了案例中包含的当前警报。 新警报可能会添加到现有案例，并且将在分配时添加到警报队列中。  队列列出了以下警报属性：

- 状态
- Severity
- 检测到的时间

从队列中选择警报以显示"警报 **详细信息"** 页。

使用搜索控件搜索特定文本的警报名称，并使用警报筛选器按以下属性对事例进行排序：

- 状态
- Severity
- 检测到的时间、开始日期和结束日期

使用筛选器控件按多个属性筛选警报，包括：

- **状态**：选择一个或多个状态值以筛选警报列表。 选项为 *"已确认**"、"已消除**"、"需要审阅*"和"已 *解决"。*
- **严重性：** 选择一个或多个警报风险严重性级别以筛选警报列表。 选项为 *"高"、"**中"* 和"*低"。*
- **检测到的时间**：选择创建警报的开始日期和结束日期。
- **策略**：选择一个或多个策略以筛选所选策略生成的警报。

### <a name="user-activity"></a>用户活动

" **用户活动** "选项卡是内部风险管理解决方案中用于内部风险分析和调查案例的最强大工具之一。 此选项卡的结构用于快速查看案例，包括所有警报的历史时间线、所有警报详细信息、用户当前的风险评分以及采取有效措施以包含案例风险的控制。

![内部风险管理用户活动](../media/insider-risk-user-activities.png)

1. **日期和时间时间筛选器**：默认情况下，在用户活动图表中显示已确认的过去六个月的警报。 通过在图表窗口的两端使用滑块控件，或在图表筛选器控件中定义特定的开始日期和结束日期，可以轻松筛选图表视图。
2. **风险警报活动和详细信息**：风险活动在用户活动图表中以彩色气泡显示。 为不同类别的风险创建气泡，气泡大小与类别的风险活动数成比例。 选择气泡以显示每个风险活动的详细信息。 详细信息包括：
    - **风险** 活动的日期。
    - 风险 **活动类别**。 例如，电子邮件 *(* 附件) 组织外部发送的附件或 (文件) *从 SharePoint Online 下载*。
    - **警报** 的风险评分。 此分数是警报风险严重性级别的数字分数。
    - 与警报关联的事件数。 还可以链接到与风险活动关联的每个文件或电子邮件。
3. **风险活动** 图例：在用户活动图表底部，颜色编码图例可帮助您快速确定每个警报的风险类别。
4. **风险活动时间：** 列出与案例关联的所有风险警报的完整时间，包括相应警报气泡中提供的所有详细信息。
5. **案例操作**：用于解决大小写的选项位于案例操作工具栏上。 可以解决案例、向用户发送电子邮件通知，或升级案例以用于数据或用户调查。

### <a name="case-notes"></a>案例注释

在 **案例的** "案例备注"选项卡中，风险分析员和调查人员可以共享有关他们针对该案例的工作的评论、反馈和见解。 备注是案例的永久添加内容，在保存便笺后无法编辑或删除。 从警报创建案例时，在"确认警报和创建内部风险案例"对话框中输入的注释将自动添加为案例注释。

案例备注仪表板显示创建便笺的用户的备注以及自保存便笺以来经过的时间。 若要搜索特定关键字的"案例注释"文本字段，请使用案例仪表板上的"搜索"按钮并输入特定关键字。

向案例添加注释：

1. 在 [Microsoft 365 合规中心](https://compliance.microsoft.com)，转到 **"内部风险管理** "并选择"案例 **"** 选项卡。
2. 选择一个案例，然后选择" **案例备注"** 选项卡。
3. 选择 **"添加大小写注释"。**
4. 在 **"添加大小写注释** "对话框中，键入该案例的注释。 Select **Save** to add the note to the case or select **Cancel** close without saving the note to the case.

### <a name="contributors"></a>参与者

在 **案例** 的"参与者"选项卡中，风险分析员和调查人员可以将其他审阅者添加到案例。 默认情况下，分配有 Insider **Risk Management Analysts** 和 **Insider Risk Management Investigators** 角色的所有用户将列为每个活动案例和已关闭案例的参与者。

所有内部风险管理案例都必须使用适当的访问控制进行管理，以保持调查的机密性和完整性。 为了帮助维护对事例的访问控制，为用户分配了以下两种类型的事例访问权限之一：

- **永久访问权限**：从警报创建案例时，自动向具有 **内部** 风险管理分析师和内部风险管理调查人员角色的用户授予永久访问权限。 永久访问在案例生存期内授予对案例的完全控制权限，并授予添加其他案例参与者的能力。
- **临时访问**：临时访问仅由具有案例永久访问权限的参与者授予用户。 通常，此访问级别授予需要向案例添加注释的用户。 具有临时访问权限的参与者拥有除：
    - 确认或消除警报的权限
    - 对事例的参与者进行编辑的权限
    - 在内容资源管理器中查看文件和消息的权限

向案例添加参与者：

1. 在 [Microsoft 365 合规中心](https://compliance.microsoft.com)，转到 **"内部风险管理** "并选择"案例 **"** 选项卡。
2. 选择案例，然后选择"参与者 **"** 选项卡。
3. 选择 **"添加参与者"。**
4. 在 **"添加参与者** "对话框中，开始键入要添加的用户的名称，然后从建议的用户列表中选择该用户。 此列表从租户订阅的 Azure Active Directory 生成。
5. 在 **"添加参与者"** 对话框中，选择参与者的访问级别。 可以选择"**永久"或**"**临时"。**
6. 选择 **"** 添加"将用户添加为参与者，或选择" **取消关闭对话框** "，而不将用户添加为参与者。

## <a name="case-actions"></a>案例操作

风险分析师和调查人员可以使用多种方法之一对案例采取行动，具体取决于案例的严重性、用户的风险历史记录和组织的风险准则。 在某些情况下，你可能需要将案例上报给用户或数据调查，以与组织的其他活动进行协作，并深入了解风险活动。 内部风险管理与其他 Microsoft 365 合规性解决方案紧密集成，可帮助你进行端到端解决方案管理。

### <a name="send-email-notice"></a>发送电子邮件通知

在大多数情况下，创建内部风险警报的用户操作是无意或无意的。 通过电子邮件向用户发送提醒通知是记录案例审阅和操作的有效方法，也是提醒用户了解公司策略或让他们刷新培训的方法。 通知是通过为内部 [风险管理基础结构创建的通知](insider-risk-management-notices.md) 模板生成的。

必须记住，向 *_用户发送电子邮件通知_** 不会将情况解析为 _Closed*。 在某些情况下，在向用户发送通知以查找更多风险活动而不打开新案例后，您可能需要保持案例打开状态。 如果要在发送通知后解决案例，则必须在发送通知后选择"解决案例"作为后续步骤。

若要向分配给案例的用户发送通知：

1. 在 [Microsoft 365 合规中心](https://compliance.microsoft.com)，转到 **"内部风险管理** "并选择"案例 **"** 选项卡。
2. 选择一个案例，然后选择案例 **操作** 工具栏上的"发送电子邮件通知"按钮。
3. 在 **"发送电子邮件通知"对话框中** ，选择"选择通知模板 **"下拉列表控件** 以选择通知的通知模板。 此选择会预填充通知上的其他字段。
4. 查看通知字段并在适当时进行更新。 此处输入的值将替代模板上的值。
5. 选择 **"** 发送"将通知发送给用户，或选择" **取消关闭对话框** "，而不向用户发送通知。 所有发送的通知将添加到"案例笔记"仪表板上的"案例注释" **队列中** 。

### <a name="escalate-for-investigation"></a>上报以进行调查

在需要针对用户的风险活动进行其他法律审查的情况下，上报案例以用于用户调查。 此升级将在 Microsoft 365 组织中打开一个新的高级电子数据展示案例。 高级电子数据展示提供了端到端工作流，用于保留、收集、审阅、分析和导出对组织内部和外部法律调查做出响应的内容。 它还可让法律团队管理整个法定保留通知工作流，以与案例所涉及的保管人进行通信。 将审阅者分配为从内部风险管理案例创建的高级电子数据展示案例中的保管人，可帮助法律团队采取适当的措施和管理内容保留。 若要详细了解高级电子数据展示事例，请参阅 Microsoft [365](overview-ediscovery-20.md)中的高级电子数据展示概述。

将案例上报给用户调查：

1. 在 [Microsoft 365 合规中心](https://compliance.microsoft.com)，转到 **"内部风险管理** "并选择"案例 **"** 选项卡。
2. 选择一个案例，然后选择案例操作工具栏上的"上报调查"按钮。
3. 在" **上报以进行调查** "对话框中，输入新用户调查的名称。 如果需要，请输入有关案例的注释，然后选择"上报 **"。**
4. 查看通知字段并在适当时进行更新。 此处输入的值将替代模板上的值。
5. Select **Confirm** to create the user investigation case or select **Cancel** to close the dialog without creating a new user investigation case.

内部风险管理案例升级到新用户调查案例后，可以在  >  Microsoft 365 合规中心电子数据展示高级区域中查看新案例。

### <a name="run-automated-tasks-with-power-automate-flows-for-the-case"></a>使用 Power Automate 流程针对这种情况运行自动任务

使用建议的 Power Automate 流，风险分析员和分析人员可以快速采取措施：

- 从 HR 或业务请求有关内部风险案例中用户的信息
- 当用户有内部风险警报时通知经理
- 添加日历提醒以跟进内部风险案例
- 在 ServiceNow 中为内部风险管理案例创建记录

若要为内部风险管理案例运行、管理或创建 Power Automate 流：

1. 在 **案例** 操作工具栏上选择"自动执行"。 
2. 选择要运行的 Power Automate 流，然后选择"**运行流"。** 
3. 完成流后，选择"完成 **"。**

若要了解有关用于内部风险管理的 Power Automate 流的信息，请参阅 [内部风险管理设置入门](insider-risk-management-settings.md#power-automate-flows-preview)。

### <a name="view-or-create-a-microsoft-teams-team-for-the-case"></a>查看或创建适用于该案例的 Microsoft Teams 团队

在设置中启用用于内部风险管理的 Microsoft Teams 集成后，每次确认警报并创建案例时，都会自动创建 Microsoft Teams 团队。 风险分析员和分析师可以快速打开 Microsoft Teams，并在案例操作工具栏上选择"查看 **Microsoft Teams** 团队"，直接导航到团队了解案例。

对于启用 Microsoft Team 集成之前打开的事例，风险分析员和分析师可以通过在事例操作工具栏上选择"创建 **Microsoft Teams** 团队"，为事例创建新的 Microsoft Teams 团队。

当案例解决时，关联的 Microsoft 团队将自动存档 (隐藏，并打开为只读) 。

若要了解有关用于内部风险管理的 Microsoft Teams 的信息，请参阅内部 [风险管理设置入门](insider-risk-management-settings.md#microsoft-teams-preview)。

### <a name="share-the-case"></a>共享案例

通过共享内部风险管理案例，风险分析师和分析师可以轻松地与组织的其他合规性利益干系人进行协作。 您可以快速与来自案例管理领域的外部利益干系人共享内部风险管理案例的链接。 若要从链接访问内部风险管理案例，利益干系人必须包括在任何内部风险管理角色组中。

>[!NOTE]
>感谢你在 ServiceNow 连接器预览期间提供的反馈和支持。 我们决定在 2020 年 11 月 30 日终止 ServiceNow 连接器的预览版并停止对内部风险管理的支持。 我们正在积极评估替代方法，以为客户提供内部风险管理中的 ServiceNow 集成。

以下共享选项可用：

- **ServiceNow：** 为 Microsoft 365 组织配置 Microsoft 365 ServiceNow 连接器后，可以轻松地与 ServiceNow 组织共享指向该案例的链接、打开事件或请求更改。 若要与 ServiceNow 共享案例，请从案例操作中选择 **"共享**  >  **服务Now"。** ServiceNow 与内部风险管理支持集成包括以下案例信息和操作：
    - **任务名称**：新 ServiceNow 任务的名称。
    - **任务说明**：新 ServiceNow 任务的说明。 此可编辑描述字段自动包含指向内部风险管理案例的链接。
    - **任务类型**： 新 ServiceNow 任务的任务 *类型，事件* 或更改 *请求*。
    - **优先级**：新 ServiceNow 任务的优先级，包括 *规划*、*低*、中等 *、高* 或 *关键*。
    - **截止日期**：完成 ServiceNow 任务的请求日期。

![与 ServiceNow 共享内部风险管理](../media/insider-risk-share-servicenow.png)

- **电子邮件**：共享电子邮件中内部风险管理案例的链接。 可以使用此共享选项选择任何本地配置的电子邮件客户端。 若要与电子邮件共享案例链接 **，请从** 案例  >  操作工具栏中选择"共享电子邮件"。
- **复制链接**：将指向内部风险管理案例的链接复制到剪贴板。 若要将案例链接复制到剪贴板 **，请从** 案例操作工具栏中选择  >  "共享副本"链接。

### <a name="resolve-the-case"></a>解决大小写

风险分析师和调查人员完成审阅和调查后，可以解决案例，以对当前包含在该案例内的所有警报采取行动。 解决案例会添加一个解决方案分类，将案例状态更改为"已关闭"，并且解决方案操作原因会自动添加到"案例笔记"仪表板上的案例 **注释队列中。** 案例解析为：

- **恶意**：策略匹配警报被评估为低风险、非严重或误报的情况的分类。
- **已确认** 的策略违反：策略匹配警报被评估为有风险、严重或恶意意图的结果的分类。

要解决案例，需要：

1. 在 [Microsoft 365 合规中心](https://compliance.microsoft.com)，转到 **"内部风险管理** "并选择"案例 **"** 选项卡。
2. 选择一个案例，然后选择案例操作工具栏上的"解决案例"按钮。
3. 在" **解决案例** "对话框中 **，选择"** 解析为"下拉列表控件以选择案例的解析分类。 选项为 **"安全"或****"已确认的策略违反"。**
4. 在" **解决大小写"** 对话框中，在"已执行操作"文本字段中输入 **解析分类的原因** 。
5. 选择 **"** 解析"以关闭案例，或选择" **取消** 关闭对话框"而不解析大小写。
