# <a name="compliance-offering-web-content-accessibility-guidelines-21"></a>合规性产品：Web 内容辅助功能准则 2.1

## <a name="about-wcag-21"></a>关于 WCAG 2.1

WCAG 2.1 提供了一个 Web 内功开发框架，除了帮助使用图形化能力有限的设备的用户，还让残障人士更易访问这些内容。 WCAG 2.0 是万维网联盟 (W3C)（一家致力于制定 Web 标准的国际组织）于 2008 年发布的，在 2018 年 6 月更新为 WCAG 2.1 版。 2012 年，国际标准化组织 (ISO) 也以 ISO/IEC 40500:2012 的形式发布了 WCAG 2.0。  
  
符合 WCAG 2.1 的内容也符合 WCAG 2.0。 对于需遵从 WCAG 2.0 的策略，WCAG 2.1 可提供替代合规方法。  
  
每个准则的遵从要求都按 A、AA 和 AAA 这三个级别进行衡量。 Microsoft 是世界各地国家/地区和政府的主要软件及云服务提供商，因此它承诺遵守各项相关[国际标准和合规控制措施](https://go.microsoft.com/fwlink/p/?linkid=2052226)。 通过坚持这些广泛的无障碍标准，Microsoft 保护政府内外各方客户都可使用 Microsoft 服务和产品。  

## <a name="microsoft-and-wcag-21"></a>Microsoft 与 WCAG 2.1

Microsoft 遵守 WCAG 2.1 (ISO/IEC 40500) 标准表明了它承诺让所有客户都更易访问技术和数据。 WCAG 2.1 (ISO/IEC 40500) 是一项国际无障碍要求，对欧洲的 EN 301 549 和美国的第 508 条规定进行了补充。  
  
Microsoft 发布了 WCAG 2.1 报告，其中反映的是完整的产品或服务。 它通常不对单个功能或组件创建报告。 有时，Microsoft 可能会对现有产品发布新的组件，或者发布现有组件的新版本 - 用户可选择单独安装它们，此时 Microsoft 可能也会对该组件发布 WCAG 2.1 报告。  
  
[下载 WCAG 2.1 (ISO/IEC 40500) 辅助功能标准](https://go.microsoft.com/fwlink/p/?linkid=2052226)
