# <a name="client-and-server-software-roadmap-for-microsoft-365"></a>Microsoft 365 的客户端和服务器软件路线图

大多数企业组织都有一个包含多个版本的操作系统、客户端软件和服务器软件的异构环境。 Microsoft 365 企业版包括您的 IT 基础结构的关键组件的最安全版本。 它还包括旨在利用云技术的生产力功能。

要最大限度提高集成了产品套件的 Microsoft 365 企业版的业务价值，请开始规划并实施迁移以下版本的策略：

- 在您的计算机上安装到适用于企业的 Microsoft 365 应用程序的 Office 客户端。
- 在 Microsoft 365 中安装在服务器上的 Office 服务器到其等效的服务。
- 在你的设备上的 windows 7 和 Windows 8.1 到 Windows 10 企业版。

>[!Note]
>对 Windows 7 的支持已于 *2020 年1月 14* 日结束。 有关详细信息，请参阅 [支持终止的详细信息](https://support.microsoft.com/help/4057281/windows-7-support-will-end-on-january-14-2020)。
>

随着时间的推移完成这些迁移，您的组织会更接近 [新式工作场所](https://www.microsoft.com/microsoft-365/blog/2018/04/27/making-it-simpler-with-a-modern-workplace/)的愿景。 此安全和集成的环境可帮助您在组织中解锁团队合作和创造力。 Microsoft 365 企业版启用并为你提供所有功能。

## <a name="migration-for-office-client-products"></a>Office 客户端产品的迁移

大型和小型组织通常使用 Office 客户端产品（如 Word、Excel 和 PowerPoint）的早期版本的组合。 这些早期版本：

- 可以使用最新的安全更新和支持修补程序进行 [更新](https://support.office.com/article/install-office-updates-2ab296f3-7f03-43a2-8e50-46de917611c5) 。 但此过程有时是手动进行的，并且可能不会在整个组织中进行扩展。
- 不会进行优化以使用 Microsoft 云技术来帮助您以数字方式转换业务。
- 请勿提供最新功能。

Microsoft 365 企业版包含适用于企业的 Microsoft 365 应用程序。 此版本的 Office 客户端产品适用于 Microsoft 365 企业版许可证。 它已从 Microsoft 云中安装和更新。 Microsoft 365 企业应用版包括安全更新和最新功能。 有关详细信息，请参阅 [关于 enterprise For Microsoft 365 Apps](https://docs.microsoft.com/deployoffice/about-microsoft-365-apps)。

### <a name="office-2007"></a>Office 2007

对于 Office 2007 版本中的 Office 版本，已通过支持的结尾。 有关详细信息，请参阅 [Office 2007 终止支持路线图](https://docs.microsoft.com/deployoffice/office-2007-end-support-roadmap)。

请考虑采取以下步骤，而不是将运行 Office 2007 的计算机升级到 Office 2010、Office 2013 或 Office 2016：

1. 为您的用户获取并分配 Microsoft 365 许可证。
2. 在其计算机上卸载 Office 2007。
3. 为企业单独安装或在 IT 部署过程中安装 Microsoft 365 应用程序。 有关详细信息，请参阅 [适用于 Microsoft 365 应用的部署指南](https://docs.microsoft.com/deployoffice/deployment-guide-microsoft-365-apps)。

适用于企业的 Microsoft 365 应用程序将自动安装更新。 它可以利用基于云的服务实现增强的安全性和生产率。

### <a name="office-2010"></a>Office 2010

对于 Office 2010 版本中的 Office 版本，支持于 *10 月13日（2020）* 结束。 有关详细信息，请参阅 [Office 2010 终止支持路线图](https://docs.microsoft.com/deployoffice/office-2010-end-support-roadmap)。

您可以考虑将运行 Office 2010 的计算机升级到 Office 2013 或 Office 2016。 但是，这两个版本必须手动更新。 因此，请考虑改用以下步骤：

1. 为您的用户获取并分配 Microsoft 365 许可证。
2. 在其计算机上卸载 Office 2010。
3. 为企业单独安装或在 IT 部署过程中安装 Microsoft 365 应用程序。 有关详细信息，请参阅 [适用于 Microsoft 365 应用的部署指南](https://docs.microsoft.com/deployoffice/deployment-guide-microsoft-365-apps)。

适用于企业的 Microsoft 365 应用程序会自动安装安全更新和新功能更新。 它可以利用 Microsoft 365 中基于云的服务，提高安全性和提高工作效率。

### <a name="office-2013-and-office-2016"></a>Office 2013 和 Office 2016

尚未确定 Office 2013 和 Office 2016 的支持结束路线图。 在这些版本（如 Office 2010）中，您仍必须 [安装安全更新](https://support.office.com/article/install-office-updates-2ab296f3-7f03-43a2-8e50-46de917611c5)。 此任务可能不会很好地扩展，具体取决于组织的规模。

不是让计算机保持最新的 Office 2013 或 Office 2016 安全更新，也不是将计算机从 Office 2013 更新到 Office 2016，请考虑采取以下步骤：

1. 为您的用户获取并分配 Microsoft 365 许可证。
2. 在其计算机上卸载 Office 2013 或 Office 2016。
3. 为企业单独安装或在 IT 部署过程中安装 Microsoft 365 应用程序。 有关详细信息，请参阅 [适用于 Microsoft 365 应用的部署指南](https://docs.microsoft.com/deployoffice/deployment-guide-microsoft-365-apps)。

适用于企业的 Microsoft 365 应用程序会自动安装安全更新和新功能更新。 它可以利用 Microsoft 365 中基于云的服务，提高安全性和提高工作效率。

## <a name="migration-for-office-server-products"></a>Office server 产品迁移

大型和小型组织通常使用 Office server 产品的较旧版本的组合，例如 Exchange Server 和 SharePoint Server。 这些早期版本：

- 应使用最新的安全更新和支持修补程序进行更新。 在某些情况下，这些更新按月发布。
- 不会进行优化以使用 Microsoft 云技术来帮助您以数字方式转换业务。
- 不要包括新的工作效率应用程序，如 Microsoft 团队。
- 请勿包括最新的安全功能，例如 Office 365 的 Exchange 和 Defender。

Microsoft 365 企业版包括基于云的 Office server 服务版本，这些版本使用一些工具作为本地版本的 Office server 软件，如 web 浏览器和 Outlook 客户端。 系统会自动更新这些服务的安全性。 因此，你的 IT 人员节省了维护和更新本地服务器所需的时间。 这些服务还提供了 Office server 软件中不存在的新功能增强功能。

使用以下资源可获取有关针对特定 Microsoft 365 工作负荷迁移用户和数据的信息：

- [将邮箱从内部部署 Exchange Server 移动到 Exchange Online](https://docs.microsoft.com/exchange/hybrid-deployment/move-mailboxes)
- [将 SharePoint 数据从 SharePoint Server 迁移到 SharePoint Online](https://docs.microsoft.com/sharepointmigration/migrate-to-sharepoint-online)
- [将 Skype for Business Online 迁移到 Microsoft 团队](https://docs.microsoft.com/microsoftteams/migration-interop-guidance-for-teams-with-skype)

### <a name="office-2007-server-products"></a>Office 2007 服务器产品

对于 Office 2007 版本中的服务器产品，已通过支持终止。 有关详细信息，请参阅以下文章：

- [Exchange 2007 支持结束路线图](exchange-2007-end-of-support.md)
- [SharePoint Server 2007 支持终止路线图](sharepoint-2007-end-of-support.md)
- [Project Server 2007 终止支持路线图](project-server-2007-end-of-support.md)
- [Office 通信服务器支持结束路线图](https://docs.microsoft.com/skypeforbusiness/plan-your-deployment/upgrade)
- [PerformancePoint Server 2007 支持结束路线图](pps-2007-end-of-support.md)

与 office 2010、Office 2013 或 Office 2016 的版本中的服务器产品升级 Office 2007 版本中的服务器产品不同，请考虑采取以下步骤：

1. 将您的 Office 2007 服务器上的数据迁移到 Microsoft 365。 若要了解详细信息或帮助，请雇用 Microsoft 合作伙伴。
2. 向您的用户展示新的功能和工作进程。
3. 如果不再需要运行 Office 2007 服务器产品的本地服务器，请将其停用。

### <a name="office-2010-server-products"></a>Office 2010 服务器产品

对 [Exchange Server 2010](exchange-2010-end-of-support.md) 的支持于 *2020 年10月13日* 结束。

对 [SharePoint Server 2010](upgrade-from-sharepoint-2010.md) 的支持将于 *2021 年 4 月 13 日* 结束。

与 Office 2013 或 Office 2016 版本中的服务器产品相比，不是在 Office 2010 版本中升级这些服务器产品，请考虑采取以下步骤：

1. 将您的 Office 2010 服务器上的数据迁移到 Microsoft 365。 有关详细信息，请参阅 [FastTrack For microsoft 365](https://fasttrack.microsoft.com/microsoft365) 或雇用 microsoft 合作伙伴。

对于 Office 2013 版本中的服务器产品，尚未确定支持的结束。 不要在 Office 2016 版本中使用服务器产品升级 Office 2013 版本中的服务器产品，请考虑采取以下步骤：

1. 将您的 Office 2013 服务器上的数据迁移到 Microsoft 365。 有关详细信息，请参阅 [FastTrack For microsoft 365](https://fasttrack.microsoft.com/microsoft365) 或雇用 microsoft 合作伙伴。
