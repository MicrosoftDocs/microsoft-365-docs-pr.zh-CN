# <a name="how-to-manage-microsoft-365-with-windows-powershell-for-delegated-access-permissions-partners"></a>如何使用 Windows PowerShell 为委派访问权限合作伙伴管理 Microsoft 365

*本文适用于 Microsoft 365 企业版和 Office 365 企业版。*

委派访问权限 (DAP) 合作伙伴是联合和云解决方案提供商 (CSP) 合作伙伴。 很多是网络或电信提供商。 他们将 Microsoft 365 订阅捆绑到其服务产品中。 在销售 Microsoft 365 订阅时，会自动为其代表 (AOBO) 权限授予对客户的租赁的管理权限，以便他们可以管理和报告这些租赁。 这些任务在 Microsoft 365 管理中心中非常困难。 使用适用于 Microsoft 365 的 PowerShell 可以更轻松地执行管理任务，例如：
- 列出所有客户 **TenantIds** 及其域 
- 确定客户租赁中的所有用户及其分配的许可证
> [!NOTE]
> 某些管理任务只能在 PowerShell 中完成。

下面的文章展示了联合和 CSP 合作伙伴如何使用 PowerShell 来管理其客户租赁：

- [使用 Windows PowerShell 为委派访问权限 (DAP) 合作伙伴将域添加到客户端租赁](add-a-domain-to-a-client-tenancy-with-windows-powershell-for-delegated-access-pe.md)
