# <a name="create-a-new-topic-preview"></a>创建新主题 (预览) 

> [!Note] 
> 本文中的内容适用于 Project Cortex 专用预览版。 [了解更多关于 Project Cortex的信息](https://aka.ms/projectcortex)。

在主题体验中，如果尚未通过索引发现一个主题，或者 AI 技术没有找到足够证据来将主题确定为主题，你可以创建新主题。

## <a name="requirements"></a>要求

若要创建新主题，需要：
- 拥有主题体验许可证。
- 对可以创建或 [**编辑主题的用户具有权限**](https://docs.microsoft.com/microsoft-365/knowledge/topic-experiences-user-permissions)。 知识管理员可以在"知识网络"主题权限设置中授予用户此权限。 

> [!Note] 
> 具有在主题中心管理主题的权限的用户 (知识) 已拥有创建和编辑主题的权限。

## <a name="to-create-a-new-topic"></a>创建新主题：

1. 在主题中心页面上，选择 **"新建"，** 然后选择 **"主题页面"。** 如果您看不到主题中心中的"新建"选项，则您可能没有所需的权限。

    ![新主题](../media/knowledge-management/k-new-topic.png)

2. 在"新主题"页上，可以填写新主题模板上的信息：

    - 在 **"命名本主题"** 部分，键入新主题的名称。
    
    - 在 **"备用** 名称"部分，键入也用于引用主题的名称或缩写词。
    
    - 在 **"简短说明"** 部分，键入主题的一两个句子说明。 此文本将用于关联的主题卡片。
    
    - 在 **"人员** "部分中，键入主题的主题专家的名称。 手动添加到主题中的人员将在主题页面中显示为固定 **人员**。
    
    - 在"**文件和页面"** 部分，选择"添加"，然后在下一页上选择关联的 OneDrive 文件或 SharePoint Online 页面。
    
    - 在"**网站"** 部分，选择"**添加"。** 在  **显示的** "网站"窗格中，选择与主题关联的网站。

    ![新主题页面](../media/knowledge-management/k-new-topic-page.png)
    
3. 如果需要向页面添加其他组件（如文本、图像、Web 部件、链接等），请选择页面中间的画布图标以查找并添加它们。

    ![向页面添加项目](../media/knowledge-management/static-icon.png)

4. 完成后，选择" **发布"** 以发布主题页。 已发布的主题页面会显示在"页面 **"选项卡** 中。

发布文章后，主题名称、备用名称、说明和固定人员将显示给查看文章的所有许可用户。 只有当查看者对项目具有 Office 365 权限时，文件、页面和网站才将显示在主题页面中。 

新主题页面由知识网络感知 *的 Web 部件组成*。 这意味着，当 AI 收集有关主题的详细信息时，这些 Web 部件中的信息将更新为用户提供建议，使页面更有用。
