# <a name="get-started-driving-adoption-of-microsoft-viva-topics"></a>开始推动 Microsoft Viva 主题的采用

开始采用之前，您需要了解信息管理和 Viva 主题中涉及的概念。 下图显示了主题发现和绘制过程中发生的情况：

![Viva 主题体系结构](../media/knowledge-management/topic-management-architecture.png)

- **发现**：用户可以通过主题卡发现他们每天使用的应用的知识 -他们还可以发现 Microsoft 搜索中的主题。
- **组织：** 主题专家 (通过主题) 来优化主题，AI 从他们的意见中学习。 主题中心包含用户可以浏览和专家可以管理的主题页面。
- **标识**：借助 Microsoft Graph 和 (AI) ， (主题、技能等) 自动组织到相关主题中。 使用安全内容对 SharePoint 内容编制索引。
- **扩展**：借助即将 (Microsoft Graph) 连接器，你可以从外部服务和数据存储库中获得知识。

有关详细信息，请查看概述 [了解](topic-experiences-overview.md) 相关信息。

请记住：

- 当更多内容可用时，主题发现得到改进。
- 即使信息以新体验呈现，数据的安全、隐私和位置也得到保留。
- 用户需要许可证才能查看 Viva 主题。
- 发现最初以英语语言内容为内容。

为了帮助准备，请思考以下问题：

- 哪些内容应该用于主题发现？
- 谁将管理主题？
- 谁将看到主题卡片和要点？
- 需要哪些主题？

查看此必备组件列表，以使用 Viva 主题：

|产品或功能 |说明 |
|:-------|:--------|
|具有新式 SharePoint 页面的 SharePoint Online |主题挖掘仅包括 SharePoint 网站上的内容，主题卡片只能在新式页面上显示。|
|Microsoft Graph |可以使用 Microsoft Graph 设置控制主题是包含还是排除在搜索或 Delve 中。 |

## <a name="plan-for-adoption"></a>规划采用

若要规划 Viva 主题的采用，需要：

![规划采用的步骤](../media/knowledge-management/km-adoption-plan-adoption.png)

1. 规划你的方法和目标方案：
    - 考虑定义方案并设置其 [优先级](#target-scenarios)。
    - 考虑 [需要参与](#identify-stakeholders) 的利益干系人及项目团队成员。  
    - 了解你想要推动哪些业务影响以及如何衡量 [成功](#create-a-success-plan)。

2. 吸引你的组织：
    - 确定需要参与的业务组和业务团队，并协调这些业务组和正在规划的方案。
    - 开始考虑如何吸引一些早期采用者获取关键的早期反馈，以便你可以循环访问最佳解决方案。
    - 开始构建社区，并思考这些不同组如何在整个组织中使用 Viva 主题。

3. 培训你的组织：大多数用户都将直观地了解主题的概念，主题卡片如何编译相关信息，以及了解并查看价值。 但是，你可能想要创建针对你自己的文化和组织定制的培训，以显示希望如何使用 Viva 主题。 一些培训资源：
    - [Project Cortex 资源中心](https://aka.ms/projectcortex)。 包括概述和特性信息、录制的办公时间视频和演示文稿，以及有关合作伙伴及其产品/服务的信息。
    - 即将推出针对最终用户的培训视频和帮助。

4. [建立一个冠军网络](#build-a-champion-network)：
    - 你可能已建立实践社区或支持者网络。 这些是社交和宣传以及让同级相互帮助的很好的方法。 他们可以分享非常有价值的成功案例。 他们可以提供建议并产生兴奋感。

### <a name="target-scenarios"></a>目标方案

你需要确定如何在组织中使用 Viva 主题，然后才能成功使用主题。 在以下情况中，信息管理和主题可帮助组织：

- 角色载入&培训：了解新组织的术语、关键项目和文化是载入的重要步骤。 轻松发现主题可帮助新员工快速完成新工作、角色或项目。
- 专业技能查找和信息共享：在管理和共享主题时，组织人员可以更轻松地查找信息和专家，帮助他们完成日常工作。
- 扩展了决策制定并改进了上市时间：当信息和专家易于访问时，您可以更轻松地做出决策并限制项目的关闭时间。

例如，以下是角色载入方案：

- 约旦 (员工) 担任新角色，或已新雇用并刚开始担任角色。 约旦希望尽快参与并提高工作效率。 但是，约旦也需要帮助来查找起点。
- 一位 (Kim) 在 Jordan 创建主题页之前担任过该角色，该页面可帮助新员工和其他任何人查找该信息。
- Kim 是一名 SME，有权查看未确认的主题页面。 未确认的主题页面是 AI 发现和创建的内容的很好的起点，Kim 能够对其进行编辑以添加专家资源、定义和固定其他资源。
- 当 Jordan 在 SharePoint 上阅读新文章时，他们将看到一个主题突出显示，将鼠标悬停在其中，以快速获取术语的定义以及与更多问题联系的联系人。 以前，约旦可能必须四处寻找此信息，并联系同事，看看谁要询问某些信息。
- 通过主题公开此信息可能很强大，因为尽管此信息以前可能可用，但它可能已被孤立且难以找到。 将其引入到约旦使用的应用程序，并帮助他们找到这些专家还可以推动参与感和社区感。 它还可以帮助他们在处理新角色时感觉更加具有能力。

确定方案后，你可以确定这些方案的优先级：

确定其优先级的一个方法就是，在显示影响与轻松实施的网格中绘制方案。 查找影响很大且易于实现的方案，将这些方案作为首要任务。 低影响和难以实现的方案是最低优先级。 当你快速获得成功且影响很大、易于实现的方案时，它可以帮助用户感到兴奋，并查看使用主题的可能性。

![高影响、易于实现的方案具有高优先级](../media/knowledge-management/topics-prioritize-scenarios.png)

选择几个主要方案，最初关注一下，与早期采用者合作，获取一些反馈，然后分步推出。 这样，你可以循环访问、改进和获取反馈，以便随着时间的推移提高采用率。 

### <a name="identify-stakeholders"></a>确定利益干系人

确定项目的利益干系人。 关键角色是执行发起人、成功所有者和支持者。

|角色 |Responsibilities |Department |
|:-------|:-------|:--------|
| 执行发起 ()    | 向公司传达高级愿景和价值   |  管理层领导   |
| 项目主管 ()  | 监督整个启动执行和推出过程 | 项目管理 |
| 知识管理员| 设置和配置 Viva 主题 | IT 部门 |
| 知识经理 | 管理主题并监督分类 | 任何部门 |
| 分类管理员 | 监督分类 | 任何部门 |
| 主题专家和主题参与者 | 生成或审阅主题和说明 | 任何部门 |
| 支持者 | 帮助宣传和管理投诉处理 | 任何部门 (员工)  |
| 租户管理员 | 配置租户级设置 | IT 部门 |
| Power Platform 管理员| 配置公共数据服务环境 | IT 部门 |
| 搜索管理员或管理员 | 配置搜索设置 | IT 部门 |

在大型组织中，你可能还具有这些角色中的多个人员，并且你需要推动他们之间的协调。 在规模较小的公司中，一个人可能担任其中几个角色。 不同角色可能更涉及项目的不同阶段。 例如，租户管理员更参与设置功能，而主题专家和冠军不会参与，直到你开始定义主题。
 
尽管我们建议在整个推出过程中完成其中每个角色，但你可能会发现，你无需全部开始使用已确定的解决方案。

### <a name="create-a-success-plan"></a>创建成功计划

使用这些指示器衡量贵组织中 Viva 主题的成功。 看：

1. 主题用法：
      - 主题印象
      - 主题数量 – 已确认和未确认在特展主题列表中。
      - 已发布的主题页数。
1. 主题卡片中的最终用户反馈。
1. 执行员工满意度调查。 Viva 主题应提高员工查找信息的能力，以便找到收集他们有关该体验的输入和反馈的方法。
1. 对搜索分析的正面影响。 由于主题出现在搜索体验中，随着时间的推移，您可能会发现放弃搜索的速率较低，因为用户更容易找到搜索中的主题。 

### <a name="build-a-champion-network"></a>构建一个冠军网络

在组织中建立一个冠军网络。 冠军很重要，因为他们可以：

- 在其团队内创建影响圈
- 驱动主题管理&维护

你可以从不同角色（知识经理和主题专家）中招募冠军。

许多冠军网络都使用 Yammer 作为他们的平台。 在 Yammer 中，用户可以发布问题并获取答案，并分享成功案例。 很难单独指出这一点，因此你可以依赖整个公司中的人员网络向同事提供建议，并展示其团队如何使用 Viva 主题，以便其他团队可以考虑自己的方案。

某些组织使用黑客攻击 (正式或非正式、虚拟或) 来收集一组人员，以处理特定项目。 例如，你可以集合主题专家，让他们一起协作，共同制定一组主题页面。

考虑如何识别你的冠军。 奖励他们的活动，给予他们一些认可，并生成可见的社区感觉和参与，以便他们感觉自己正在做贡献，并且也会从他们的投资获得一些收益。

现在，你已准备好推出，想要确保鼓励持续参与。

- 为冠军维护活动的 Yammer 组。
- 共享成功案例。
- 定期主持参与活动以共享情景或引入新功能。
- 设置人员挑战并运行竞争。

## <a name="next-steps"></a>后续步骤

准备好推出 Viva 主题后，你需要让人员参与。

- 开始引入功能集，让他们考虑其方案。
- 收集利益干系人并创建方案。
- 推动社区，并考虑如何吸引他们。
- 然后完成准备步骤。 其中一些可能是技术准备情况，一些业务准备情况。
- 最后，社交和推广。
