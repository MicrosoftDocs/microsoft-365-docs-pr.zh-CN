# <a name="get-started-driving-adoption-of-microsoft-viva-topics"></a>推动采用 Microsoft Viva 主题入门

在开始采用之前，你需要了解知识管理和 Viva 主题中涉及的概念。 下图显示了主题发现和绘制期间会发生什么情况：

![Viva 主题体系结构](../media/knowledge-management/topic-management-architecture.png)

- **发现**：用户可以通过主题卡片发现他们每天使用的应用的知识 -他们还可以在 Microsoft 搜索中发现主题。
- **组织：** 主题专家 (，) 主题页面优化主题，AI 从他们的意见中学习。 主题中心包含用户可以浏览和专家可以管理的主题页面。
- **标识**：使用 Microsoft Graph 和 (AI) ， (知识、技能等) 自动组织到相关主题中。 使用安全内容对 SharePoint 内容编制索引。
- **扩展**：即将推出 Microsoft Graph (连接器) ，你可以从外部服务和数据存储库中获得知识。

有关详细信息，请查看概述 [了解](topic-experiences-overview.md) 简介。

请记住：

- 当更多内容可用时，主题发现得到改进。
- 即使信息以新体验呈现，数据的安全、隐私和位置也得到保留。
- 用户需要许可证才能查看 Viva 主题。
- 发现最初以英语语言内容为内容。

为了帮助准备，请思考以下问题：

- 哪些内容应该用于主题发现？
- 谁将管理主题？
- 谁将看到主题卡片和突出显示？
- 哪些主题是预期主题？

查看此必备组件列表，以使用 Viva 主题：

|产品或功能 |说明 |
|:-------|:--------|
|具有新式 SharePoint 页面的 SharePoint Online |主题挖掘仅包括 SharePoint 网站上的内容，主题卡片只能在新式页面上显示。|
|Microsoft Graph |可以使用 Microsoft Graph 设置控制主题是包含还是排除在搜索或 Delve 中。 |

## <a name="plan-for-adoption"></a>规划采用

若要规划 Viva 主题的采用，您需要：

![规划采用的步骤](../media/knowledge-management/km-adoption-plan-adoption.png)

1. 规划你的方法和目标方案：
    - 考虑定义方案并设置 [其优先级](#target-scenarios)。
    - 考虑 [需要参与](#identify-stakeholders) 的利益干系人及项目团队成员。  
    - 了解你想要推动哪些业务影响以及如何衡量 [成功](#create-a-success-plan)。

2. 吸引你的组织：
    - 确定需要参与的业务组和业务团队，并协调这些业务组和正在规划的方案。
    - 开始考虑如何吸引一些早期采用者获取关键的早期反馈，以便你可以循环访问到最佳解决方案。
    - 开始构建社区，并考虑这些不同组如何在组织中使用 Viva 主题。

3. 培训组织：大多数用户都将直观地了解主题的概念，以及主题卡片如何编译相关信息，以及了解和查看价值。 但是，你可能想要创建针对你自己的文化和组织定制的培训，以显示您希望如何使用 Viva 主题。 一些培训资源：
    - [Project Cortex 资源中心](https://aka.ms/projectcortex)。 包括概述和特性信息、录制的办公时间视频和演示文稿，以及有关合作伙伴及其产品/服务的信息。
    - 即将推出针对最终用户的培训视频和帮助。

4. [建立一个冠军网络](#build-a-champion-network)：
    - 你可能已建立实践社区或支持者网络。 这些是社交和宣传以及让同级相互帮助的不错方法。 他们可以分享有价值的成功案例。 他们可以提供建议并产生情绪。

### <a name="target-scenarios"></a>目标方案

确定您希望如何在组织中使用 Viva 主题，以便成功使用它们。 在以下情况中，知识管理和主题可帮助组织：

- 角色载入&培训：了解新组织的术语、关键项目和文化是载入的重要步骤。 轻松发现主题可帮助新员工快速上手完成新工作、角色或项目。
- 专业技能查找和信息共享：在管理和共享主题时，组织人员可以更轻松地查找信息和专家，以帮助他们在日常工作。
- 扩展了决策制定并改进了上市时间：当信息和专家易于访问时，您可以更轻松地做出决策并限制项目的时间。

#### <a name="example-scenario-for-role-onboarding"></a>角色载入的示例方案

HR 经理需要向新员工提供相关信息，帮助他们快速上手加入公司及其团队。 他们希望将其指向需要快速高效地载入的正确资源、文档和团队成员。 他们希望寻找一种解决方案，让新员工能够快速找到他们需要的信息，而无需在多个存储库中搜索或离开他们已在使用的应用程序。

例如：

- 约旦 (员工) 担任新角色，或已新雇用并刚开始担任角色。 约旦希望尽快参与并提高工作效率。 但是，约旦还需要帮助寻找起点。
- 一个 (在) 创建主题页之前担任该角色的同事 Kim，该页面可帮助新员工和查找该信息的任何人。
- Kim 是一名 SME，有权查看未经确认的主题页面。 未确认的主题页面是 AI 发现和创建的内容的很好的起点，Kim 能够编辑它们以添加专家资源、定义和固定其他资源。
- 当 Jordan 在 SharePoint 上阅读新文章时，他们将看到一个主题突出显示，并悬停在该主题上方，以快速获取术语的定义以及与更多问题联系的联系人。 以前，约旦可能必须四处寻找此信息，并联系同事，看看谁要询问一些信息。
- 通过主题公开此信息可能很强大，因为尽管此信息之前可能可用，但它可能已被孤立且难以查找。 将其引入到约旦使用的应用程序，并帮助他们找到这些专家还可以促进参与和社区感。 它还可以帮助他们在处理新角色时感觉更加具有能力。

自动执行此方案时，可以确保：

- 新员工可以快速与正确项目上的合适人员联系。
- 新员工可即时访问工作流中的最新项目信息。
- 搜索时间大大减少。
- 载入时间大大减少。

#### <a name="example-scenario-for-customer-call-center"></a>客户呼叫中心的示例方案

你可以使客户服务快速查找文件和专家，以帮助解决异常税务问题，并快速记录答案，以便其他人通过 Viva 主题轻松访问。

例如，支持代表需要快速查找知识库文章、文档和策略，以便他们可以为客户提供支持。 他们希望在正确的时间找到正确的信息，而无需手动筛选多个数据库、存储库或应用程序，或调度调用。 他们正在寻找一个解决方案，允许他们在对话流中保持主呼叫提示和访问策略、法规和准则，以便可以快速响应问题并更新案例。

使用 Viva 主题自动执行此方案时，可以确保：

- 支持呼叫次数减少。
- 到第 2 层和第 3 层支持的升级会减少。
- 给定案例上的回叫次数减少。
- 客户满意度提高。

#### <a name="prioritize-your-scenarios"></a>确定方案优先级

确定方案后，您可以确定这些方案的优先级：

确定其优先级的一种方式是在显示影响与轻松实施的网格中绘制方案。 查找具有高影响且易于实现的方案，将这些方案作为首要任务。 低影响和难以实现的方案是最低优先级。 当你快速获得成功且影响很大、易于实现的方案时，它可以帮助用户感到轻松，并查看使用主题的可能性。

![高影响、易于实现的方案具有高优先级](../media/knowledge-management/topics-prioritize-scenarios.png)

选择一些最初重点关注的关键方案，与早期采用者合作，获取一些反馈，然后分步推出。 这样，你可以循环访问、改进和获取反馈，以便随着时间的推移提高采用率。

### <a name="identify-stakeholders"></a>确定利益干系人

确定项目的利益干系人。 关键角色是执行发起人、成功所有者和支持者。

|角色 |Responsibilities |Department |
|:-------|:-------|:--------|
| 执行发起 ()    | 向公司传达高级愿景和价值   |  行政领导   |
| 项目主管 ()  | 监督整个启动执行和推出过程 | 项目管理 |
| 知识管理员| 设置和配置 Viva 主题 | IT 部门 |
| 知识经理 | 管理主题并监督分类 | 任何部门 |
| 分类管理员 | 监督分类 | 任何部门 |
| 主题专家和主题参与者 | 生成或审阅主题和说明 | 任何部门 |
| 支持者 | 帮助宣传和管理投诉处理 | 任何部门 (员工)  |
| 租户管理员 | 配置租户级别设置 | IT 部门 |
| Power Platform 管理员| 配置常见数据服务环境 | IT 部门 |
| 搜索管理员或管理员 | 配置搜索设置 | IT 部门 |

在较大的组织中，你可能还具有多个担任这些角色的人，并且你将需要推动他们之间的协调。 在一家较小的公司中，一个人可能担任其中几个角色。 不同角色在项目的不同阶段中可能涉及更多。 例如，租户管理员更参与设置功能，而主题专家和支持者不会参与，直到你开始定义主题。
 
尽管我们建议在整个推出过程中完成其中每个角色，但你可能会发现，你无需所有角色都开始使用已识别的解决方案。

### <a name="create-a-success-plan"></a>创建成功计划

使用这些指示器衡量组织中 Viva 主题的成功。 看：

1. 主题用法：
      - 主题印象
      - 主题数量 – 已确认和未确认在特用主题列表中。
      - 已发布的主题页数。
1. 来自主题卡的最终用户反馈。
1. 执行员工满意度调查。 Viva 主题应提高员工查找信息的能力，以便找到收集他们有关该体验的输入和反馈的方法。
1. 对搜索分析的正面影响。 由于主题出现在搜索体验中，随着时间的推移，你可能会看到放弃的搜索的较低速率，因为用户更容易在搜索中查找主题。 

### <a name="build-a-champion-network"></a>构建一个支持者网络

在组织中建立一个支持者网络。 冠军非常重要，因为他们可以：

- 在其团队内创建一个影响圈
- 驱动主题管理&维护

你可以从不同角色（知识经理和主题专家）中招募冠军。

许多支持者网络都使用 Yammer 作为他们的平台。 在 Yammer 中，用户可以发布问题并获取答案，并分享成功案例。 很难单独显示该词，因此你可以依赖整个公司中的人员网络向同事提供建议，并展示他们的团队如何使用 Viva 主题，以便其他团队能够考虑自己的方案。

某些组织使用黑客 (正式或非正式、虚拟或) 来收集一组人员以处理特定项目。 例如，你可以收集主题专家，让他们一起共同制定一组主题页面。

考虑如何识别你的冠军。 奖励他们的活动，给予他们一些认可，并生成可见的社区感觉和参与，以便他们感觉自己正在做贡献，并且还从投资中获得一些收益。

现在，你已准备好推出，希望确保鼓励持续参与。

- 为支持者维护活动的 Yammer 组。
- 分享成功案例。
- 定期主持参与活动以共享情景或引入新功能。
- 设置人员挑战并运行竞争。

## <a name="next-steps"></a>后续步骤

准备好推出 Viva 主题后，你需要让人员参与。

- 开始引入功能集，让他们考虑其方案。
- 收集利益干系人并创建方案。
- 推动社区，并考虑如何吸引他们。
- 然后完成准备步骤。 其中一些可能是技术准备情况，还有一些业务准备情况。
- 最后，进行社交和推广。
