## <a name="microsoft-365-defender"></a>Microsoft 365 Defender

[Microsoft 365 Defender](./microsoft-threat-protection.md) 是一个统一的攻破前和入侵后企业防御套件。 它在本机协调跨终结点、标识、电子邮件和应用程序进行检测、预防、调查和响应，以提供针对复杂攻击的集成保护。

Microsoft 365 Defender 将 [Microsoft Defender for Endpoint、Microsoft](https://www.microsoft.com/microsoft-365/windows/microsoft-defender-atp)Defender for Office [365、Microsoft](https://www.microsoft.com/microsoft-365/exchange/advance-threat-protection) [Defender for Identity、Azure](https://azure.microsoft.com/features/azure-advanced-threat-protection/) [Active Directory Identity Protection](/azure/active-directory/identity-protection/overview-identity-protection)和 Microsoft Cloud App [Security](https://www.microsoft.com/microsoft-365/enterprise-mobility-security/cloud-app-security) 的功能合并到单个解决方案中。

### <a name="mitre-demonstrated-real-world-detection-response-and-protection-from-advanced-attacks"></a>MITRE：演示了实际检测、响应和高级攻击保护

MITRE 的测试方法的核心是模拟实际攻击，以了解解决方案能否充分检测和响应它们。 虽然测试侧重于终结点检测和响应，但 MITRE 的模拟 APT29 攻击跨越多个攻击域，从而创造超越终结点保护之外的支持防御者的机会。 Microsoft 使用 Microsoft 365 Defender 在终结点以外扩展了防御者可见性。

- ATT&基于 CK 的 Microsoft 365 Defender 评估 - 2020 年 5 月： [在现实检测中处于领先水平](https://www.microsoft.com/security/blog/2020/05/01/microsoft-threat-protection-leads-real-world-detection-mitre-attck-evaluation/)

    Microsoft 365 Defender 在攻击链阶段提供近 100% 的覆盖范围。 它可引导对攻击者活动进行开箱即用可见性。 可见性大大减少了依赖特定配置更改的安全运营中心和供应商解决方案的手动工作。 Microsoft 365 Defender 的可见性差距最小，这会降低攻击者未经检测的运行能力。

[Microsoft Defender 防病毒](/windows/security/threat-protection/microsoft-defender-antivirus/microsoft-defender-antivirus-in-windows-10) 在独立测试中一直执行高性能，显示它在防病毒市场中如何成为首要选择。 请记住，这些测试仅提供防病毒结果，不测试其他安全保护。

Microsoft Defender 防病毒是[](https://www.youtube.com/watch?v=Xy3MOxkX_o4) [Microsoft Defender for Endpoint Windows 10](/windows/security/threat-protection/microsoft-defender-atp/microsoft-defender-advanced-threat-protection)安全堆栈中的下一代保护功能，可解决当今最新且最复杂的威胁。 在某些情况下，客户可能甚至不知道他们受到保护，因为网络攻击在市场活动启动后数 [毫秒就停止](https://cloudblogs.microsoft.com/microsoftsecure/2018/03/07/behavior-monitoring-combined-with-machine-learning-spoils-a-massive-dofoil-coin-mining-campaign)。 这是因为 Microsoft Defender 防病毒和其他终结点保护平台 ([EPP ](https://www.microsoft.com/security/blog/2019/08/23/gartner-names-microsoft-a-leader-in-2019-endpoint-protection-platforms-magic-quadrant/) 支持) Defender for Endpoint 检测并阻止首次看到恶意软件。 他们使用 [机器学习](https://cloudblogs.microsoft.com/microsoftsecure/2018/06/07/machine-learning-vs-social-engineering)、 [人工智能](https://cloudblogs.microsoft.com/microsoftsecure/2018/02/14/how-artificial-intelligence-stopped-an-emotet-outbreak)、行为分析和其他高级技术。

## <a name="endpoint-detection--response"></a>终结点检测&响应

适用于终结点的 Microsoft Defender [终结点检测和响应](/windows/security/threat-protection/microsoft-defender-atp/overview-endpoint-detection-response) 功能提供接近实时且可操作的高级攻击检测。 安全分析员可以有效地确定警报的优先级，了解整个泄露范围，并采取响应措施来修正威胁。

Microsoft Defender for Endpoint 的 EDR 和终结点保护功能已收到来自行业测试和发布的积极结果。 [SC Labs 在](https://www.scmagazine.com/home/reviews/sc-product-reviews-endpoint-security/) 2020 年 6 月评估了终结点安全工具，并针对终结点 [5/5 星](https://www.scmagazine.com/review/microsoft-defender-advanced-threat-protection/)提供了 Microsoft Defender。 他们调用了适用于 Endpoint 的 Microsoft Defender，能够使用一整套安全性功能保护组织免受现代威胁环境的影响。 SC Labs 还将终结点安全解决方案标识为全面统一的解决方案。 他们还认可终结点保护与终结点检测和响应功能的融合，因为解决方案现在完全覆盖攻击链。

### <a name="mitre-industry-leading-optics-and-detection-capabilities"></a>MITRE：业界领先的光学镜头和检测功能

MITRE 测试了产品检测目标攻击组 APT3 (或 UPS) 所使用的技术的能力。 为了隔离检测功能，所有保护和防护功能都已关闭。 Microsoft 很高兴成为第一批注册基于 ATT 和 CK 框架的 MITRE 评估的 EDR&之一。 如今，该框架被普遍视为最全面的攻击者技术和策略目录。

- ATT&CK 的 Microsoft Defender for Endpoint 评估 - 2018 年 12 月：领先光学和[检测功能](https://www.microsoft.com/security/blog/2018/12/03/insights-from-the-mitre-attack-based-evaluation-of-windows-defender-atp/)  |  [分析](https://techcommunity.microsoft.com/t5/Windows-Defender-ATP/MITRE-evaluation-highlights-industry-leading-EDR-capabilities-in/ba-p/369831)

    Microsoft Defender for Endpoint 在整个攻击链中全面覆盖攻击者技术。 要点包括遥测的广度、威胁智能的优势，以及通过机器学习、启发和行为监视的高级自动检测。

## <a name="to-what-extent-are-tests-representative-of-protection-in-the-real-world"></a>测试在什么程度上代表现实世界中的保护？

独立安全行业测试旨在以无偏不及的方式评估最佳的防病毒和安全产品。 但是，除了本文重点介绍的评估中经过测试外，Microsoft 发现一组更广泛的威胁。 在平均月份中，Microsoft 安全产品识别超过 1 亿个新威胁。 即使独立测试人员可以获取并测试这些威胁的 1%，在 20 个或 30 个产品中，该测试也是一百万个。 换句话说，恶意软件形势的广泛程度使得评估针对实际威胁的保护质量变得困难。

Microsoft Defender for Endpoint 中的[](https://cloudblogs.microsoft.com/microsoftsecure/2017/12/11/detonating-a-bad-rabbit-windows-defender-antivirus-and-layered-machine-learning-defenses)功能提供了未纳入行业防病毒测试的额外保护层，并解决了一些最新且最复杂的威胁。 将 AV 与 Defender for Endpoint 的其余部分隔离可部分了解 Microsoft 的安全堆栈在现实世界中的运行方式。 例如，攻击面减少和终结点检测&响应功能可以帮助防止恶意软件首先进入设备。 我们已经证明 [，Microsoft Defender for Endpoint 组件](https://query.prod.cms.rt.microsoft.com/cms/api/am/binary/RE2ouJA) 可以捕获 Microsoft Defender 防病毒在这些行业测试中错过的示例。 它更代表 Microsoft 安全套件如何有效地保护现实世界中的客户。
