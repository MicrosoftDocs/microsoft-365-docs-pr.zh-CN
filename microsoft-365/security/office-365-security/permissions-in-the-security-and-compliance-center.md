# <a name="permissions-in-the-security--compliance-center"></a>安全与合规中心内的权限

通过安全&合规性中心，可以向执行合规性任务（例如，设备管理、数据丢失防护、电子数据展示、保留等）的人员授予权限。 这些人员只能执行你明确授予其访问权限的任务。 若要访问安全与&合规中心，用户必须是全局管理员或一个或多个安全与合规中心&组的成员。

安全与合规&根据 RBAC 和域解决方案和服务 (基于) 访问控制。 该模型与 Exchange 所使用的权限模型相同，因此，如果你熟悉 Exchange，在安全 & 合规中心授予权限将非常相似。 但是，请务必记住，Exchange 角色组和安全 & 合规中心角色组不共享成员身份或权限。 尽管二者都有一个组织管理角色组，但这两个角色组并不相同。 角色组授予的权限以及角色组的成员都有区别。 以下是安全与&中心角色组的列表。

![安全与合规中心中的权限&页](../../media/992c20ca-e82e-497c-9c8d-6fab212deb80.png)

## <a name="relationship-of-members-roles-and-role-groups"></a>成员、角色和角色组之间的关系

**角色**可授予执行一组任务的权限；例如，事例管理角色可以让人员处理电子数据展示事例。

角色组 **是一** 组角色，可让人们在"安全与合规&"工作;例如，合规性管理员角色组包括案例管理、内容搜索和组织配置 (的角色以及其他) 因为作为合规性管理员的人员需要这些权限才能执行其工作。

安全与&中心包括适用于最常见任务的默认角色组以及需要分配给人员的功能。 建议你只将单个用户 **作为成员** 添加到默认角色组中。

![显示角色组与角色和成员之间关系的图表](../../media/2a16d200-968c-4755-98ec-f1862d58cb8b.png)

## <a name="permissions-needed-to-use-features-in-the-security--compliance-center"></a>使用安全与合规中心中的&功能所需的权限

下表列出了安全 & 合规中心中提供的默认角色组，以及默认情况下分配给角色组的角色。 若要向用户授予权限以执行合规性任务，请将用户添加到适当的安全与&合规中心角色组。

安全与合规中心 &内的管理权限只能让用户访问安全与合规中心自 &身提供的合规性功能。 如果您要授予不在安全 & 合规中心的其他合规性功能的权限，例如 Exchange 邮件流规则 (也称为传输规则) ，需要使用 Exchange 管理中心。

若要了解如何授予对安全和合规性&的访问权限，请查看 [如何向用户授予对 Microsoft 365 合规管理中心的访问权限](grant-access-to-the-security-and-compliance-center.md)。
