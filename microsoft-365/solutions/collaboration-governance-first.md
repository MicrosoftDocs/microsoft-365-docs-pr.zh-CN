# <a name="governance-first"></a>管理先行

始终最好的方法是，通过明确的愿景开始迁移到 Microsoft 365：

- 你的关键业务目标是什么？

- 工具和功能如何为整个组织和各个员工提供价值？

- 您将如何衡量是否成功？

明确的远景表述针对不可避免的决策权衡提供了关键指导，您需要考虑一下您的治理计划。 Formality 的程度和记录调控计划所需的深度应与您要实现的结果一致。 因此，远景可为管理中的上下文和投资提供框架。

Microsoft 365 云为支持业务的技术创建了新的范例。 这些新范例改变了采用、管理和管理这些技术的方式。 

Microsoft 365 调控应补充控制本地环境的现有策略。  但是，由于云使新商机能够支持业务，因此了解这些机会并考虑在用户目标、业务风险和行业和管理法规要求之间实现适当平衡的管理方法是非常重要的。

首先考虑调控，即开始将你的旅程迁移到云，并通过一些关键决策实现总体成功的最佳位置。 其中一些决策包括：

- 是否支持外部共享以及在哪些情况下？ [了解有关外部共享的更多信息](https://docs.microsoft.com/microsoft-365/solutions/collaborate-with-people-outside-your-organization)。

- 如何设置组、网站和团队并将其设置？ [了解如何控制谁可以创建组](https://docs.microsoft.com/microsoft-365/admin/create-groups/manage-creation-of-groups)。

- 哪些类型的内容可以在每种类型的环境中发布–哪些功能需要启用以防止用户在不适当的位置意外发布信息？ [了解 Microsoft 365 中的信息保护](https://docs.microsoft.com/microsoft-365/compliance/protect-information)。

没有对这些问题的完美答案–正确的答案是与组织的风险和好处相平衡的答案。

## <a name="what-does-governance-mean-for-microsoft-365"></a>对于 Microsoft 365，调控的含义是什么？

Microsoft 365 的治理规划是为了确保您在保护关键信息资产时不会将风险降至最低。 治理包括几个关键领域，每个关键领域都需要进行规划、协调，并适应不断变化的组织需求和技术的发展。 

- **运营保证** –保持平台正常运行，以确保性能。 这在很大程度上是由 Microsoft 提供的大部分工作所组成的 IT 任务。

- **信息保证** –在整个生命周期中管理内容，将信息视为一项资产，包括记录管理、合规性和安全性。 信息保证需要了解业务目标和管理法规要求。 理想情况下，您希望使用自动策略，以防止用户在需要做出明智决定时对其发出错误或警告他们。 如果不能通过策略实施，则需要规划教育和培训。

- **成果保证** -筹划网站或团队所有者在正确的方向上实现业务成果。 成果保证在很大程度上是指为内容作者和网站、组和团队所有者提供指导，确保适当的培训能够确保他们了解如何在整体组织标准和实践中做出正确的选择来获得预期的成果。

治理规划应首先进行，但必须是在迭代过程中持续重新访问的内容。 随着时间的推移，数据空间随时间推移而变化，您将需要重新访问您的管理决策，以适应不断变化的业务需求和功能可用性。 您可能会发现需要进行更多的监督，以确保符合标准。 您可能会发现，您需要较少的监督，以鼓励更具创造性的核心功能应用。 您可能会发现需要更新角色和职责，以反映解决方案中的变化或用户使用不同应用程序的方式的变化。

例如，Microsoft 365 的一项关键功能是能够轻松地与外部用户和组织协作。 有些行业中的一些组织选择阻止所有外部共享。 其他用户在特定方案或特定组织中启用外部共享。 在外部共享时，不会进行不当或错误的治理决策–而且在许多不同的情况下，您可以在某些情况下在 Microsoft 365 中启用不同的设置，在某些情况下允许共享，并阻止在其他情况下共享。 但是，如果只是开始从本地环境迁移到云，请考虑外部共享决策，并了解与不同可用选项相关的优势和风险，这一点非常重要。

## <a name="why-think-about-governance-first"></a>为什么要首先考虑治理？

您在 Microsoft 365 方面的投资仅与您启用的内容和体验的价值一样好，因此在您的旅程开始时考虑到公司治理，可确保在您有机会了解和评估每个决策的影响之前，不会锁定或启用过多的操作。 有多个 "旋钮和拨号"，您可以打开 Microsoft 365 管理中心以及各个网站和团队。 有效的治理计划是实现业务目标的关键因素，但管辖是关于平衡风险和好处的。 如果我们将所有内容都锁定，则当用户需要执行此操作时，将会找到解决规则的方法。

首先考虑公司治理，让你能够：

- **平衡风险和好处**。 在部署的早期阶段仔细检查调控性决策，使组织能够了解整个环境–业务、法规、法律、法规遵从性，并规划一种解决方案来优化所有结果–平衡风险，有时接受风险和其他时间，选择不接受风险。 如果你事先做出了明智的决策，你将获得更好的成果，而无需大量增加和非托管内容。

- **适应不同的组织和不同类型的内容和方案**。 治理决策是每个组织的独特决策–一种大小不适合所有情况。 每个组织不仅有不同的管理需求，而且对于 Microsoft 365 中的每种内容类型，管理决策通常并不相同。 例如，团队内容的管辖可能不同于 intranet 内容治理和 OneDrive 中的单个内容。

- **与业务优先级对齐**。 若要确定在 Microsoft 365 中构建的解决方案的关键业务优先级，开始考虑公司治理的时间。 这些关键业务成果定义了调控规划的上下文。 这一点很重要，因为您的业务目标将帮助您定义在治理方面需要花费的时间和精力。 例如，如果在整个组织中改进内容发现并不重要，则您可能不需要在实施或规划文件命名约定方面花费过多时间。 另一方面，如果您希望帮助减少整个企业中各个存储库中同一文档的多个版本的实例，则您的管理决策将需要流程和策略和培训，以确保内容作者了解如何命名文件并遵循 "一份文档副本" 的指导，以确保仅在一个位置发布内容。 它还意味着您需要一个过程，以确保在同一目的中不需要创建多个网站或团队。

## <a name="how-should-we-be-communicating-about-governance"></a>我们应如何与公司治理进行通信？

可以通过打开或关闭 Microsoft 365 中的功能来实施许多治理决定。 这有助于强制实施您的治理标准，但可能不会帮助组织中的人员了解他们的可用内容，以及为什么 (或为什么不) 。 

传统上，政府团队创建了长文档，概述了每个治理决策。 遗憾的是，这些较长的文档很少会得到阅读，这意味着依赖于要强制执行的人员的治理要求很少有效。 如果您想了解如何传达关键管理决策，请尽量避免创建针对多个访问群体的长文档。 相反，请考虑以下替代方法：

- **在您创建的解决方案中直接嵌入管理决策**。 如果要确保敏感信息在 Microsoft 365 中受到保护，请实现灵敏度标签以确保您的用户不会意外公开他们不应看到的信息。 阻止无法提供适当保护的应用程序。 [了解有关敏感度标签的详细信息](https://docs.microsoft.com/microsoft-365/compliance/sensitivity-labels)。

- **通过培训强化**。 修改 [microsoft 365 学习途径](https://docs.microsoft.com/office365/customlearning) 等解决方案，以确保通过 microsoft 提供的培训加强组织特定的期望。

- **作为网站（而不是文档）传递**。 在 SharePoint 通信网站中创建您自己的 Microsoft 365 采用中心，以确保您的内容作者和网站和团队所有者不仅了解 "操作方法"，还了解 "应如何"。 创建针对不同业务应用场景的特定主题页面，以提供指导和最佳实践，以便在 SharePoint 和团队和其他应用程序中充分利用或启用不同的功能，以实现业务成果。

## <a name="what-are-the-key-success-factors"></a>关键成功因素是什么？

合并这些关键成功因素以确保成功的公司治理规划 Microsoft 365。

### <a name="identify-your-governance-core-team"></a>确定您的治理核心团队

治理是团队运动。 确保您的核心治理计划团队包含来自业务和 IT 部门的代表。 您的团队可能需要在首次部署开始时更频繁地实现，然后定期查看新功能和新的业务期望。

通过关键管理决策，但不要一次尝试做出每个决策。 制定有关设置和命名和外部访问的关键决策，然后继续执行余下的决定。

除了前面讨论的外部共享之外，针对每个组织的另一个关键决策是如何设置网站和组。 例如，考虑以下情况：

- 由于用户很容易创建 Microsoft 365 组，因此您可能需要打开组创建，以使其不会被请求创建，以代表其他人创建。 若要避免 "组数量剧增"，您可以创建自定义工作流，以便在创建组后立即向组创建者发送电子邮件，并将其作为分组、培训和其他信息。  (例如，"你刚刚创建了一个网站/团队、社区，等等。另一方面，你的责任和链接到了你的培训 ..." ) 另一方面，这取决于你的业务，你可能想要控制谁可以创建组。

- 如果您想要控制 SharePoint 网站的创建方式，可以隐藏 SharePoint 起始页上的 " **创建网站** " 链接。 创建自定义窗体，或创建自定义网站设计和网站脚本，以使用预定义的网站结构自动设置新的新式 SharePoint 网站，其中包括指向您的管理和培训最佳实践的链接。

### <a name="align-decisions-to-business-goals"></a>根据业务目标调整决策

业务成果目标应为您的管理决策的主要驱动因素：

- 了解影响组织的法规要求。 您的一些关键决策和实现要求可能需要与您在其中运行的环境相一致。

- 请不要在 "我们一直在做的事情" 中进行补漏白。 许多组织的云和扩大全球化带来了新的机会。 如果未与业务目标保持一致，则为挑战假设。 您是否可以放宽某些方案或某些人的规则？ 您是否可以使用 Microsoft 365 的功能获取相同的业务成果，而不是完全阻止活动？

- 如果你锁定所有内容，用户将找到解决办法。 请尝试了解您的用户要启用的关键业务方案，并根据需要提供指导和培训。 

- 请考虑结合查看关键网站和活动的活动，以确保网站和团队所有者关注您的管理指南。

### <a name="define-roles-and-responsibilities"></a>定义角色和职责

除了您的治理团队之外，还有其他几个关键角色或团队需要考虑 Microsoft 365 的成功之处。 有些角色可以由同一个人组合或填充，而其他角色可能不适合所有组织。 大多数组织都将 Microsoft 365 作为一个整体发起人，并有一个人作为一个整体或仅针对 intranet 的筹划委员会或365团队。 [了解有关 Microsoft 365 中提供的全面管理员角色和权限的详细信息](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles)。

### <a name="revisit-as-business-and-technology-changes"></a>业务和技术变化的重新访问

Microsoft 365 的治理不是 "一次，完成"。 制定一个计划，以了解 Microsoft 365 中的变化，并根据需要调整您的管理准则。  (这是为什么创建大型调控文档不是一个好主意的重要原因。 企业和技术发生变化。 更新网页非常简单。 更新和重新分发文档要困难得多。 ) 

### <a name="have-a-strategy-for-communicating-governance-policies-and-guidelines"></a>有一种策略，用于交流治理策略和指南

最终，您的治理规划工作是否成功取决于您对组织成员的预期交流程度。 除了前面介绍的建议之外，请考虑以下事项：

- 您能否将 "如何处理" 与您的 "操作方法" 培训结合在一起？ 换句话说，可以为 Microsoft 365 创建 "用户资源中心"，在此可以提供资源和培训以帮助用户提供有效的指导和培训，以帮助用户有效采用 Microsoft 365 吗？ 最佳治理内容提供的指导可帮助所有用户采用并从 Microsoft 365 获得最大价值。

- 是否应为网站和团队所有者创建证书和重新认证？

- 您是否可以为您的组织创建 Microsoft 365 的拥护者计划？ 此外，请考虑加入全球 Microsoft 365 冠军程序，以获取您自己的拥护者的想法和方法，并与 Microsoft 内部和周围的 minded 人员和思维领导者进行连接。
